# 提取方法论（RIA-TV++ 版）

## 概述

借鉴 cangjie-skill 的 RIA-TV++ 流水线，从同一作者的多本小说中提取「写作决策框架」。

**核心理念**：大佬写的不是「文字」，是「决策」。我们要提取决策框架，不是表面特征。

---

## RIA-TV++ 流水线

### RIA 是什么

- **R**（Reading）：读原文，理解内容
- **I**（Interpretation）：用自己的话重写，理解深层含义
- **A**（Appropriation）：应用到新场景，验证可执行性

### TV 是什么

- **T**（Triple Verification）：三重验证
  - 跨书验证：同一规则在 ≥2 本书中出现
  - 频率验证：出现频率 ≥5%
  - 独特性验证：不是所有作者都会这样做

### ++ 是什么

- **E**（Execution）：可执行步骤
- **B**（Boundary）：边界与盲点

---

## 与传统方法的区别

| 维度 | 传统方法 | RIA-TV++ |
|------|---------|----------|
| 提取目标 | 表面特征 | 决策框架 |
| 提取方法 | 统计频率 | 逆向分析决策逻辑 |
| 验证方式 | 无 | 三重验证 |
| 输出内容 | 「短句为主」 | 「如果X则Y」+原文引用+边界 |
| 应用方式 | 按模板写 | 按决策框架思考 |

## 提取流程

```
原文 txt
    │
    ├─→ Phase 1: Adler分析 → 整书理解（结构/解释/批判/应用）
    │
    ├─→ Phase 2: 并行提取 → 8个提取器
    │       ├─→ 心智模型提取器 → mental-models.md
    │       ├─→ 决策启发式提取器 → decision-heuristics.md
    │       ├─→ 节奏直觉提取器 → rhythm-intuition.md
    │       ├─→ 表达DNA提取器 → expression-dna.md
    │       ├─→ 反模式提取器 → anti-patterns.md
    │       ├─→ 书名简介提取器 → synopsis-patterns.md
    │       ├─→ 章纲模板提取器 → chapter-template.md
    │       └─→ 去AI策略提取器 → de-ai-strategy.md
    │
    ├─→ Phase 3: 三重验证 → 跨书+频率+独特性
    │
    ├─→ Phase 4: RIA++构造 → 结构化输出（R+I+A1+A2+E+B）
    │
    └─→ Phase 5: 合成输出 → SKILL.md + meta.json
```

## Phase 1：Adler 分析（整书理解）

**借鉴**：Mortimer Adler《如何阅读一本书》的分析阅读法

### 四步分析

| 步骤 | 问题 | 目的 |
|------|------|------|
| **结构** | 这本书在讲什么？整体结构是什么？ | 理解全书框架 |
| **解释** | 作者在这里做了什么选择？为什么？ | 理解决策逻辑 |
| **批判** | 如果换一种选择会怎样？哪种更好？ | 评估决策质量 |
| **应用** | 这个决策规则可以用在什么场景？ | 提取可执行规则 |

### 精读策略

| 书数 | 策略 |
|------|------|
| 1本 | 精读全书（至少读50%章节） |
| 2-3本 | 精读评分最高的1本 + 其余各读关键章节 |
| 4-6本 | 精读评分最高的1本 + 其余各读前3章+中段3章+后3章 |

### 精读内容

| 位置 | 读什么 | 目的 |
|------|--------|------|
| 前3章 | 开篇方式、破冰方式、人设展示 | 提取开篇决策 |
| 第1/4处 | 1-2章 | 提取中段节奏 |
| 第1/2处 | 1-2章 | 提取冲突升级方式 |
| 第3/4处 | 1-2章 | 提取高潮铺垫方式 |
| 最后3章 | 结局、伏笔回收、情绪终点 | 提取结局决策 |
| 转折点 | 2-3个关键转折场景 | 提取决策规则 |

## Phase 2：并行提取（8个提取器）

### 提取器 1：心智模型提取器

**输入**：Phase 1 的精读场景

**提取维度**：
- 故事观：作者怎么看「好故事」
- 角色观：作者怎么看「好角色」
- 冲突观：作者怎么看「好冲突」
- 爽感观：作者怎么看「爽」

**方法**：
1. 选3-5个「关键场景」
2. 对每个场景做「四步分析」（结构/解释/批判/应用）
3. 提取「心智模型」

**输出**：`references/mental-models.md`

### 提取器 2：决策启发式提取器

**输入**：Phase 1 的精读场景

**提取维度**：
- 开篇决策规则
- 章尾断点决策规则
- 信息揭示决策规则
- 节奏控制决策规则
- 角色决策规则
- 冲突设计决策规则
- 爽点设计决策规则

**方法**：
1. 找「转折点」——情绪/节奏发生变化的地方
2. 分析「触发条件」——什么导致了变化
3. 提取「决策规则」——如果X，则Y

**输出**：`references/decision-heuristics.md`

### 提取器 3：节奏直觉提取器

**输入**：Phase 1 的精读场景

**提取维度**：
- 读者预期管理规则
- 情绪曲线控制规则
- 信息差设计规则
- 承诺兑现节奏规则

**方法**：
1. 从多本书的「转折点」分析
2. 分析「读者预期」——读者此刻期待什么
3. 分析「情绪曲线」——应该上升还是下降
4. 分析「信息差」——读者知道什么不知道什么

**输出**：`references/rhythm-intuition.md`

### 提取器 4：表达DNA提取器

**输入**：Phase 1 的精读场景

**提取维度**：
- 信息密度规则
- 感官选择规则
- 情绪表达规则
- 对话原则规则
- 句式选择规则

**方法**：
1. 分析「信息密度」——每段传递几个信息点
2. 分析「感官选择」——优先用哪个感官
3. 分析「情绪表达」——直接命名还是行为展示
4. 分析「对话原则」——对话的目的是什么

**输出**：`references/expression-dna.md`

### 提取器 5：反模式提取器

**输入**：Phase 1 的精读场景 + 全文扫描

**提取维度**：
- 词汇反模式
- 句式反模式
- 结构反模式
- 情节反模式
- 角色反模式
- 对话反模式

**方法**：
1. 找「缺失」——作者从不做的事
2. 分析「为什么」——为什么不做
3. 提取「反模式」——决策禁区

**输出**：`references/anti-patterns.md`

### 提取器 6：书名简介提取器

**输入**：每本书的书名 + 简介 + 标签

**提取维度**：
- 书名命名规则
- 简介开头策略
- 简介信息顺序
- 简介结尾钩子类型
- 标签选择规则

**方法**：
1. 统计多本书的书名字数、结构（动宾/偏正/主谓）
2. 分析简介的开头方式（背景/反差/危机）
3. 分析简介的信息排列顺序
4. 分析结尾钩子的类型

**输出**：`references/synopsis-patterns.md`

### 提取器 7：章纲模板提取器

**输入**：`sources/*.txt`

**提取维度**：
- 章节命名模式
- 章节结构模式
- 核心事件密度
- 章首钩子分布
- 章尾断点策略

**方法**：
1. 抽取全书章名，统计字数、结构分布
2. 抽取每章首段和末段，分析开篇方式和断点方式
3. 统计每章核心事件数量

**输出**：`references/chapter-template.md`

### 提取器 8：去AI策略提取器

**输入**：`sources/*.txt` + `de-ai-modules/*.md`

**提取维度**：
- 连接词频率
- 句长分布
- 副词频率
- 标点习惯
- 语序规律
- 数字精确度
- 得地的混用
- 情感表达模式

**方法**：
1. 从原文中抽样5-10段分析作者写作习惯
2. 对照 de-ai-modules 的每个模块触发条件
3. 选择 3-5 个匹配度最高的模块

**输出**：`references/candidates/de-ai-strategy.md`

## Phase 3：三重验证

### 验证规则

| 验证 | 标准 | 不通过处理 |
|------|------|-----------|
| **跨书验证** | 同一决策规则在 ≥2 本书中出现 | 标记为「可能偶然」，降低权重 |
| **频率验证** | 出现频率 ≥5%（非偶发） | 标记为「低频特征」，不写入 |
| **独特性验证** | 不是所有作者都会这样做 | 标记为「通用规则」，不写入 |

### 通用规则黑名单

- 每章要有钩子（太通用）
- 对话要有潜台词（太通用）
- 角色要有动机（太通用）
- 冲突要有升级（太通用）

### 单本书例外

如果只有1本书，跳过跨书验证，只做频率验证和独特性验证。

## Phase 4：RIA++ 构造

每个决策规则必须包含：

| 维度 | 说明 |
|------|------|
| **R**（原文引用） | 原文中的一段话，标注出处 |
| **I**（决策解读） | 作者在这里做了什么选择，为什么 |
| **A1**（书中案例） | 这个规则在书中的具体应用 |
| **A2**（触发场景） | 什么条件下使用这个规则 |
| **E**（可执行步骤） | 具体怎么执行 |
| **B**（边界与盲点） | 这个规则的局限性 |

详见 `ria-construction.md`
