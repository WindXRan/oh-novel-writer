# 三重验证规则

## 概述

借鉴 cangjie-skill 的 RIA-TV++ 三重验证方法，过滤通用特征和偶发特征，确保提取的决策框架具有独特性和可复现性。

---

## 验证规则

### 规则 1：跨书验证（Cross-Book Verification）

**标准**：同一决策规则必须在 ≥2 本书中出现

**验证方法**：
1. 统计每本书中该规则的出现次数
2. 计算出现该规则的书籍数量
3. 如果只在1本书中出现，标记为「可能偶然」

**处理**：
- ≥2 本书出现 → 通过
- 仅1本书出现 → 标记为「可能偶然」，降低权重

**原因**：如果只在1本书中出现，可能是偶然，不是作者的固定决策模式。

### 规则 2：频率验证（Frequency Verification）

**标准**：出现频率 ≥5%（非偶发）

**验证方法**：
1. 统计该规则在全书中的出现次数
2. 计算出现频率（出现次数 / 总章节数）
3. 如果频率 <5%，标记为「低频特征」

**处理**：
- 频率 ≥5% → 通过
- 频率 <5% → 标记为「低频特征」，不写入 SKILL.md

**原因**：如果出现频率太低，说明不是作者的稳定习惯，可能是偶然。

### 规则 3：独特性验证（Uniqueness Verification）

**标准**：满足以下任一锚点即通过

**锚点 1：频率锚点**
- 该规则在本书中出现频率 > 60%（是稳定模式，非偶然）
- 示例：「开篇用他人对话碎片」在6本书的第1章都出现 → 通过

**锚点 2：对比锚点**
- 与同类题材主流写法不同
- 示例：古言通常用旁白交代背景，但这位作者用角色视角碎片 → 通过

**锚点 3：反例锚点**
- 能找到「不这样做」的对比案例
- 示例：其他作者用「说道」标签，这位作者用动作beat替代 → 通过

**都不满足** → 判定为通用规则，写入 `rejected/`

**通用规则黑名单**（直接淘汰，不走锚点判断）：
- 每章要有钩子（太通用）
- 对话要有潜台词（太通用）
- 角色要有动机（太通用）
- 冲突要有升级（太通用）
- 节奏要有起伏（太通用）
- 开篇要吸引人（太通用）
- 结局要圆满（太通用）

**处理**：
- 满足任一锚点 → 通过
- 在黑名单中 → 不写入
- 都不满足 → 标记为「通用规则」，写入 `rejected/`

## 验证流程

### Step 0：建立对比基准

独特性验证需要参照系。在验证前，先建立「主流写法」基准：

| 基准来源 | 读取方式 |
|---------|---------|
| **其他作者的 SKILL.md** | 扫描 `.claude/skills/story-style/` 下其他作者的 SKILL.md，提取其决策规则作为对比基准 |
| **通用网文写作常识** | 如果没有其他作者的 SKILL.md，用以下常识作为基准：每章有钩子、对话有潜台词、角色有动机、冲突要升级、节奏有起伏、开篇要吸引人、结局要圆满 |

**输出**：内部参考（不需要写入文件），用于独特性验证的对比锚点判断。

### Step 1-3：执行三重验证

```
提取的决策规则
    │
    ├─→ Step 0：建立对比基准（读取其他作者SKILL.md或用通用常识）
    │
    ├─→ 跨书验证
    │       ├─≥2本书 → 继续
    │       └─仅1本书 → 标记「可能偶然」，降低权重
    │
    ├─→ 频率验证
    │       ├─≥5% → 继续
    │       └─<5% → 标记「低频特征」，不写入
    │
    └─→ 独特性验证（用Step 0的基准对比）
            ├─满足任一锚点 → 写入 SKILL.md
            ├─在黑名单 → 不写入
            └─都不满足 → 不写入
```

---

## 通过率预期

- 输入：Phase 2 提取的所有决策规则（约 30-50 个）
- 输出：验证通过的决策规则（约 15-30 个）
- 通过率：50-70%

---

## 例外情况

### 单本书场景

如果只有1本书，跳过跨书验证，只做频率验证和独特性验证。

**原因**：无法做跨书验证，但仍然可以通过频率和独特性过滤。

### 短篇小说

如果总章节数 <20，频率阈值降低到 10%。

**原因**：短篇的样本量小，5%的阈值太严格。

### 关键规则例外

某些关键规则即使频率低，也可以保留：
- 开篇决策规则（只出现1次，但很关键）
- 结局决策规则（只出现1次，但很关键）
- 高潮决策规则（只出现1次，但很关键）

**原因**：这些规则虽然出现次数少，但对写作质量影响大。

## 验证示例

### 示例 1：通过验证（频率锚点）

**规则**：如果需要快速建立代入感 → 用「危机」开场

- 跨书验证：3本书中都出现 ✅
- 频率验证：出现频率 100%（每本书的第1章） ✅
- 独特性验证：频率锚点通过（6本书第1章都用危机开场，频率100% > 60%） ✅
- **结果**：写入 SKILL.md

### 示例 2：未通过验证（黑名单）

**规则**：每章要有钩子

- 跨书验证：所有书都出现 ✅
- 频率验证：出现频率 100% ✅
- 独特性验证：在通用规则黑名单中 ❌
- **结果**：不写入 SKILL.md

### 示例 3：通过验证（对比锚点）

**规则**：用动作beat替代「说道」标签

- 跨书验证：4本书中出现 ✅
- 频率验证：出现频率 80% ✅
- 独特性验证：对比锚点通过（同类题材通常用「说道」，这位作者用动作beat） ✅
- **结果**：写入 SKILL.md

### 示例 4：未通过验证（都不满足）

**规则**：对话要有潜台词

- 跨书验证：所有书都出现 ✅
- 频率验证：出现频率 95% ✅
- 独特性验证：频率锚点不满足（虽然频率高，但这是写作常识）❌ 对比锚点不满足（同类题材都这样）❌ 反例锚点不满足（找不到不这样做的对比案例）❌
- **结果**：写入 `rejected/`

### 示例 5：关键规则例外

**规则**：第1章用他人对话开场

- 跨书验证：仅1本书出现 ⚠️
- 频率验证：出现频率 100%（每本书的第1章） ✅
- 独特性验证：频率锚点通过（每本书第1章都用，频率100% > 60%） ✅
- **结果**：关键规则例外，写入 SKILL.md
