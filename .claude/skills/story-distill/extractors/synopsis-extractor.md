# 书名与简介风格提取器 prompt

你的任务：从小说中提取作者的书名命名模式和简介写作规则。

## 读取数据

- 读取每本书的**书名**
- 读取每本书的**简介/文案**（番茄小说页面上的简介文本，通常在小说开头或 book-overviews 中）
- 读取每本书的**标签/分类**

## 提取内容

### 书名分析（每本书）

| 维度 | 问题 |
|------|------|
| 字数 | 书名几个字？ |
| 句式 | 陈述句？疑问句？短语？ |
| 信息量 | 透露了什么？角色？设定？冲突？ |
| 悬念感 | 是否制造疑问？ |
| 类型信号 | 如何暗示题材？ |

### 简介分析（每本书）

| 维度 | 问题 |
|------|------|
| 开头策略 | 第一句是什么？对话？设定？冲突？ |
| 信息顺序 | 先交代什么再交代什么？ |
| 结尾钩子 | 简介末尾用什么钩子？ |
| 字数范围 | 短简介还是长简介？ |
| 人称选择 | 第一人称？第三人称？ |

### 标签分析（每本书）

| 维度 | 问题 |
|------|------|
| 标签数量 | 通常用几个？ |
| 标签选择 | 偏题材标签还是情绪标签？ |
| 标签顺序 | 哪个放第一？ |

## 输出格式

```markdown
## 书名模式

**字数偏好**：{N}字（范围：{min}-{max}字）

**命名结构**：{结构描述}

**信息量**：{透露什么}

**悬念感**：{高/中/低}

**决策规则**：
- 如果写{题材}，书名用{结构}
- 书名避免{X}

**示例**：
1. {书名1} — {命名逻辑}
2. {书名2} — {命名逻辑}
3. {书名3} — {命名逻辑}

## 简介模式

**开头策略**：{类型}（频率：{X}%）

**信息顺序**：{顺序描述}

**结尾钩子**：{钩子类型}（频率：{X}%）

**字数范围**：{N}-{M}字（平均：{avg}字）

**人称选择**：{人称}

**决策规则**：
- 简介第一句用{X}
- 简介必须包含{X}元素
- 简介末尾用{X}钩子
- 简介避免{X}

**示例**：
1. {书名1}简介：
> {完整简介文本}

2. {书名2}简介：
> {完整简介文本}

## 标签模式

**标签数量**：{N}个（范围：{min}-{max}）

**第一个标签**：{标签}

**必选标签**：{标签1}、{标签2}

**决策规则**：
- 第一个标签必须是{X}
- 标签数量控制在{N}个
- 必选标签：{标签1}、{标签2}
```

## 硬性要求

- 每条规则附原文示例（完整引用简介文本，不要截断）
- 书名和简介的决策规则必须是「如果X，则Y」格式
- 输出写入 `{输出目录}/references/candidates/synopsis-patterns.md`
