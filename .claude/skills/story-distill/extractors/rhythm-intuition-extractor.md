# 节奏直觉提取器 prompt

你的任务：从小说中提取作者的节奏直觉。

## 读取数据

- 读取 `{输出目录}/references/book-overviews/*.md`（每本书的结构分析）
- 读取 `{输出目录}/sources/*.txt`（原文）

**精读章节定义**：
- 前3章（开篇）
- 第N/4章±1章（1/4处）
- 第N/2章±1章（中段）
- 第3N/4章±1章（3/4处）
- 最后3章（结局）
- 全书转折点（book-overviews 中标注的）

## 提取内容

提取 **4-8 条**节奏规则，覆盖以下4类（每类至少1条）：

| 类型 | 从哪里找 | 找什么 |
|------|---------|--------|
| **读者预期管理** | 读者会期待X但作者给Y的场景 | 怎么满足+意外？ |
| **情绪曲线控制** | 虐→甜→虐→甜交替处 | 虐甜比例？交替节奏？ |
| **信息差设计** | 读者知道但角色不知道的场景 | 信息差怎么制造紧迫感？ |
| **承诺兑现节奏** | 伏笔埋设和回收处 | 伏笔多久回收？怎么兑现？ |

**量化要求**：尽量从原文中统计：
- 甜虐比例（甜场景数:虐场景数）
- 伏笔回收周期（平均多少章回收）
- 情绪周期（多少章一个完整的情绪起伏）

## 每条规则必须包含

| 维度 | 要求 |
|------|------|
| **核心规则** | 一句话概括 |
| **R（原文引用）** | 直接引用书中段落，≤150字，附书名+章节号 |
| **量化数据** | 如有统计（如甜虐比例），附上 |
| **A1（书中案例）** | 至少1个具体应用 |
| **A2（触发场景）** | 什么条件下使用这个规则 |
| **B（边界说明）** | 局限性 |

## 输出格式

```markdown
## 读者预期管理

**核心规则**：满足读者期待，但多给一步意外

**R（原文引用）**：
> {原文片段}
> ——《{书名}》第{X}章

**量化数据**：6本书中，80%的期待满足场景都附带一个意外

**A1（书中案例）**：
- 《{书名}》第{X}章：读者期待男女主和好 → 作者给和好，但加一个新障碍

**A2（触发场景）**：
- 当读者可能期待X时，给X+1（满足+意外）

**B（边界说明）**：
- 每次都加意外会让读者疲惫，需要偶尔纯粹满足
```

## 节奏曲线分析（增强维度）

在提取节奏直觉的同时，分析作者的**宏观节奏曲线**：

### 分析内容

| 维度 | 从原文中统计什么 | 产出什么 |
|------|----------------|---------|
| **情绪密度** | 按章统计冲突词/紧张词/情绪词频率 | 每章的情绪密度值（0-10） |
| **高低点分布** | 识别情绪高点章和低谷章 | 高低点位置 + 间隔章数 |
| **节奏模式** | 高低点的排列规律 | 节奏模式类型（锯齿型/波浪型/阶梯型） |
| **冲突密度** | 每章核心冲突数量 | 冲突密度表 + 平均每章冲突数 |

### 节奏曲线产出格式

```markdown
## 节奏曲线

### 情绪密度分布
| 章节 | 情绪密度(0-10) | 标记 |
|------|---------------|------|
| 第1章 | 7 | 高点 |
| 第2章 | 4 | 低谷 |
| 第3章 | 8 | 高点 |
...

### 节奏模式
- 模式类型：{锯齿型/波浪型/阶梯型}
- 高低点间隔：{平均N章}
- 冲突密度：{平均每章X个核心冲突}

### 节奏公式
- 每{N}章安排一个情绪高点
- 高点后必须接{N}章低谷（喘息期）
- 连续低谷不超过{N}章
```

---

## 硬性要求

- 每条规则必须有原文引用
- 有统计数据优先（如「甜虐比例6:4」「伏笔平均15章回收」）
- 节奏曲线必须有按章的情绪密度数据，不能凭感觉
- 输出写入 `{输出目录}/references/candidates/rhythm-intuition.md`
