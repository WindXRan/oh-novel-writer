# 去AI策略提取器 prompt（5.5a 基础策略）

你的任务：分析作者的写作习惯，从去AI模块库中选择匹配的模块。

**职责定位**：这是"预测"阶段——基于作者风格预选需要执行的去AI模块。
- 5.5a（本文件）= 基于作者风格的"基础策略"（预测：作者不用连接词 → 预选删连接词）
- 5.5b（post-write-scanner）= 基于 AI 输出的"查漏补缺"（校正：AI 用了太多连接词 → 补充删连接词）
- 两者是"预测 + 校正"关系，不是重复计算

## 读取数据

- 读取 `{输出目录}/sources/*.txt`（原文，抽样5-10段）
- 读取 `{skill目录}/de-ai-modules/*.md`（去AI模块库）

## 分析维度

从原文中分析以下维度：

| 维度 | 找什么 | 阈值参考 |
|------|--------|----------|
| **连接词频率** | 「然而」「但是」「因此」等每段出现几次？ | 作者密度 ≥1.0次/百字才触发 |
| **句长分布** | 平均句长？>30字的长句占比？ | 长句占比 >30% 触发 |
| **副词频率** | 「微微」「轻轻」「缓缓」每章出现几次？ | >5次/章 触发 |
| **标点习惯** | 句号密度？感叹号频率？分号冒号使用？ | 句号密度 <1.0次/百字触发 |
| **语序规律** | 是否都是「主谓宾」结构？有无倒装？ | 主谓宾占比 >90% 触发 |
| **重复模式** | 有无词/句式重复习惯？ | 同词重复 >3次/段 触发 |
| **数字精确度** | 数字是精确还是模糊？ | 精确数字 >80% 触发模糊化 |
| **得地的混用** | 「得」「地」出现频率 | ≥1次/千字 触发替换 |
| **情感表达模式** | 情绪描写是否单一？是否依赖固定短语？ | 重复情绪短语 >3种 触发 |
| **流畅度** | 是否有「卡顿感」？有无省略主语？ | 异常定义：连续3句主谓宾结构无变化 + 句长标准差 <5字 |

## 选择规则

1. 读取每个模块的「触发条件」
2. 对照作者的写作习惯，判断是否匹配
3. 选择 **3-5 个**匹配度最高的模块
4. 标点后处理**必选**（最后执行）
