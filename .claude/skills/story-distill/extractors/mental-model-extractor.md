# 心智模型提取器 prompt

你的任务：从小说中提取作者的核心心智模型。

## 读取数据

- 读取 `{输出目录}/references/book-overviews/*.md`（每本书的结构分析）
- 读取 `{输出目录}/sources/*.txt`（原文）

**精读章节定义**（从原文中重点读这些位置）：
- 前3章（开篇）
- 第N/4章±1章（1/4处）
- 第N/2章±1章（中段）
- 第3N/4章±1章（3/4处）
- 最后3章（结局）
- 全书转折点（book-overviews 中标注的）

## 提取内容

提取作者的 **3-5 个**核心心智模型，必须覆盖以下4个维度（可多不可少）：

| 维度 | 从哪里推断 | 什么算「真信念」 |
|------|-----------|----------------|
| **故事观** | 开篇方式、冲突设计、结局处理 | 跨≥2本书用同一种结构模式 |
| **角色观** | 角色塑造、人设展示、角色弧线 | 跨≥2本书用同一种角色设计逻辑 |
| **冲突观** | 冲突类型、升级方式、解决方式 | 跨≥2本书用同一种冲突模式 |
| **爽感观** | 爽点设计、延迟满足、释放节奏 | 跨≥2本书用同一种爽点设计 |

**单本书时**：从同一本书的不同场景中找重复模式（≥3次出现）。

### 信念形成路径（增强维度）

从角色选择和情节走向中推断作者的信念形成过程：

| 推断来源 | 示例 |
|---------|------|
| 角色在关键节点的选择 | 「主角在X场景选择了Y，说明作者认为Y比Z更重要」 |
| 情节的因果走向 | 「作者让X行为导致Y结果，说明作者信奉Z因果观」 |
| 反派的失败方式 | 「反派因为X而失败，说明作者认为X是不可持续的」 |
| 配角的评价 | 「配角对X的评价是Y，反映了作者的价值判断」 |

**注意**：这是推断，不是事实。标注「推断」。

## 每个模型必须包含

| 维度 | 要求 | 示例 |
|------|------|------|
| **名称** | 一句话概括 | 「好故事=读者和主角一起醒来发现处境」 |
| **R（原文引用）** | 直接引用书中段落，≤150字，附书名+章节号 | 见下方示例 |
| **I（决策解读）** | 用自己的话说明作者做了什么选择 | 「作者选择用信息不对称制造代入感，而不是用旁白交代背景」 |
| **A1（书中案例）** | 这个模型在书中的具体应用 | 「《惜花芷》第1章：读者和花芷一起发现花家危机」 |
| **A2（触发场景）** | 什么条件下使用这个模型 | 「当需要快速建立代入感时」 |
| **B（边界说明）** | 这个模型的局限性 | 「不适合需要读者提前知道背景的悬疑类故事」 |

## 输出示例

```markdown
## 故事观

**核心信念**：好故事 = 读者和主角一起「醒来」发现处境

**R（原文引用）**：
> 造孽哦，家门不幸啊。爹爹，四叔叔他们真的死了吗……在说什么东西？陈汐头疼欲裂，不是离婚吗？
> ——《一睁眼前夫成了我懒汉相公》第1章

**I（决策解读）**：
作者选择让读者和主角同时处于信息盲区——读者不知道发生了什么，主角也不知道。这样读者的探索欲和主角的求生欲合二为一，代入感极强。这不是「先交代背景再进入故事」，而是「直接把你扔进故事里」。

**A1（书中案例）**：
- 《一睁眼前夫》第1章：陈汐醒来发现自己穿越，读者和她一起困惑
- 《分手了秦少》第1章：盛惜被分手，读者和她一起不知道秦钰的真实意图

**A2（触发场景）**：
- 当需要快速建立代入感时，用「主角不知道」开场
- 当不希望读者提前猜到结局时，用信息盲区开场

**B（边界说明）**：
- 不适合悬疑类故事（悬疑需要读者比角色知道得多）
- 不适合需要快速交代世界观的架空设定
```

## 硬性要求

- 每个模型必须有原文引用，不能凭空编造
- 每个模型必须有至少1个书中案例（A1）
- 区分「作者明确表达的」vs「从行为推断的」——在 I 中标注
- 发现矛盾直接记录，不要调和
- 输出写入 `{输出目录}/references/candidates/mental-models.md`
