# 审稿红线提取器 prompt

你的任务：从小说中提取作者的**审稿红线**——这些写法在作者的书中**从未出现**，说明作者刻意回避。

## 读取数据

- 读取 `{输出目录}/references/book-overviews/*.md`（每本书的结构分析）
- 读取 `{输出目录}/sources/*.txt`（原文，全文扫描）
- 读取 `{输出目录}/references/candidates/anti-patterns.md`（反模式提取器的产出，作为基础）

## 与反模式提取器的关系

| 维度 | 反模式提取器（已有） | 审稿红线提取器（本文件） |
|------|-------------------|----------------------|
| 目标 | 提取「作者不用什么」 | 提取「审稿时直接打回什么」 |
| 粒度 | 词汇/句式级 | 段落/章节级 |
| 输出 | 替代方案 | 严重等级 + 判定标准 |
| 用途 | 写作时参考 | 审稿时作为检查清单 |

## 提取内容

提取 **8-15 条**审稿红线，覆盖以下6类（每类至少1条）：

| 类型 | 从哪里找 | 找什么 |
|------|---------|--------|
| **节奏红线** | 全书节奏波动 | 什么节奏问题作者绝不容忍？ |
| **角色红线** | 角色一致性 | 什么角色问题作者绝不容忍？ |
| **对话红线** | 对话场景 | 什么对话问题作者绝不容忍？ |
| **情绪红线** | 情绪曲线 | 什么情绪问题作者绝不容忍？ |
| **结构红线** | 章节结构 | 什么结构问题作者绝不容忍？ |
| **表达红线** | 文字层面 | 什么表达问题作者绝不容忍？ |

## 每条红线必须包含

| 维度 | 要求 |
|------|------|
| **红线名称** | 一句话概括 |
| **严重等级** | 🔴 必须修改 / 🟡 建议修改 |
| **判定标准** | 什么情况下触发这条红线 |
| **R（原文引用）** | 作者用什么替代？附原文 |
| **修改方向** | 发现这个问题时，往哪个方向改 |

## 输出格式

写入 `{输出目录}/references/candidates/review-redlines.md`
