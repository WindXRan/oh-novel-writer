# 爽点分布提取器 prompt

你的任务：从小说中提取爽点分布模式。

## 读取数据

- 读取 `{输出目录}/references/book-overviews/*.md`（每本书的结构分析）
- 读取 `{输出目录}/sources/*.txt`（原文，全文扫描）

## 提取内容

提取作者的**爽点设计规律**，覆盖以下4类：

| 类型 | 从哪里找 | 找什么 |
|------|---------|--------|
| **打脸爽** | 被轻视后反超的场景 | 谁被打脸？用什么方式？铺垫了多久？ |
| **逆袭爽** | 从低谷到高峰的转折 | 逆袭的触发条件是什么？情绪对比怎么制造？ |
| **发糖爽** | 男女主感情升温的场景 | 发糖的频率？发糖前有什么铺垫？ |
| **揭秘爽** | 真相大白/信息差消除的场景 | 揭秘的时机？读者期待了多久？ |
| **权力反转爽** | 弱势方变强势方 | 反转的转折点是什么？ |

## 分析维度

### 爽点位置分析

| 维度 | 统计什么 |
|------|---------|
| **爽点位置** | 哪些章有爽点？（章节号） |
| **爽点间隔** | 平均多少章一个爽点？ |
| **爽点密度** | 每个爽点持续几段？ |
| **爽点类型** | 打脸/逆袭/发糖/揭秘各占多少？ |

### 爽点前置条件分析

| 维度 | 找什么 |
|------|--------|
| **铺垫长度** | 爽点前有多少章的铺垫/压抑？ |
| **情绪对比** | 爽点前的情绪基调是什么？（虐/平/紧张） |
| **信息差** | 爽点利用了什么信息差？（读者知道但角色不知道？） |
| **角色反应** | 被爽的角色有什么反应？（震惊/后悔/愤怒） |

## 每个爽点必须包含

| 维度 | 要求 |
|------|------|
| **爽点名称** | 一句话概括，如「宁祯当众揭穿三姨太」 |
| **爽点类型** | 打脸/逆袭/发糖/揭秘/权力反转 |
| **R（原文引用）** | 爽点的核心段落，≤150字，附书名+章节号 |
| **前置条件** | 铺垫了多久？用了什么情绪对比？ |
| **爽感来源** | 为什么爽？（信息差释放/权力反转/情绪积压后爆发） |
| **A2（触发场景）** | 什么条件下安排这个类型的爽点 |

## 输出格式

```markdown
## 爽点分布

### 爽点密度
- 平均每{N}章一个爽点
- 爽点类型分布：打脸{X}%、逆袭{Y}%、发糖{Z}%、揭秘{W}%

### 爽点清单

| # | 章节 | 类型 | 爽感来源 | 铺垫长度 |
|---|------|------|---------|---------|
| 1 | 第{X}章 | 打脸 | 信息差释放 | {N}章 |
| 2 | 第{X}章 | 权力反转 | 情绪积压后爆发 | {N}章 |

### 爽点设计公式

**打脸公式**：
- 前置：{角色被轻视/被嘲笑/被低估}
- 触发：{角色展示真实实力/揭穿谎言}
- 释放：{对手震惊/后悔/愤怒}

**逆袭公式**：
- 前置：{角色处于低谷/被压迫/走投无路}
- 触发：{获得关键资源/发现关键信息/做出关键选择}
- 释放：{地位逆转/关系逆转/权力逆转}

**发糖公式**：
- 前置：{误会/冷战/距离}
- 触发：{一方主动/意外事件/危机中的保护}
- 释放：{亲密接触/坦白心意/双向奔赴}
```

## 硬性要求

- 每个爽点必须有原文引用
- 爽点必须有「前置条件」分析（铺垫了多久）
- 爽点密度必须有统计数据，不能凭印象
- 爽点设计公式必须从原文中推断，不能从写作教材搬来
- 输出写入 `{输出目录}/references/candidates/satisfaction-points.md`
