# story-review · 一致性审查

## 功能

从多个视角审查小说，发现：
- 设定矛盾（人物前后不一致）
- 时间线冲突
- 伏笔断线（埋了没回收）
- 角色行为逻辑问题
- 情节漏洞

## 使用方式

```
/story-review
```

## 审查维度

1. **事实一致性**：人物属性、设定是否前后矛盾
2. **时间线**：事件顺序是否合理
3. **伏笔状态**：哪些伏笔已回收，哪些断线
4. **角色行为**：行为是否符合人设
5. **情绪弧线**：情感变化是否自然

## 输出

审查报告，按严重程度分级：
- S1：严重矛盾（必须修复）
- S2：逻辑漏洞（建议修复）
- S3：轻微问题（可选修复）
- S4：优化建议
