# story-rewrite_vPlan：仿写引擎 vPlan

> 全书规划先行，纯写作出稿。质量最高，速度最快。

---

## 文件结构

```
novel-download-authors/{作者名}/{源书名}/
├── 源文/                        # 拆章后的源文章节
├── 蒸馏/mode-b/
│   ├── style_profile_N.json     # 脚本统计指纹（定量数据，不传prompt）
│   └── style_guide_N.md         # LLM 风格指南+节奏骨架

{新书名}/
├── 设定/
│   ├── 新书概念.md              # 书名/类型/NPC映射/卖点
│   ├── story_bible.md           # 世界观
│   ├── 全书弧线骨架.md          # Phase 1：全书情感曲线/角色成长/伏笔清单
│   ├── 章节顺序.md              # Phase 1：全书源文映射
│   └── 支线大纲.md              # Lv3：原创支线规划
├── 大纲/章纲_N.md               # 每章一个
└── 正文/第N章.txt
```

---

## 去重等级

| 等级 | 做什么 | 风险 | 何时 |
|------|--------|------|------|
| **Lv1** | 按弧线需求重排源文章节顺序 | 3/10 | Phase 1.5 |
| **Lv2** | 替换每个关键情节的触发逻辑 | 1/10 | Phase 1.4 章纲 |
| **Lv3** | 加一条源文没有的原创支线 | 0 | Phase 1 + 章纲 |

**默认 Lv1+Lv2**。

**Lv2 触发逻辑标准**（必须满足至少2条）：
1. **起因不同**：源文因A触发，新书因B触发（A≠B）
2. **动机不同**：源文角色出于X动机，新书角色出于Y动机（X≠Y）
3. **后果不同**：源文事件导致M结果，新书事件导致N结果（M≠N）
4. **参与人不同**：源文是这两个人互动，新书换成另外两个人

❌ 不合格：「源文男主因租房被抓→新书男主因租房被误会」（只换了表面，因果链一样）
✅ 合格：「源文男主因租房被抓→新书男主因帮室友搬家被误会成小偷」（起因+场景+参与人全换）

## Phase 1：全书规划（跑一次，全部完成再写章）

```
1.0 拆章+风格指纹（脚本）
  ↓
1.1 新书概念+世界观（主agent） ┐ 并行
1.2 题材识别（主agent）         ┘
  ↓
1.3 全书弧线骨架（1 agent）    ┐ 并行
1.4 风格分析+节奏提取（10 agents）┘
  ↓
1.5 章纲生成（10 agents）
1.6 全书章节顺序映射（1 agent）
```

### 1.0 拆章 + 风格指纹
```bash
# 拆章
python .claude/skills/story-rewrite_vPlan/tools/source_chapter_splitter.py split <源文.txt> novel-download-authors/{作者名}/{源书名}/源文/

# 风格指纹
for i in range(start, end+1):
    python .claude/skills/story-rewrite_vPlan/tools/style_analyzer.py novel-download-authors/{作者名}/{源书名}/源文/第{i}章.txt --json | Out-File -FilePath novel-download-authors/{作者名}/{源书名}/蒸馏/mode-b/style_profile_{i}.json -Encoding utf8
```

**字数约束**：2000-2500字，硬上限3000字。不看源文字数，直接固定。

### 1.1 新书概念 + 世界观
- {新书名}/设定/新书概念.md：书名、类型、核心卖点、**NPC命名映射表**（必填）、故事弧线、差异化
- {新书名}/设定/story_bible.md

**人设差异化检查（必填）**：

| 检查项 | 源文 | 新书（必须不同） |
|--------|------|-----------------|
| CP互动模式 | [如：腼腆男+冰山女] | [如：毒舌男+傲娇女] |
| 女主独特身份 | [如：漫画家] | [如：独立游戏开发者] |
| 相识方式 | [如：租房乌龙进局子] | [如：游戏排位赛互喷] |

❌ 不合格：只改名字，性格内核和关系模型照搬
✅ 合格：CP互动模式+女主身份+相识方式至少2项完全不同

### 1.2 题材识别
从源文标签选题材，应用 prompts/genre-management.md 中对应的 fatigue 词表

### 1.3 全书弧线骨架（1个 agent）

```
请为《{新书名}》设计全书弧线骨架。

【新书概念】请读取：{新书名}/设定/新书概念.md
【源文总章数】{N}章

【输出】保存到：{新书名}/设定/全书弧线骨架.md

## 全书情感曲线
| 章范围 | 情绪类型 | 强度(1-10) | 功能 | 设计理由 |
|--------|---------|-----------|------|---------|
| 1-3    | 好奇→紧张 | 5→6→7 | 开局 | 用误会建立悬念，逐章升温 |
| 4-6    | 轻松→心动 | 4→5→7 | 日常→推进 | 缓一缓再甜 |
| 7-10   | 冲突→和解→高潮 | 7→5→8→9 | 第一个小高潮 | 打破关系再重建 |
| 11-20  | ... | ... | ... | ... |
| ... |

⚠️ 弧线设计原则：
- 不能连续3章同一情绪类型
- 每10章必须有1个情绪峰值（强度≥8）
- 开局强度5-7，逐章升温
- 每批结尾强度≥8（留钩子）
- 全书有2-3个大高潮（强度9-10）

## 角色成长主线
### 男主：从[A]到[B]
| 阶段 | 章范围 | 状态 | 关键转折 |
|------|--------|------|---------|

### 女主：从[C]到[D]
| 阶段 | 章范围 | 状态 | 关键转折 |
|------|--------|------|---------|

## 核心伏笔清单
| 伏笔 | 埋设阶段 | 预计回收阶段 | 优先级 |
|------|---------|-------------|--------|
```

### 1.4 风格分析 + 节奏提取（并行 10 agent × N批）

每批10章，每章1个 agent。每个 agent 读源文一次，同时输出风格指南+节奏骨架。

```
每个 Task 的 prompt：
"""
你是一位文学风格分析专家。请分析《{源书名}》第N章，输出两部分：

【源文】请读取：novel-download-authors/{作者名}/{源书名}/源文/第N章.txt
【风格指纹】请读取：novel-download-authors/{作者名}/{源书名}/蒸馏/mode-b/style_profile_N.json（定量数据参考，不要编造）

## 第一部分：风格指南（8维度+排除项）

### 叙事声音与语气
（附原文例句）

### 对话风格

### 场景描写特征

### 转折与衔接手法

### 节奏特征

### 词汇偏好

### 情绪表达方式

### 独特习惯

### ⚠️ 风格排除项
（过于独特的、模仿了会暴露抄源文的特征，写章时必须回避）

## 第二部分：节奏骨架（只保留抽象模式，不得有具体事件）

1. 情绪强度变化：[低→中→高→低]
2. 钩子位置和类型：[章末钩子是悬念/反转/甜/虐？]
3. 爽点公式：[抽象描述，如"弱者展示实力→强者震惊"]
4. 节奏模式：[几段日常+几段冲突+几段甜？]
5. 场景切换频率

【输出】两部分合并保存到：novel-download-authors/{作者名}/{源书名}/蒸馏/mode-b/style_guide_N.md
"""
```

### 1.5 章纲生成（并行 10 agent × N批）

每批10章，每章1个 agent。每个 agent 读全书弧线骨架+节奏骨架，输出章纲。

⚠️ 各批可以并行，因为有全书弧线骨架保证方向一致。

```
每个 Task 的 prompt：
"""
请为《{新书名}》第N章生成章纲。

⚠️ 严格只输出以下 7 个字段，禁止添加任何其他字段。
⚠️ 禁止展开场景和对话，不要把章纲写成半成品小说。
⚠️ 禁止在章纲中出现"对应源文XX""参考源文XX"等映射词，章纲必须看起来是 100% 原创故事。
⚠️ 字数控制在 10-15 行内。

**强制改写清单**（每章至少改3项）：
- [ ] 场景：源文在A发生，新书必须换到B
- [ ] 触发物：源文用物品X触发，新书必须换物品Y
- [ ] 对话内容：源文角色说了P，新书角色必须说Q
- [ ] 情绪反应：源文角色先惊后喜，新书角色可以先怒后悟
- [ ] 互动对象：源文是男女主互动，新书可以加配角或换人
- [ ] 时间线：源文是白天发生，新书可以改晚上

❌ 禁止：只换人名和地点，事件因果链完全照搬

【新书概念】请读取：{新书名}/设定/新书概念.md
【全书弧线骨架】请读取：{新书名}/设定/全书弧线骨架.md（本章在全书中的位置、该阶段的情绪目标、角色状态）
【前序章纲】请读取：{新书名}/大纲/ 下已有章纲文件（如有）
【风格指南-节奏骨架】请读取：novel-download-authors/{作者名}/{源书名}/蒸馏/mode-b/style_guide_X.md（只看「第二部分：节奏骨架」，不得看源文原文）
【源文】⚠️ 不得读取源文原文！只参考节奏骨架，事件必须全部原创。

## 第N章 [章名]

- 核心事件：[2-3句话，自己编的原创事件]
- 因果逻辑：[A→B→C→D，每个环节一句话]
- 触发逻辑：[⚠️ Lv2 去重：为什么发生，必须和源文不同]
- 节奏模式：[情绪强度变化，如 低→高→低]
- 钩子：[章末钩子，1句话]
- 爽点来源：[爽感公式，如"身份反转""装逼打脸"]
- 原创元素：[⚠️ 至少1个源文完全没有的元素：新人物/新道具/新支线/新关系，不得为空]

【输出】保存到：{新书名}/大纲/章纲_N.md
"""
```

### 1.6 全书章节顺序映射（1个 agent）

```
全书章纲完成后，按弧线骨架的需求，从源文池子里挑章节映射。

读取：{新书名}/设定/全书弧线骨架.md + 全部章纲
按弧线需求从源文池子里挑章节（功能匹配，不按顺序）
保存到：{新书名}/设定/章节顺序.md

| 新书章号 | 源文章号 | 功能 | 匹配理由 |
|---------|---------|------|---------|
| 1 | 1 | 开局 | 弧线要求紧张开局 |
| 2 | 3 | 日常 | 弧线要求第2章缓一缓 |
| ... |
```

## Phase 2：纯写作（每批循环）

### B4 写章（并行 10 agent）

```
每个 Task 的 prompt：
"""
请写出《{新书名}》第N章正文。

你是《{新书名}》的作者本人。请以作者的身份写出这一章。

【章纲】请读取：{新书名}/大纲/章纲_N.md
【风格指南】请读取：novel-download-authors/{作者名}/{源书名}/蒸馏/mode-b/style_guide_X.md
（X = 章节顺序.md 中新书第N章对应的源文章号，只看「第一部分：风格指南」，跳过节奏骨架）
【全书弧线骨架】请读取：{新书名}/设定/全书弧线骨架.md（本章在全书中的位置、该阶段的情绪目标）
【新书概念】请读取：{新书名}/设定/新书概念.md（核对角色名、关系、设定）

【字数硬约束】
- 目标：2000-2500字，硬上限3000字
- 不得超出，写到2500字左右必须收尾

【反洗稿规则】
- 禁止照搬源文的句子结构（只换名词不行）
- 禁止复制源文的对话内容（即使换了角色名）
- 禁止模仿风格指南中的「排除项」特征
- 可以借鉴：节奏模式、爽点公式、情绪弧线
- 必须原创：具体场景描写、对话内容、人物反应、因果细节

【质量要求】
- 每章至少1个让读者笑/心动/紧张的瞬间
- 对话必须有信息量，不得水对话
- 结尾必须有钩子，让读者想看下一章
- 场景描写用具体细节，不用空泛形容词
- 角色行为必须符合章纲中的因果逻辑，不得突然降智

【输出】保存到：{新书名}/正文/第N章.txt
"""
```

### B5 循环
1. 检查是否还有未写章节
2. 有 → 启动下一批10个B4 agent
3. 无 → 进入 Phase 3 收尾
