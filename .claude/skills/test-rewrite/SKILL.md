# test-rewrite：仿写引擎 v3.0

> Mode B 三轮并行 Agent。章纲即真相。

---

## 文件结构

```
novel-download-authors/{作者名}/{源书名}/
├── 源文/                        # 拆章后的源文章节
├── 蒸馏/mode-b/
│   ├── style_profile_N.json     # 脚本统计指纹（定量数据，不传prompt）
│   └── style_guide_N.md         # LLM 8维风格指南

{新书名}/
├── 设定/
│   ├── 新书概念.md              # 书名/类型/NPC映射/卖点
│   ├── story_bible.md           # 世界观
│   ├── 章节顺序.md              # Lv1：按弧线需求重排
│   ├── 支线大纲.md              # Lv3：原创支线规划
│   └── 全书弧线骨架.md          # Phase 1：全书情感曲线/角色成长/伏笔清单
├── 大纲/章纲_N.md               # 每章一个
└── 正文/第N章.txt
```

---

## 去重等级

| 等级 | 做什么 | 风险 | 何时 |
|------|--------|------|------|
| **Lv1** | 按功能重排章节顺序 | 3/10 | B3 完成后 |
| **Lv2** | 替换每个关键情节的触发逻辑 | 1/10 | B3 写章纲 |
| **Lv3** | 加一条源文没有的原创支线 | 0 | Phase 1.5 + B3 |

**默认 Lv1+Lv2**。

**Lv2 触发逻辑标准**（必须满足至少2条）：
1. **起因不同**：源文因A触发，新书因B触发（A≠B）
2. **动机不同**：源文角色出于X动机，新书角色出于Y动机（X≠Y）
3. **后果不同**：源文事件导致M结果，新书事件导致N结果（M≠N）
4. **参与人不同**：源文是这两个人互动，新书换成另外两个人

❌ 不合格：「源文男主因租房被抓→新书男主因租房被误会」（只换了表面，因果链一样）
✅ 合格：「源文男主因租房被抓→新书男主因帮室友搬家被误会成小偷」（起因+场景+参与人全换）

## Phase 1：初始化

### 1.0 拆章 + 风格指纹
```bash
# 拆章
python .claude/skills/test-rewrite/tools/source_chapter_splitter.py split <源文.txt> novel-download-authors/{作者名}/{源书名}/源文/

# 风格指纹
for i in range(start, end+1):
    python .claude/skills/test-rewrite/tools/style_analyzer.py novel-download-authors/{作者名}/{源书名}/源文/第{i}章.txt --json | Out-File -FilePath novel-download-authors/{作者名}/{源书名}/蒸馏/mode-b/style_profile_{i}.json -Encoding utf8
```

**字数约束**：2000-2500字，硬上限3000字。不看源文字数，直接固定。

### 1.1 新书概念
{新书名}/设定/新书概念.md：书名、类型、核心卖点、**NPC命名映射表**（必填）、故事弧线、差异化

**人设差异化检查（必填）**：

| 检查项 | 源文 | 新书（必须不同） |
|--------|------|-----------------|
| CP互动模式 | [如：腼腆男+冰山女] | [如：毒舌男+傲娇女] |
| 女主独特身份 | [如：漫画家] | [如：独立游戏开发者] |
| 相识方式 | [如：租房乌龙进局子] | [如：游戏排位赛互喷] |

❌ 不合格：只改名字，性格内核和关系模型照搬
✅ 合格：CP互动模式+女主身份+相识方式至少2项完全不同

### 1.2 世界观
{新书名}/设定/story_bible.md

### 1.3 题材识别
从源文标签选题材，应用 prompts/genre-management.md 中对应的 fatigue 词表

### 1.4 全书弧线骨架（1个 agent，Phase 1 跑一次）

```
请为《{新书名}》设计全书弧线骨架。

【新书概念】请读取：{新书名}/设定/新书概念.md
【源文总章数】{N}章

【输出】保存到：{新书名}/设定/全书弧线骨架.md

## 全书情感曲线
| 章范围 | 情绪类型 | 强度(1-10) | 功能 | 设计理由 |
|--------|---------|-----------|------|---------|
| 1-3    | 好奇→紧张 | 5→6→7 | 开局 | 用误会建立悬念，逐章升温 |
| 4-6    | 轻松→心动 | 4→5→7 | 日常→推进 | 缓一缓再甜 |
| 7-10   | 冲突→和解→高潮 | 7→5→8→9 | 第一个小高潮 | 打破关系再重建 |
| 11-20  | ... | ... | ... | ... |
| ... |

⚠️ 弧线设计原则：
- 不能连续3章同一情绪类型
- 每10章必须有1个情绪峰值（强度≥8）
- 开局强度5-7，逐章升温
- 每批结尾强度≥8（留钩子）
- 全书有2-3个大高潮（强度9-10）

## 角色成长主线
### 男主：从[A]到[B]（用一句话概括全书成长）
| 阶段 | 章范围 | 状态 | 关键转折 |
|------|--------|------|---------|
| 开局 | 1-10 | 社恐新生 | 遇到女主，开始改变 |
| 发展 | 11-30 | 逐渐打开心扉 | ... |
| 高潮 | 31-50 | 面对最大挑战 | ... |
| 结局 | 51-60 | 完成成长 | ... |

### 女主：从[C]到[D]
（同上格式）

## 核心伏笔清单
| 伏笔 | 埋设阶段 | 预计回收阶段 | 优先级 |
|------|---------|-------------|--------|
| 男主是女主游戏搭子 | 1-10章 | 20-30章 | high |
| 女主怕打雷的原因 | 1-10章 | 40-50章 | medium |
| ... |
```

⚠️ 全书弧线骨架是粗略规划，B3 每批细化时以此为准。B3.5 重排时也参照此骨架。

### 1.4 Lv3：原创支线规划
从候选支线选一个或自定义，规划开端→发展→高潮→结局，保存到 {新书名}/设定/支线大纲.md

### 1.5 情节公共性分类

在写章纲前，先对源文前10章进行情节分类，保存到 `{新书名}/设定/情节分类.md`：

| 类型 | 定义 | 去重要求 | 示例 |
|------|------|---------|------|
| **公共情节** | 网文通用桥段，无版权 | 保留框架，改细节 | 军训、开学、室友聚餐、打游戏 |
| **半公共情节** | 常见但有特定编排 | 改场景+互动+结果 | 迎新晚会、图书馆偶遇、醉酒 |
| **独创情节** | 源文原创，有辨识度 | **必须整体替换** | 派出所乌龙、教学楼送外套、舞台合唱 |

**独创情节判定标准**（满足任一即为独创）：
1. 该情节在同类型网文中很少见（如：因租房被抓进局子）
2. 该情节是源文的标志性桥段（如：雨中送伞+怕打雷）
3. 该情节链有3个以上连续环节高度特定（如：醉酒→教学楼→女主送外套→纸条）

**独创情节替换规则**：
- 禁止只改触发逻辑，必须整体替换情节模块
- 只保留"功能"（如：制造男女主初次相遇）
- 必须设计全新情节模块实现相同功能

## Phase 2：写作（Mode B 三轮并行）

### B1 每批次：风格指纹（只跑新 10 章，不传 prompt）
⚠️ 不是 Phase 1.0！Phase 1.0 跑一次拆章+全本指纹，B1 每批次只跑本批 10 章的指纹更新。

```bash
for i in 本批起始..本批结束:
    python .claude/skills/test-rewrite/tools/style_analyzer.py novel-download-authors/{作者名}/{源书名}/源文/第{i}章.txt --json | Out-File -FilePath novel-download-authors/{作者名}/{源书名}/蒸馏/mode-b/style_profile_{i}.json -Encoding utf8
```

### B2 第一轮：风格分析 + 节奏提取（并行 10 agent）
```
每个 Task 的 prompt：
"""
你是一位文学风格分析专家。请分析《{源书名}》第N章，输出两部分：

【源文】请读取：novel-download-authors/{作者名}/{源书名}/源文/第N章.txt
【风格指纹】请读取：novel-download-authors/{作者名}/{源书名}/蒸馏/mode-b/style_profile_N.json（定量数据参考，不要编造）

## 第一部分：风格指南（8维度+排除项）

### 叙事声音与语气
（附原文例句）

### 对话风格

### 场景描写特征

### 转折与衔接手法

### 节奏特征

### 词汇偏好

### 情绪表达方式

### 独特习惯

### ⚠️ 风格排除项
（过于独特的、模仿了会暴露抄源文的特征，B4写章时必须回避）

## 第二部分：节奏骨架（只保留抽象模式，不得有具体事件）

1. 情绪强度变化：[低→中→高→低]
2. 钩子位置和类型：[章末钩子是悬念/反转/甜/虐？]
3. 爽点公式：[抽象描述，如"弱者展示实力→强者震惊"]
4. 节奏模式：[几段日常+几段冲突+几段甜？]
5. 场景切换频率

【输出】两部分合并保存到：novel-download-authors/{作者名}/{源书名}/蒸馏/mode-b/style_guide_N.md
"""
```

### B3 第二轮：章纲生成（并行 10 agent）
```
每个 Task 的 prompt：
"""
请为《{新书名}》第N章生成章纲。

⚠️ 严格只输出以下 6 个字段，禁止添加任何其他字段（不要写"风格数据""场景设计""情节节点""合规自检"等额外段落）。
⚠️ 禁止展开场景和对话，不要把章纲写成半成品小说。
⚠️ 禁止在章纲中出现"对应源文XX""参考源文XX"等映射词，章纲必须看起来是 100% 原创故事。
⚠️ 字数控制在 10-15 行内（每行字段紧凑，不要长段落）。

**强制改写清单**（每章至少改3项）：
- [ ] 场景：源文在A发生，新书必须换到B
- [ ] 触发物：源文用物品X触发，新书必须换物品Y
- [ ] 对话内容：源文角色说了P，新书角色必须说Q
- [ ] 情绪反应：源文角色先惊后喜，新书角色可以先怒后悟
- [ ] 互动对象：源文是男女主互动，新书可以加配角或换人
- [ ] 时间线：源文是白天发生，新书可以改晚上

❌ 禁止：只换人名和地点，事件因果链完全照搬

**⚠️ 独创情节强制替换**：
请读取「情节分类.md」，如果当前章节包含"独创情节"，禁止只改触发逻辑，必须整体替换：
1. 识别源文情节的"功能"（如：让男女主第一次见面）
2. 设计全新情节模块实现相同功能
3. 新情节必须：触发方式不同、场景不同、互动方式不同、结果不同

❌ 错误示例：源文"因租房被抓进局子" → 新书"因帮室友搬家被误会"（只改触发，情节链一样）
✅ 正确示例：源文"因租房被抓进局子" → 新书"游戏展当志愿者遇设备故障"（全新情节模块）

【新书概念】请读取：{新书名}/设定/新书概念.md
【情节分类】请读取：{新书名}/设定/情节分类.md
【全书弧线骨架】请读取：{新书名}/设定/全书弧线骨架.md（本章在全书中的位置、该阶段的情绪目标、角色状态）
【前序章纲】请读取：{新书名}/大纲/ 下已有章纲文件（如有）
【章节顺序】请读取：{新书名}/设定/章节顺序.md（如有）→ 查找"新书第N章 = 源文第X章"
【风格指南-节奏骨架】请读取：novel-download-authors/{作者名}/{源书名}/蒸馏/mode-b/style_guide_X.md（只看「第二部分：节奏骨架」，不得看源文原文）
【源文】⚠️ 不得读取源文原文！只参考节奏骨架，事件必须全部原创。

## 第N章 [章名]

- 核心事件：[2-3句话，自己编的原创事件]
- 因果逻辑：[A→B→C→D，每个环节一句话]
- 触发逻辑：[⚠️ Lv2 去重：为什么发生，必须和源文不同]
- 节奏模式：[情绪强度变化，如 低→高→低]
- 钩子：[章末钩子，1句话]
- 爽点来源：[爽感公式，如"身份反转""装逼打脸"]
- 原创元素：[⚠️ 至少1个源文完全没有的元素：新人物/新道具/新支线/新关系，不得为空]

【输出】保存到：{新书名}/大纲/章纲_N.md
"""
```

### B3.5 重排下一批源文映射（主 agent 跑，与 B4 并行）

**触发时机**：B3 写完本批章纲后，**与 B4 写本批章节并行**（不阻塞 B4）

**执行方式**：主 agent 自己跑（不并行子 agent），1 次 LLM 调用。

**步骤**：
1. 读取 {新书名}/设定/全书弧线骨架.md，确认下一批章节的情绪目标
2. 读取本批 10 个章纲，分析剧情走向
3. 按弧线骨架的需求，从源文池子里挑下一批章节（功能匹配，不按顺序）
4. 保存到 {新书名}/设定/章节顺序.md

```
例（11-20 章的弧线目标：甜宠→冲突→和解→高潮）
从源文里挑：
| 新书章号 | 源文章号 | 功能 | 匹配理由 |
|---------|---------|------|---------|
| 11 | 23 | 甜宠 | 弧线要求甜宠开局 |
| 12 | 18 | 甜宠 | 继续甜 |
| 13 | 25 | 冲突 | 弧线要求第3章转冲突 |
| ... |
```

```
例（11-20 章写完后，重排 21-30）：
| 新书章号 | 源文章号 | 功能 | 章节核心 |
|---------|---------|------|---------|
| 21 | 23 | 甜宠 | 修闹钟+男主护妻 |
| 22 | 25 | 冲突 | 白月光上门 |
| 23 | 24 | 甜宠 | 送饭+耳根红 |
| ... |
```

### B4 第三轮：写章（并行 10 agent）
```
每个 Task 的 prompt：
"""
请写出《{新书名}》第N章正文。

你是《{新书名}》的作者本人。请以作者的身份写出这一章。

【章纲】请读取：{新书名}/大纲/章纲_N.md
【风格指南】请读取：novel-download-authors/{作者名}/{源书名}/蒸馏/mode-b/style_guide_X.md
（X = 章节顺序.md 中新书第N章对应的源文章号，只看「第一部分：风格指南」，跳过节奏骨架）
【全书弧线骨架】请读取：{新书名}/设定/全书弧线骨架.md（本章在全书中的位置、该阶段的情绪目标）
【新书概念】请读取：{新书名}/设定/新书概念.md（核对角色名、关系、设定）

【字数硬约束】
- 目标：2000-2500字，硬上限3000字
- 不得超出，写到2500字左右必须收尾

【反洗稿规则】
- 禁止照搬源文的句子结构（只换名词不行）
- 禁止复制源文的对话内容（即使换了角色名）
- 禁止模仿风格指南中的「排除项」特征
- 可以借鉴：节奏模式、爽点公式、情绪弧线
- 必须原创：具体场景描写、对话内容、人物反应、因果细节

【质量要求】
- 每章至少1个让读者笑/心动/紧张的瞬间
- 对话必须有信息量，不得水对话
- 结尾必须有钩子，让读者想看下一章
- 场景描写用具体细节，不用空泛形容词
- 角色行为必须符合章纲中的因果逻辑，不得突然降智

【输出】保存到：{新书名}/正文/第N章.txt
"""
```

### B5 循环
1. 检查是否还有未写章节
2. 有 → 回到 B1 启动下一批（B3.5 已生成新映射，下一批 B3 用新映射）
3. 无 → 进入 Phase 3 收尾
