# story-long-write：长篇网文写作

长篇网文写作全流程，从选题到完本。

**核心信念：网文写作是工程，不是灵感。**

**架构原则：主线程只做协调，agent自己读文件。**

---

## 流程总览

```
Phase 1-2（选题+设定）     Phase 3（大纲+细纲）     Phase 4（正文并行）      Phase 5（质量检查）
┌─────────────┐          ┌─────────────┐          ┌──────────────┐          ┌─────────────┐
│    agent     │          │    agent     │          │ 50个agent并行│          │    agent     │
├─────────────┤          ├─────────────┤          ├──────────────┤          ├─────────────┤
│ 设定+角色   │ ──────→  │ 大纲+细纲   │ ──────→  │ 写正文       │ ──────→  │ 去AI味+审查 │
└─────────────┘          └─────────────┘          └──────────────┘          └─────────────┘
   5分钟                    10分钟                    10分钟                   10分钟
```

**总耗时：~35分钟（50章）**

**opencode限制**：只支持 `task(subagent_type: "general")`，不支持自定义agent类型。

**解决方案**：在prompt中告诉general agent去读取哪些参考文件，让它获得对应能力。

---

## 场景选择

| 场景 | 触发条件 | 执行流程 |
|------|----------|----------|
| **开书** | "帮我开书" / 项目目录为空 | 完整 Phase 1→2→3→4→5 |
| **日更续写** | "日更"/"续写"/"继续写" 且项目已有正文 | 加载 `references/workflow-daily.md` |
| **快速开书** | "/快写" + 书名 + 题材 | 跳过选题，直接 Phase 2→3→4→5 |

## Phase 1：选题方向（5分钟）

如果用户已有方向 → 直接进入 Phase 2。

如果用户没有方向：

问用户：**「你想写什么类型？有没有喜欢的书想对标？你的优势是什么？」**

根据回答做匹配：
- 脑洞好 → 系统文、诸天流、无限流
- 文笔好 → 仙侠、历史、文艺向都市
- 节奏感好 → 都市爽文、重生文、游戏文
- 生活经验丰富 → 行业文、都市日常、种田文

---

## Phase 2：核心设定（5分钟）

调用 general agent 创建设定：

```
task(subagent_type: "general", prompt: "
你是网文设定专家。

任务：创建小说核心设定和角色。

项目目录：{dir}
书名：{书名}
题材：{题材}
一句话梗概：{梗概}

请先读取以下参考文件获取设定模板：
- .claude/agents/story-architect.md（了解故事架构师的能力）
- .claude/agents/character-designer.md（了解角色设计师的能力）

然后创建：
1. 设定/核心设定.md（基本信息、主角设定、世界观、核心冲突、三卷规划）
2. 设定/角色/{主角名}.md（主角卡：外貌、性格、家庭、动机链、人物弧线）
3. 设定/角色/{配角1-5}.md（配角卡）
4. 设定/关系.md（角色关系图）
5. 设定/题材定位.md（核心梗解析、爽点设计）

要求：
- 角色性格鲜明，有2-3个特点
- 核心梗要有差异化
- 爽点设计要具体
")
```

---

## Phase 3：大纲+细纲（10分钟）

调用 general agent 创建大纲：

```
task(subagent_type: "general", prompt: "
你是网文大纲专家。

任务：创建大纲和细纲。

项目目录：{dir}
书名：{书名}
题材：{题材}

请先读取以下参考文件获取大纲方法：
- .claude/agents/story-architect.md（了解大纲排布能力）
- 设定/核心设定.md（了解核心设定）
- 设定/角色/*.md（了解角色设定）

然后创建：
1. 大纲/大纲.md（三卷结构：卷名、章节、字数、核心事件、状态变化）
2. 大纲/卷纲_第一卷.md（章节规划表、情绪弧线、人物弧线）
3. 大纲/细纲_第001章.md 到 大纲/细纲_第050章.md

每章细纲格式：
```
## 第N章：{章名}

### 上一章结尾状态
{描述}

### 核心事件
一句话

### 章首钩子
类型：{钩子类型}
内容：...

### 爽点
类型：{爽点类型}
内容：...

### 章尾钩子
类型：{钩子类型}
内容：...
期待度：强/中/弱

### 字数目标
2000字
```

要求：
- 第1章上一章结尾状态写"无（故事开始）"
- 每章必须有爽点
- 第50章结尾要有悬念
")
```

## Phase 4：正文并行（10分钟）

Phase 3 完成后，启动 50 个 general agent 并行写正文。

**主线程职责**：只分配任务（章节号），不读取任何文件

**每个 agent 的调用方式**：

```
task(subagent_type: "general", prompt: "
你是网文写作专家。

任务：写第{N}章正文，约2000字。

项目目录：{dir}

请先读取以下参考文件获取完整写作能力：
- .claude/agents/narrative-writer.md
  - 场景展开法（进入场景→展开子事件三层→收尾）
  - 情绪弧线执行（情弦理论、三机位法、拉扯节奏）
  - 开篇创作（前100字事件密度>=3）
  - 收尾创作（禁止升华式收束）
  - 去AI味6 Gate（禁用词替换、句式去套路、心理外化、节奏打碎、对话去腔调、结尾去升华）
  - 节长达标（每章>=2000字，字数不足时的扩充流程）
  - 禁止事项（禁止写总结感悟、禁止连续排比、禁止直接写情绪词等）
- .claude/skills/story-deslop/references/banned-words.md
  - 一级禁用词（出现即替换）
  - 二级禁用词（高频出现时替换）
  - 替换策略速查

然后读取任务文件：
- 大纲/细纲_第{N}章.md（本章细纲）
- 设定/角色/*.md（角色设定）
- 正文/第{N-1}章_*.md（上一章，如存在）

按narrative-writer的场景展开法写正文：
1. 进入场景：主角此刻在哪、在做什么（1-2句切入）
2. 展开子事件：每个子事件三层展开
   - 发生层：这件事出现了（1-2句叙事，含具体细节，~30-40字）
   - 感知层：主角注意到什么细节（2-3句感官/物件，至少2个不同感官，~60-80字）
   - 反应层：身体如何回应（1-3句身体动作，~40-60字）
3. 收尾：钩子或情绪定格（1-2句）

写入：正文/第{N}章_{章名}.md
完成后返回章名和字数。
")
```

## Phase 4.5：合并TXT（1分钟）

正文完成后，自动合并为TXT文件，截取前45000字：

```powershell
$projectDir = "{项目目录}"
$bookName = "{书名}"
$genre = "{题材}"
$maxChars = 45000

$synopsis = ""
if (Test-Path "$projectDir/设定/核心设定.md") {
    $synopsis = "【简介】`n" + ((Get-Content "$projectDir/设定/核心设定.md" -Raw) -split "## 一句话梗概" | Select-Object -Last 1 -split "##")[0].Trim()
}

$files = Get-ChildItem -Path "$projectDir/正文" -Filter "*.md" | Sort-Object Name

$chars = 0
$body = ""

foreach ($f in $files) {
    $c = Get-Content -Path $f.FullName -Raw -Encoding UTF8
    if ($chars + $c.Length -gt $maxChars) {
        break
    }
    $body += "`r`n`r`n$c`r`n`r`n"
    $chars += $c.Length
}

$header = "========================================`r`n${bookName}`r`n========================================`r`n作者：AI生成`r`n题材：${genre}`r`n`r`n${synopsis}`r`n`r`n========================================`r`n`r`n"
Set-Content -Path "$projectDir\${bookName}.txt" -Value ($header + $body) -Encoding UTF8
Write-Host "已生成：${bookName}.txt（约${chars}字）"
```

## Phase 5：质量检查（10分钟）

正文完成后，执行质量检查：

### 5.1 去AI味

启动 50 个 general agent 并行去AI味：

```
task(subagent_type: "general", prompt: "
你是网文润色专家。

任务：对第{N}章进行去AI味处理。

项目目录：{dir}
文件：正文/第{N}章_{章名}.md

请先读取以下参考文件获取完整去AI味能力：
- .claude/agents/narrative-writer.md
  - Gate A 禁用词替换：命运齿轮/如潮水般/仿佛春风/心猛地一沉等全部替换
  - Gate B 句式去套路：连续排比/刻意对称/空洞抒情打散
  - Gate C 心理描写外化：情绪词→身体状态
  - Gate D 节奏打碎：长句拆短、同构句打散（单段<=3句、短句先行、口语化）
  - Gate E 对话去腔调：所有角色同一语气→差异化
  - Gate F 结尾去升华：大段抒情收尾→安静细节收尾
  - 系统性去AI三遍法：Pass1去泛化→Pass2去书面化→Pass3回人味
- .claude/skills/story-deslop/references/banned-words.md
  - 一级禁用词（出现即替换）
  - 二级禁用词（高频出现时替换）
  - 替换策略速查

然后对正文文件执行完整去AI味流程，修改后覆盖原文件，返回修改统计。
")
```

### 5.2 编辑评测

调用 general agent 审查：

```
task(subagent_type: "general", prompt: "
你是网文编辑专家。

任务：审查第1-50章的质量。

项目目录：{dir}

请先读取以下参考文件获取审查能力：
- .claude/agents/narrative-writer.md（了解五维评分、9项检查）
- .claude/skills/story-deslop/references/banned-words.md（禁用词表）

然后抽查第1、2、3、25、26、48、49、50章，检查：
1. AI味分级（轻度/中度/重度）
2. 禁用词命中
3. 格式合规
4. 节奏均匀度
5. 五维评分（代入感/节奏/信息密度/去AI度/情绪弧线）

输出审查报告，保存到 审查报告.md。
")
```

### 5.3 一致性检查

调用 general agent 检查事实一致性：

```
task(subagent_type: "general", prompt: "
你是一致性检查员。

任务：检查第1-50章的事实矛盾。

项目目录：{dir}

请先读取以下参考文件获取检查能力：
- .claude/agents/consistency-checker.md（了解S1-S4分级、检查流程）

然后检查：
1. 角色属性是否前后一致
2. 伏笔是否合理埋设/回收
3. 时间线是否自洽
4. 跨卷衔接是否顺畅

输出S1-S4分级报告。
")
```
