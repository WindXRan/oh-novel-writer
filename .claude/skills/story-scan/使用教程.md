## 1. 这是什么？能帮我做什么？

**番茄指数** 是一个自动追踪番茄小说排行榜数据的工具。它每天自动抓取番茄小说四大榜单的数据，并通过可视化看板帮你分析趋势。

### 适合谁用？

| 角色 | 能帮你做什么 |
|------|-------------|
| **网文作者** | 了解哪些题材正在上升、哪些在衰退，找到创作方向 |
| **编辑/运营** | 追踪榜单变化，发现潜力新书，监控市场动态 |
| **数据分析师** | 获取结构化的排行榜数据，导出 Excel/CSV 做进一步分析 |
| **普通读者** | 发现热门新书，看看哪些书在榜上 |

### 核心能力

- **自动抓取**：四大榜单（男频新书/阅读、女频新书/阅读），每个分类 Top 30
- **趋势分析**：自动对比昨日数据，标记新上榜、掉榜、排名变化
- **AI 洞察**（可选）：接入 AI 服务自动生成市场趋势摘要
- **创作辅助**：题材热度、竞品对标、读者画像、创作方向建议
- **数据导出**：支持 Excel、CSV、JSON 三种格式

## 2. 环境准备（只需做一次）

### 2.1 安装 Python

本项目需要 **Python 3.9 或更高版本**。

**检查是否已安装：**

打开命令提示符（按 `Win+R`，输入 `cmd`，回车），输入：

```bash
python --version
```

如果显示 `Python 3.9.x` 或更高版本，说明已安装，跳到 2.2。

如果没有安装：
1. 访问 https://www.python.org/downloads/
2. 点击黄色的 "Download Python 3.x.x" 按钮
3. 下载后双击安装，**务必勾选 "Add Python to PATH"**（安装界面底部的复选框）
4. 点击 "Install Now"
5. 安装完成后重新打开命令提示符，再次输入 `python --version` 确认

### 2.2 下载项目

将项目文件夹放到你喜欢的位置，例如 `C:\mimo\FanqieZhiShu`。

### 2.3 安装依赖

打开命令提示符，进入项目目录：

```bash
cd C:\mimo\FanqieZhiShu
```

安装 Python 依赖：

```bash
pip install -r requirements.txt
```

安装浏览器（爬虫需要）：

```bash
playwright install chromium
```

等待下载完成即可。整个过程大约 2-5 分钟。

## 3. 快速开始：三步看数据

### 第一步：抓取数据

```bash
python scrape_fanqie_ranks.py
```

爬虫会自动访问番茄小说网站，抓取四大榜单的数据。预计耗时 **10-15 分钟**。

你会看到类似这样的输出：

```
[12:30:00] 开始抓取男频新书榜...
[12:30:05] 📖 抓取男频新书榜 - 都市
[12:30:08] ✅ 都市: 30 本书
...
```

> **提示：** 爬虫支持断点续传。如果中途断网或电脑重启，再次运行同一命令会从上次中断的地方继续，不会重复抓取。

### 第二步：构建看板数据

```bash
python scripts/build_latest.py
```

这一步会分析抓取到的数据，生成趋势对比和摘要。耗时约 **1-2 分钟**（不含 AI 分析）。

输出类似：

```
✅ 女频新书榜: 15 个分类已处理
✅ 男频新书榜: 13 个分类已处理
...
```

### 第三步：打开看板

```bash
python -m http.server 8000
```

然后打开浏览器，访问 **http://localhost:8000**

你会看到番茄指数的主界面，包含侧边栏分类列表、书籍卡片、趋势标签等。

> **提示：** 按 `Ctrl+C` 可以停止服务器。

## 4. 功能详解

### 4.1 榜单看板（主页）

**访问地址：** `http://localhost:8000`

这是最核心的页面，展示当日排行榜数据。

#### 页面布局

```
┌─────────────────────────────────────────────────────┐
│  顶部导航栏：[女频新书] [女频阅读] [男频新书] [男频阅读]  │
│                [趋势风向标] [创作灵感]  ⚙️             │
├──────────┬──────────────────────────────────────────┤
│          │                                          │
│  分类    │   风向标区域（趋势统计 + AI 摘要）          │
│  列表    │                                          │
│          │   [搜索框] [筛选▼] [排序▼] [导出▼]        │
│  ·都市   │                                          │
│  ·古风   │   ┌──────┐ ┌──────┐ ┌──────┐            │
│  ·玄幻   │   │ 书1  │ │ 书2  │ │ 书3  │            │
│  ·仙侠   │   │ 作者  │ │ 作者  │ │ 作者  │            │
│  ·...    │   │ 阅读量│ │ 阅读量│ │ 阅读量│            │
│          │   └──────┘ └──────┘ └──────┘            │
│          │   ┌──────┐ ┌──────┐ ┌──────┐            │
│  日期    │   │ 书4  │ │ 书5  │ │ 书6  │            │
│  ◀ 今日 ▶│   └──────┘ └──────┘ └──────┘            │
└──────────┴──────────────────────────────────────────┘
```

#### 操作说明

**切换榜单类型：**
- 点击顶部导航栏的 Tab（女频新书/女频阅读/男频新书/男频阅读）
- 也可以用 URL 参数：`http://localhost:8000/?rank=male_new`

**切换分类：**
- 点击左侧边栏的分类名称（都市、古风、玄幻...）
- 当前选中的分类会高亮显示

**搜索书籍：**
- 在搜索框中输入书名或作者名，实时过滤

**筛选和排序：**
- **筛选**：全部 / 新上榜 / 排名上升 / Top 10 / Top 20
- **排序**：按排名 / 按阅读量 / 按书名

**查看历史数据：**
- 使用底部的日期导航：`◀` 上一天 / `▶` 下一天
- 点击日期可以打开日历选择器，跳转到指定日期

**导出数据：**
- 点击"导出"下拉按钮，选择 Excel / CSV / JSON 格式
- 导出的是当前分类的数据

**复制书籍信息：**
- 点击任意书籍卡片，自动复制书名、作者、阅读量、简介和链接到剪贴板

#### 风向标区域

页面顶部的风向标区域展示：

| 内容 | 说明 |
|------|------|
| 新增上榜 | 今天新进入该分类的书籍数量 |
| 掉出榜单 | 昨天在榜但今天消失的书籍数量 |
| 排名上升 | 排名比昨天提高的书籍数量 |
| AI Signal Brief | AI 生成的趋势摘要（需配置 AI 服务） |

趋势标签会显示在每本书的卡片上：
- 🟢 **新上榜** — 首次进入该分类
- 🔺 **+3** — 排名上升 3 位
- 🔻 **-2** — 排名下降 2 位
- ➖ **持平** — 排名不变

### 4.2 趋势风向标

**访问地址：** `http://localhost:8000/trend.html`

多日趋势分析页面，帮你发现市场风向变化。

#### 功能模块

**全站热点：**
- 热门分类热度排名（如：都市 > 古风 > 玄幻）
- 具体分类热度排名
- 高频题材词云（出现频率越高的题材字号越大）

**风向强度：**
- 综合评分（0-100 分）
- 诊断文字（如"热度上升，新书活跃"）

**关键指标：**
- 新增上榜数 / 掉出榜单数 / 活跃天数

**日线波动：**
- 柱状图展示每日上榜和掉榜的节奏

**新上榜焦点 / 上升作品 / 增长焦点：**
- 紧凑列表展示值得关注的书籍

**每日风向摘要：**
- 按日期排列的趋势观察记录

#### 时间范围

点击页面上的时间范围按钮切换分析周期：
- **7 日** — 近一周趋势
- **14 日** — 近两周趋势
- **30 日** — 近一月趋势
- **全部** — 全部数据

#### 分类筛选

使用顶部的分类下拉框，可以查看特定分类的趋势数据。

### 4.3 创作灵感

**访问地址：** `http://localhost:8000/author.html`

面向网文作者的数据分析工具，帮你找到创作方向。

#### 四大分析模块

**① 题材热度分析**
- **高频题材词云**：展示近期出现频率最高的题材关键词
- **Top 6 题材趋势图**：展示热门题材随时间的变化曲线
- **高频共现题材对**：经常一起出现的题材组合（如"重生+复仇"、"穿越+系统"）

**② 竞品对标分析**
- 选择一个分类，查看该分类的：
  - 共性关键词（头部作品都用了哪些词）
  - 标题模式（常见标题结构）
  - 阅读量分布（Top 10 / Top 20 / Top 30 的阅读量区间）
  - 简介钩子（高阅读量作品的简介特点）

**③ 创作方向建议**
- 选择一个分类（如"古风"、"都市日常"），AI 会生成：
  - 推荐题材（当前有潜力的方向）
  - 差异化机会（竞争较少的细分领域）
  - 示例书名（参考用）
  - 饱和题材（竞争激烈、不建议进入的方向）
  - 综合建议
- 跨分类混搭推荐：组合不同分类的热门元素

**④ 读者偏好画像**
- 选择一个分类，查看：
  - 热门元素（读者最关注的关键词）
  - 情感偏好（甜宠/虐恋/爽文/日常的占比）
  - 金手指偏好（系统/空间/重生/穿越的占比）
  - 背景设定偏好（现代/古代/架空/末世的占比）

#### 数据导出

- 每个模块右上角都有独立的导出按钮
- 页面顶部有"导出全部"按钮，一键将所有分析数据导出为 Excel

### 4.4 数据导出

所有页面都支持数据导出，格式包括：

| 格式 | 适用场景 |
|------|---------|
| **Excel (.xlsx)** | 打开即用，适合做图表、分享给同事 |
| **CSV** | 轻量格式，适合导入其他工具（如 pandas、Tableau） |
| **JSON** | 结构化数据，适合程序处理 |

## 5. 日常使用流程

### 每日推荐流程

```
┌──────────────────┐
│  1. 抓取今日数据   │  python scrape_fanqie_ranks.py
│     (~10-15分钟)  │
└────────┬─────────┘
         ▼
┌──────────────────┐
│  2. 构建看板数据   │  python scripts/build_latest.py
│     (~1-2分钟)    │
└────────┬─────────┘
         ▼
┌──────────────────┐
│  3. 打开看板查看   │  python -m http.server 8000
│     http://localhost:8000 │
└──────────────────┘
```

### 快捷命令汇总

```bash
# 抓取数据
python scrape_fanqie_ranks.py

# 构建看板（纯规则摘要，不需要 AI）
python scripts/build_latest.py

# 构建看板（带 AI 分析）
API_BASE_URL=https://api.deepseek.com/v1 API_KEY=你的密钥 API_MODEL=deepseek-chat python scripts/build_latest.py

# 启动预览
python -m http.server 8000

# 停止预览
# 在运行服务器的终端按 Ctrl+C
```

### 数据积累

- **第 1 天**：只能看到当日榜单，无趋势对比
- **第 2 天起**：自动对比昨日数据，显示排名变化
- **第 7 天起**：7 日趋势分析可用
- **第 14 天起**：14 日趋势分析可用
- **第 30 天起**：30 日趋势分析可用

**建议每天运行一次爬虫**，数据积累越久，趋势分析越准确。

## 6. 配置 AI 分析（可选）

AI 分析功能可以生成更智能的市场趋势摘要和创作建议，但不是必须的。不配置时系统会使用基于规则的摘要。

### 支持的 AI 服务

| 服务 | API_BASE_URL | API_MODEL | 说明 |
|------|-------------|-----------|------|
| DeepSeek | `https://api.deepseek.com/v1` | `deepseek-chat` | 性价比高，推荐 |
| OpenAI | `https://api.openai.com/v1` | `gpt-4o-mini` | 效果最好，价格较高 |
| Moonshot | `https://api.moonshot.cn/v1` | `moonshot-v1-8k` | 国内服务 |
| 硅基流动 | `https://api.siliconflow.cn/v1` | `Qwen/Qwen2.5-7B-Instruct` | 有免费额度 |
| 本地 Ollama | `http://localhost:11434/v1` | `qwen2.5:7b` | 完全免费，需本地部署 |

### 配置方式

**方式一：环境变量（推荐用于命令行）**

```bash
# Windows CMD
set API_BASE_URL=https://api.deepseek.com/v1
set API_KEY=sk-你的密钥
set API_MODEL=deepseek-chat
python scripts/build_latest.py

# Windows PowerShell
$env:API_BASE_URL="https://api.deepseek.com/v1"
$env:API_KEY="sk-你的密钥"
$env:API_MODEL="deepseek-chat"
python scripts/build_latest.py

# Linux/Mac
API_BASE_URL=https://api.deepseek.com/v1 API_KEY=sk-你的密钥 API_MODEL=deepseek-chat python scripts/build_latest.py
```

**方式二：创建 .env 文件（推荐用于固定配置）**

在项目根目录创建 `.env` 文件：

```
API_BASE_URL=https://api.deepseek.com/v1
API_KEY=sk-你的密钥
API_MODEL=deepseek-chat
```

然后直接运行 `python scripts/build_latest.py`，程序会自动读取。

**方式三：前端页面配置（仅影响前端展示）**

1. 打开看板页面
2. 点击右上角的齿轮图标 ⚙️
3. 填入 API Base URL、API Key、Model
4. 点击"保存"

此配置保存在浏览器中，仅影响前端页面的 AI 展示，不影响后端构建。

### AI 分析的输出

配置 AI 后，以下功能会增强：

| 功能 | 无 AI | 有 AI |
|------|-------|-------|
| 分类趋势摘要 | 规则模板（短小、固定格式） | AI 生成（更详细、更有洞察） |
| 全站热点分析 | 规则统计 | AI 生成的市场趋势解读 |
| 创作方向建议 | 不可用 | AI 生成的分类分析和建议 |

## 7. 常见问题

### Q: 运行 `python` 提示"不是内部或外部命令"

**原因：** Python 没有添加到系统 PATH。

**解决：**
1. 重新安装 Python，安装时勾选 "Add Python to PATH"
2. 或者手动添加：搜索"环境变量" → 编辑 Path → 添加 Python 安装路径

### Q: 爬虫运行到一半卡住了

**原因：** 可能是网络波动或页面加载超时。

**解决：** 直接关闭终端，重新运行 `python scrape_fanqie_ranks.py`。爬虫会自动从上次中断的地方继续。

### Q: 爬虫报错 "playwright install chromium"

**原因：** 浏览器未安装。

**解决：** 运行 `playwright install chromium`，等待下载完成。

### Q: 打开看板页面是空白的

**原因：** 还没有运行过爬虫和构建。

**解决：** 先执行步骤 1（抓取数据）和步骤 2（构建看板），然后再打开页面。

### Q: 看板页面显示"加载失败"

**原因：** 服务器未启动或端口被占用。

**解决：**
1. 确认已运行 `python -m http.server 8000`
2. 如果端口被占用，换一个端口：`python -m http.server 8080`，然后访问 `http://localhost:8080`

### Q: 趋势页面显示"暂无数据"

**原因：** 只有一天的数据，无法做趋势对比。

**解决：** 连续运行爬虫 2 天以上，趋势数据会自动生成。

### Q: AI 分析没有生效

**原因：** 未配置 AI 服务或配置错误。

**解决：**
1. 确认 `.env` 文件中的 API Key 和 URL 正确
2. 确认 API 账户有余额
3. 运行构建时观察输出是否有 "AI 总结已生成" 字样

### Q: 如何查看更早的历史数据？

在榜单页面底部的日期导航栏，点击 `◀` 可以查看前一天的数据。点击日期数字可以打开日历选择器，跳转到任意有数据的日期。

### Q: 数据文件越来越大怎么办？

数据文件会随着使用时间增长。如果磁盘空间紧张：
- `data/fanqie_*_ranks_YYYYMMDD.json` 是每日快照，可以删除不需要的历史日期
- `data/trends/` 是趋势归档，可以清理早期文件
- `data/author/` 是分析数据，删除后重新构建会自动再生成

**注意：** 删除数据后，对应日期的趋势分析将不可用。

### 数据文件命名规则

| 文件类型 | 命名格式 | 示例 |
|---------|---------|------|
| 每日快照 | `fanqie_{prefix}_ranks_YYYYMMDD.json` | `fanqie_female_new_ranks_20260529.json` |
| 最新数据 | `latest_{prefix}_ranks.json` | `latest_female_new_ranks.json` |
| 日期索引 | `dates_{prefix}.json` | `dates_female_new.json` |
| 市场热点 | `market_summary_{prefix}.json` | `market_summary_female_new.json` |
| 趋势归档 | `trends/{prefix}_YYYY-MM-DD.json` | `trends/female_new_2026-05-29.json` |
| 爬虫状态 | `task_state_{prefix}_YYYYMMDD.json` | `task_state_female_new_20260529.json` |
| 题材趋势 | `author/theme_trends_{prefix}.json` | `author/theme_trends_female_new.json` |
| 竞品分析 | `author/competitive_analysis_{prefix}.json` | `author/competitive_analysis_female_new.json` |
| 读者画像 | `author/reader_profile_{prefix}.json` | `author/reader_profile_female_new.json` |
| 创作建议 | `author/creation_suggestions_{prefix}.json` | `author/creation_suggestions_female_new.json` |

**榜单前缀对照：**
- `female_new` — 女频新书榜
- `female_read` — 女频阅读榜
- `male_new` — 男频新书榜
- `male_read` — 男频阅读榜

**API 接口文件：**
- `api/latest/{prefix}_all.json` — 该榜单所有分类的完整数据
- `api/latest/{prefix}_index.json` — 该榜单的分类索引
- `api/latest/{prefix}_{category}.json` — 指定分类的数据
