# skill-iterate：A-B 协作优化 skill

## 三种输入模式

| 模式 | 用户输入 | 流程 |
|------|---------|------|
| **接方案** | 已有具体方案，需要 A-B 审查打磨 | 直接进 A-B 循环，A 优化方案，B 审查 |
| **给目标** | 有改进方向但没方案 | A 先出方案，再走 A-B 循环 |
| **全自动** | 只给 skill 路径 | explore 分析问题 → A 出方案 → A-B 循环 |

## 流程

1. **读取目标 skill**：读取用户指定的 SKILL.md
2. **判断输入模式**：
   - 用户贴了方案文本 → 接方案模式
   - 用户说了改进方向 → 给目标模式
   - 用户只给了路径 → 全自动模式
3. **启动 A-B 循环**（最多 3 轮）：

### 每轮循环

**Step 1：Agent A 提出/优化方案**
- 启动 Task(subagent_type="general")
- 指令：「你是 skill 优化执行者。{模式对应指令}。当前 SKILL.md 内容：{内容}。提出具体修改方案，包括改哪些部分、改成什么。」
- 返回：修改方案

**Step 2：Agent B 审查**
- 启动 Task(subagent_type="general")
- 指令：「你是 skill 审查者，负责挑战和找漏洞。当前 SKILL.md：{内容}。Agent A 的修改方案：{方案}。审查这个方案：架构是否清晰？有没有遗漏？有没有过度设计？有没有违反职责分离原则？如果方案合格，回复「通过」。否则给出具体改进建议。」
- 返回：审查结果

**Step 3：判断**
- B 回复包含「通过」→ 结束循环，执行修改
- B 给出建议 → 进入下一轮，A 根据建议修改
- 达到 3 轮 → 输出当前最佳方案

4. **执行修改**：用 edit 工具按最终方案修改 SKILL.md
5. **输出总结**：改了什么、为什么改

## 注意事项

- A 和 B 都是 general agent，不要用 explore（它不擅长推理）
- 每轮 A 的 prompt 要包含 B 上一轮的反馈
- 如果用户没给优化方向，先用 explore agent 快速分析 skill 的问题点
- 修改必须保持 skill 原有的职责边界，不越界
