# 题材管理（对标InkOS）

## 概述

题材管理是仿写的基础，不同题材有不同的：
- 章节类型
- fatigue词表
- 节奏规则
- 爽点类型
- 审计维度
- 题材禁忌

---

## 内置题材

### 1. 都市（urban）

```yaml
name: 都市
id: urban
chapterTypes: ["商战章", "社交章", "布局章", "过渡章", "回收章"]
fatigueWords: ["冷笑", "不可思议", "震惊", "难以置信", "深吸一口气", "眼中闪过一丝", "仿佛", "不禁", "宛如", "竟然", "核心动机", "信息边界"]
pacingRule: "每2-3章一个小回报：商业收益、人脉拓展、对手受挫、信息优势"
satisfactionTypes: ["商战碾压", "身份揭示", "人脉兑现", "对手打脸", "资源收割", "地位跃升"]
```

**题材禁忌**：
- 无逻辑的商业奇迹（没有铺垫的暴富）
- 反派降智配合主角表演
- 无视现实法律和商业规则
- 用"一个电话搞定"跳过具体操作过程
- 女性角色沦为花瓶或奖励

**语言铁律**：
- 人物内心独白必须口语化、直觉化
- ✗"他迅速分析了当前的债务状况" → ✓"他把那叠皱巴巴的白条翻了三遍"
- 法律/商业术语必须匹配设定年代的真实语感

### 2. 玄幻（xuanhuan）

```yaml
name: 玄幻
id: xuanhuan
chapterTypes: ["战斗章", "布局章", "过渡章", "回收章"]
fatigueWords: ["冷笑", "蝼蚁", "倒吸凉气", "瞳孔骤缩", "不可置信", "轰然炸裂", "满场死寂", "难以置信", "仿佛", "不禁", "宛如", "竟然"]
pacingRule: "三章内必有明确反馈：打脸、收益兑现、信息反转、地位变化"
satisfactionTypes: ["打脸", "升级突破", "收益兑现", "智斗碾压", "身份揭示", "底牌亮出"]
```

**题材禁忌**：
- 主角为推剧情突然仁慈、犯蠢、讲武德
- 同质资源不写衰减默认全额结算
- 用"暴涨""海量"跳过数值结算
- 无铺垫的能力觉醒
- 反派像木桩一样排队送死

**数值规则**：
- 设定不可吃书：前文确立的设定数值后文不可无升级过程地随意改变
- 金手指四维约束：能力上限、附加代价、触发条件、成长路径
- 同质资源重复吞噬必须写明衰减

```yaml
name: 仙侠

pacingRule: "修炼/悟道与战斗交替，每3-5章一次小突破或关键收获"
satisfactionTypes: ["悟道突破", "斗法碾压", "法宝收获", "身份揭示", "天劫渡过", "因果了结"]
```

**题材禁忌**：
- 主角为推剧情突然仁慈、犯蠢
- 修为无铺垫跳跃式突破
- 法宝凭空出现解决危机
- 天道规则前后矛盾

**修炼规则**：
- 境界突破必须有积累过程
- 同质资源重复炼化必须写明衰减
- 法宝体系分品级，使用有代价

### 4. 恐怖（horror）

```yaml
name: 恐怖
id: horror
chapterTypes: ["氛围章", "事件章", "揭示章", "过渡章", "回收章"]
fatigueWords: ["毛骨悚然", "不寒而栗", "浑身发冷", "头皮发麻", "鸡皮疙瘩", "心跳加速", "仿佛", "不禁", "宛如", "竟然"]
pacingRule: "氛围递进：安全感→微妙不适→确认异常→恐惧升级→高潮→喘息，循环推进"
satisfactionTypes: ["真相揭示", "成功逃脱", "反杀怪物", "谜团解开", "同伴获救", "规则发现"]
```

**题材禁忌**：
- 恐怖源头过早完全暴露（未知才恐怖）
- 主角无脑刚正面解决一切
- 用打脸/升级等爽文套路替代恐怖氛围

**恐惧层级**：
- 第一层：不适感（微妙的错位、违和）
- 第二层：不安（确认有异常，但看不清全貌）
- 第三层：恐惧（威胁明确化，逃生本能启动）
- 第四层：绝望（规则被打破，安全感彻底崩塌）

## 使用方式

### 1. 选择题材

在Phase 1初始化时，根据源文选择题材：
```
源文标签：现代言情|穿越|甜宠|大佬
→ 选择题材：言情/甜宠（romance）
```

### 2. 应用题材规则

在写作时，应用题材的：
- fatigue词表（去AI）
- 节奏规则（章纲）
- 爽点类型（设计）
- 题材禁忌（避免）

### 3. 审计时检查

在审计时，检查是否符合题材规则：
- 是否违反题材禁忌
- 是否符合节奏规则
- 是否有足够爽点

---

## 自定义题材

如果内置题材不匹配，可以自定义：

```yaml
name: 自定义题材
id: custom
chapterTypes: ["类型1", "类型2"]
fatigueWords: ["词1", "词2", "词3"]
pacingRule: "节奏规则"
satisfactionTypes: ["爽点1", "爽点2"]
auditDimensions: [1,2,3,6,7,8,9,10,13,14,15,16,17,18,19,24,25,26]
```
