# 去AI系统（对标InkOS）

## 概述

本系统对标InkOS的去AI能力，包含：
1. fatigue词表（每个题材独立）
2. AI检测规则（11条确定性规则）
3. 5种修订模式
4. anti-detect流程（检测→改写→重检测循环）
5. 跨章重复检测

## 2. AI检测规则（11条确定性规则）

### 规则1：连接词过频
**检测**：然而、但是、因此、所以、于是、接着、然后
**阈值**：每1000字超过5次
**修复**：用动作切换或段落分隔替代

### 规则2：转折词过频
**检测**：虽然...但是、尽管...还是、即使...也
**阈值**：每1000字超过3次
**修复**：直接写结果，省略转折

### 规则3："了"字过频
**检测**：了
**阈值**：每100字超过8个
**修复**：用动作完成态替代

### 规则4：句式规律
**检测**：连续3句以上结构相同
**修复**：打破句式，长短交替

### 规则5：段落长度单一
**检测**：连续5段长度相近（±10字）
**修复**：段落长度差异化

### 规则6：情绪标签化
**检测**：感到、觉得、意识到、明白
**阈值**：每1000字超过3次
**修复**：情绪外化为动作或细节

### 规则7：叙述者结论
**检测**：显然、当然、毫无疑问、不用说
**修复**：删掉，让读者自己感受

### 规则8：群像反应笼统
**检测**：全场震惊、众人皆惊、所有人都
**修复**：具体化为1-2个角色的反应

### 规则9：fatigue词
**检测**：fatigue词表中的词
**修复**：替换为更自然的表达

### 规则10：抽象描述
**检测**：强大的力量、巨大的能量、无尽的可能
**修复**：用具体细节落地

### 规则11：重复表达
**检测**：同一段落中相同词汇出现3次以上
**修复**：替换为同义词或改写

## 3. 5种修订模式

### 3.1 polish（轻微修改）
**适用**：质量较好，只需微调
**操作**：
- 修正错别字
- 调整标点
- 优化个别词汇

### 3.2 spot-fix（定点修复）
**适用**：局部有问题，其他部分正常
**操作**：
- 定位问题段落
- 只重写问题部分
- 保持其他部分不变

### 3.3 rewrite（大幅重写）
**适用**：整体质量差，需要大幅修改
**操作**：
- 保留核心情节
- 重写大部分内容
- 保持情绪弧线

### 3.4 rework（结构调整）
**适用**：结构有问题，需要调整顺序
**操作**：
- 重新组织段落
- 调整情节顺序
- 优化节奏

### 3.5 anti-detect（反检测改写）
**适用**：AI痕迹明显，需要专门处理
**操作**：
- 执行11条检测规则
- 逐条修复问题
- 循环直到通过

---

## 4. Anti-detect流程

```
Step 1: 检测
  - 执行11条确定性规则
  - 标记所有问题
  - 输出：detection_report.json

Step 2: 改写
  - 根据检测报告逐条修复
  - 优先修复critical问题
  - 输出：revised_chapter.txt

Step 3: 重检测
  - 对改写后的内容重新检测
  - 如果仍有问题，回到Step 2
  - 循环直到所有问题清除

Step 4: 输出
  - 最终版本
  - 检测报告（通过/未通过）
```

## 5. 跨章重复检测

### 5.1 检测内容
- 相同句式在多章出现
- 相同比喻在多章出现
- 相同过渡句在多章出现
- 相同情绪描写在多章出现

### 5.2 检测方式
- 提取每章的"句式指纹"
- 跨章比对相似度
- 标记重复率>30%的章节

### 5.3 修复方式
- 替换重复的句式
- 使用不同的比喻
- 调整过渡方式

## 6. 执行流程

### 6.1 写作阶段（预防）

在Writer的system prompt中注入：
```
## 去AI规则

### 禁用词
{fatigue词表}

### 禁用句式
- 连续使用"然而"、"但是"、"因此"
- 连续3句以上结构相同
- "感到"、"觉得"、"意识到"
- "全场震惊"、"众人皆惊"

### 必须做到
- 句式长短交替
- 段落长度差异化
- 情绪外化为动作
- 群像反应具体化
```

### 6.2 审计阶段（检测）

在Auditor中添加AI检测维度：
```
维度14：AI痕迹
- 执行11条确定性规则
- 检测fatigue词
- 检测句式规律
- 检测跨章重复
```

### 6.3 修订阶段（修复）

如果审计不通过，执行anti-detect流程：
```
Step 1: 检测 → detection_report.json
Step 2: 改写 → revised_chapter.txt
Step 3: 重检测 → 循环直到通过
```

## 9. 与审计系统集成

在10维度审计中，维度7（AI痕迹）使用本系统：

```
维度7：AI痕迹
- 执行11条确定性规则
- 检测fatigue词
- 检测句式规律
- 检测跨章重复
- 评分标准：
  - 0个critical问题：9-10分
  - 1-2个critical问题：7-8分
  - 3-4个critical问题：5-6分
  - 5个以上critical问题：3-4分
```
