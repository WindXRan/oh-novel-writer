# Truth Files 模板

**每本书维护 5 个 truth files，作为审计和写作的唯一事实来源。**

---

## 文件清单

| 文件 | 用途 | 更新时机 |
|------|------|---------|
| `current_state.md` | 世界状态：角色位置、关系网络、已知信息、情感弧线 | 每章写完后 Observer 更新 |
| `pending_hooks.md` | 未闭合伏笔：铺垫、对读者的承诺、未解决冲突 | 每章写完后 Observer 更新 |
| `chapter_summaries.md` | 各章摘要：出场人物、关键事件、状态变化、伏笔动态 | 每章写完后 Observer 生成 |
| `character_matrix.md` | 角色交互矩阵：相遇记录、信息边界 | 每章写完后 Observer 更新 |
| `emotional_arcs.md` | 情感弧线：按角色追踪情绪变化和成长 | 每章写完后 Observer 更新 |

## Observer 更新规则

每章写完后，spawn 1 个 Observer agent：

```
读取本章正文 + 上一版 truth files，输出 JSON delta：
{
  "current_state": { "角色位置": {...}, "关系网络": {...}, "已知信息": {...} },
  "pending_hooks": { "新增": [...], "状态更新": [...] },
  "chapter_summary": { "章名": "...", "出场人物": [...], "关键事件": [...], ... },
  "character_matrix": { "新增相遇": [...], "信息边界更新": [...] },
  "emotional_arcs": { "苏念": {...}, "陆沉": {...} }
}
```

代码层做 immutable update + Zod 校验，坏数据直接拒绝。
