# story-rewrite：超级仿写引擎

## 核心原理

**仿写 = 学习成功模式 + 创新具体内容**

```
源文 → 成功因子提取 → 新书
     → 文风蒸馏    → 新书
     → 结构映射    → 新书
```

**不是照搬情节，而是学习源文为什么火，然后用新的人物和场景重新演绎。**

## 设计原则

### 分层分析，逐层深入

```
Phase 1（快速扫描）        Phase 2（深度分析）
├── 全书统计指纹           ├── 本区间10章逐章分析
├── 章节标题→卷结构        ├── 本区间统计指纹
└── 输出：新书概念         └── 输出：章纲+正文
```

**为什么这样分**：
- LLM 处理10章左右的内容是靠谱的
- 全文分析会导致 token 爆炸、深度不够、遗漏关键信息
- 每个区间独立蒸馏，保证文风准确性

## 速查表

```
输入：源文 txt
输出：完整小说

Phase 1：初始化（轻量化）
  1.0 创建目录 + 拆章 + 快速扫描
  1.1 新书概念（含设定+简介）
  1.2 初始化真相文件

Phase 2：写作（每区间10章，循环执行）
  2.1 提取源文 + 深度分析（逐章分析+蒸馏）
  2.2 章纲细化
  2.3 写新章（3步管线）
  2.4 去AI（每10章执行一次anti-detect）
  2.5 状态沉淀 → 下一区间

Phase 3：收尾
  3.1 一致性终检
  3.2 导出
```

## Phase 1：初始化（轻量化）

### 1.0 创建目录 + 拆章 + 快速扫描

**Step 1：创建目录**

⚠️ **重要**：目录名必须使用新书的名字，不能用"仿写项目"、"新书"等临时名称。
- 如果书名尚未确定，先完成1.1新书概念生成，确定书名后再创建目录
- 如果已有"仿写项目"等临时目录，在确定书名后必须立即重命名

```bash
mkdir -p {书名}/设定
mkdir -p {书名}/大纲
mkdir -p {书名}/正文
mkdir -p {书名}/真相文件
```

**Step 2：源文全书拆章（⚠️ 缓存机制）**

**源文位置**：`novel-download-authors/{作者名}/{书名}.txt`

**拆章缓存**：`novel-download-authors/{作者名}/{书名}/` （公共位置，多次仿写共用）

```bash
# 检查是否已拆章
if novel-download-authors/{作者名}/{书名}/ 目录存在且有文件:
    跳过拆章，直接使用缓存
else:
    python source_chapter_splitter.py split <源文.txt> novel-download-authors/{作者名}/{书名}/
```

**仿写书目录**：`{新书名}/` （只放新书内容，不放源文）

**Step 3：快速扫描（⚠️ 缓存机制）**

```bash
# 检查是否已扫描
if novel-download-authors/{作者名}/{书名}/统计指纹.md 存在:
    跳过扫描，直接使用缓存
else:
    python style_analyzer.py <源文.txt> --name "源文全书"
    扫描章节标题，识别卷结构
```

**缓存的好处**：
- 同一本书多次仿写时，不用重复拆章和扫描
- 源文拆章在公共位置，多个仿写项目共用
- 仿写书目录只放新书内容，干净整洁

---

### 1.1 新书概念（含设定+简介）

基于快速扫描结果，设计新书：

**新书概念**（`{书名}/设定/新书概念.md`）：
- 书名候选（3个）
- 类型定位
- 核心卖点（保留哪些成功因子）
- 人物设定（沿用类型模板，换具体身份背景）
- 故事弧线（三幕结构，几句话）
- 关键转折点（位置+事件，一行一个）
- 与源文的差异化

**世界观设定**（`{书名}/设定/story_bible.md`）：
- 世界规则
- 角色设定
- 地点设定
- 关键道具

**简介**（`{书名}/简介.md`）：
- 3个版本（标准版/悬念版/甜宠版）

---

### 1.2 初始化真相文件

**⚠️ 真相文件用 Markdown 格式，不是 JSON**

```
{书名}/真相文件/
├── current_state.md        # 当前状态
├── pending_hooks.md        # 伏笔池
├── chapter_summaries.md    # 章节摘要
├── character_matrix.md     # 角色关系矩阵
└── emotional_arcs.md       # 情绪弧线
```

初始化时只填框架，具体内容在 Phase 2 逐步填充。

## Phase 2：写作（每区间10章）

### 继续仿写流程

**⚠️ 继续仿写时，必须执行完整的Phase 2流程，不能跳过任何步骤！**

```
继续仿写{书名}，从第{N}章开始
    ↓
Step 1: 检查当前进度（正文/目录）
    ↓
Step 2: 确定本区间（如第33-40章）
    ↓
Step 3: 提取源文第33-40章
    ↓
Step 4: 深度分析（逐章分析+蒸馏）
    ↓
Step 5: 章纲细化（基于源文章节分析）
    ↓
Step 6: 写新章（子agent）
    ↓
Step 7: 去AI
    ↓
Step 8: 状态沉淀
```

**⚠️ 不能跳过Step 3-5直接写新章！**

### 执行流程

```
┌─────────────────────────────────────────────────────┐
│  循环执行，每次处理10章                                │
│                                                       │
│  2.1 提取源文 + 深度分析（逐章分析+蒸馏）              │
│      ├── 逐章分析（核心事件/情绪弧线/钩子）             │
│      ├── 蒸馏 Layer A：统计指纹                       │
│      └── 蒸馏 Layer B：LLM 8维度分析                  │
│      ↓                                                │
│  2.2 章纲细化（基于深度分析，含源文对应）               │
│      ↓                                                │
│  2.3 写新章（3步管线）                                │
│      ↓                                                │
│  2.4 去AI（每10章执行一次anti-detect）                │
│      ↓                                                │
│  2.5 状态沉淀（更新真相文件）→ 下一区间                │
└─────────────────────────────────────────────────────┘
```

### 2.1 提取源文 + 深度分析

**⚠️ 每个区间必须重新分析，不能复用其他区间**

**Step 1：提取本区间源文（⚠️ 从公共缓存提取）**

```bash
# 从公共缓存提取本区间源文
python source_chapter_splitter.py extract novel-download-authors/{作者名}/{书名}/ <start> <end> <{新书名}/源文_{start}-{end}.txt>
```

**注意**：
- 源文拆章在 `novel-download-authors/{作者名}/{书名}/` （公共位置）
- 提取到 `{新书名}/源文_{start}-{end}.txt` （临时文件，用完可删除）

**Step 2：逐章分析（情节结构）（⚠️ 缓存机制）**

```bash
# 检查是否已分析（在公共缓存中）
if novel-download-authors/{作者名}/{书名}/源文分析_{start}-{end}.md 存在:
    跳过分析，直接使用缓存
else:
    使用 prompts/chapter-analyzer.md 分析本区间每章
    保存到公共缓存
```

输出：`novel-download-authors/{作者名}/{书名}/源文分析_{start}-{end}.md`

**Step 3：蒸馏 Layer A（统计指纹）（⚠️ 缓存机制）**

```bash
# 检查是否已蒸馏（在公共缓存中）
if novel-download-authors/{作者名}/{书名}/蒸馏_{start}-{end}.md 存在:
    跳过蒸馏，直接使用缓存
else:
    python style_analyzer.py {新书名}/源文_{start}-{end}.txt
    保存到公共缓存
```

**⚠️ 输入必须是本区间的10章，不是全书！**

**Step 4：蒸馏 Layer B（LLM 8维度分析）**

使用 `prompts/style-analysis.md` 分析本区间的：
1. 叙事声音与语气
2. 对话风格
3. 场景描写特征
4. 转折与衔接手法
5. 节奏特征
6. 词汇偏好
7. 情绪表达方式
8. 独特习惯

**⚠️ Step 4 必须引用 Step 3 的统计数据，禁止引用全书指标！**

输出：`novel-download-authors/{作者名}/{书名}/蒸馏_{start}-{end}.md`

### 2.2 章纲细化（套结构，换内容）

**⚠️ 章纲只借用源文的行文结构（情绪弧度/动作链/钩子位置），具体事件/冲突/动作必须全部原创**

**章纲模板**：

```markdown
## 第{N}章 [新书章名]

### 源文对应
- 源文第{M}章：[源文章名]
- 核心事件类型：[类型]
- 情绪弧线：[情绪流动]
- 钩子类型：[钩子]

### 新书情节（内容全部原创，与源文不同）
- 核心事件：[本章要发生什么]
- 利益冲突：[冲突是什么]（⚠️ 必须更换，不能和源文一样）
- 情绪弧线：[情绪流动]
- 动作链：[谁→做了什么→结果]（⚠️ 动作与反应全部更换）
- 钩子设计：[章末钩子]
- 爽点设计：[爽点来源]

### 关键细节（确保和源文不同）
- 场景：[在哪里发生]（⚠️ 必须更换，不能照搬源文场景）
- 新书NPC：[新名字/新设定]（⚠️ 所有NPC名字必须更换，配角设定必须差异化）
- 角色动态：[谁做了什么，关系变化]
- 信息揭示：[本章揭示什么新信息]
- 转折点：[关键转折是什么]

### 字数目标
- 目标：2000-2500字
- 硬上限：3000字（超出则精简）
- 硬下限：1800字（不足则扩充）
```

输出：`{书名}/大纲/章纲_{start}-{end}.md`

---

### 2.3 写新章（⚠️ 3步管线）

**Step 1：Writer（创作）**

读取：
- 章纲
- 真相文件（真相文件目录下所有 .md 文件）
- 蒸馏_{start}-{end}.md（**文风指南，核心参考**）

**⚠️ 文风规则主要用蒸馏数据，不要加太多通用规则，会死板！**

输出：第N章初稿（目标2000-2500字，硬上限3000字）

**Step 2：Auditor（质量审计）**

读取：第N章初稿

**⚠️ 必须使用 word_counter.py 统计字数，不要自己数！**

```bash
python word_counter.py {书名}/正文/第{chapter}.txt
```

使用 `prompts/auditor-system.md` 进行审计：
1. 字数合规（目标2000-2500字，硬上限3000字，硬下限1800字）← 用word_counter.py统计
2. 情绪弧线
3. 钩子设计
4. 人设一致性
5. AI痕迹

输出：审计报告（Markdown格式）

**如果不通过**：根据审计报告修复后重新提交

**Step 3：写入正文**

输出：`{书名}/正文/第{N}章.txt`

**并行执行（⚠️ 必须用 Task 工具启动子 agent）**：

```
⚠️ 禁止主 agent 直接写正文！
⚠️ 必须使用 Task 工具启动子 agent！

for chapter in range(start, end+1):
    Task(
        description=f"写第{chapter}章",
        subagent_type="general",
        prompt=f"""
【章纲】{本章章纲内容}

【文风指南】⚠️ 核心参考，请先读取：novel-download-authors/{作者名}/{书名}/蒸馏_{start}-{end}.md

【真相文件】请先读取：{书名}/真相文件/ 下所有 .md 文件

【输出要求】
- 输出到：{书名}/正文/第{chapter}章.txt
- 字数：目标2000-2500字，硬上限3000字，硬下限1800字
- ⚠️ 写完后必须用 word_counter.py 统计字数：python word_counter.py {书名}/正文/第{chapter}.txt
- 必须先读取蒸馏文件再开始写作
- 文风规则以蒸馏文件为准（只学句子节奏/对话密度/段长，不抄内容）
- ⚠️ 【仿写规则】只保留源章的行文结构（动作链/冲突推进/钩子位置）。利益冲突必须更换，动作与反应全部换，具体事件/场景/对话全部原创。和源文"换名字不改骨架"=洗稿，会被平台判定。
- ⚠️ 【防抄袭】所有NPC名字必须更换，所有配角设定必须差异化，所有具体场景必须更换，所有名场面必须重新设计。
- 输出纯正文，不要输出元数据或分析报告
"""
    )
```

### 2.4 去AI（每10章执行一次）

**⚠️ 每写完10章，执行一次anti-detect**

使用 `prompts/de-ai-system.md`：
- 11条确定性规则检测
- fatigue词表检查
- anti-detect流程（检测→改写→重检测循环）

**执行方式**：
```
对本区间10章逐章检测：
  Step 1: 检测 → detection_report.json
  Step 2: 如果有critical问题 → 改写
  Step 3: 重检测 → 循环直到通过

⚠️ 改写后必须用 word_counter.py 重新统计字数
```

### 2.5 状态沉淀

## 字数统计规则

**与番茄小说网站保持一致（番茄标准字数）**：
- 统计所有非空格字符
- 包括：汉字、标点符号、数字、英文字母
- 不包括：空格、换行符

**使用方式**：
```python
from word_counter import count_words

# 统计文本字数（番茄标准）
words = count_words("这是一段测试文本")
```

**命令行**：
```bash
python word_counter.py <txt文件或目录>
```

**输出示例**：
```
字数（番茄标准）：2345
```

## 常见错误

❌ **错误1**：Phase 1 深度分析全文
✅ **正确**：Phase 1 只做快速扫描（统计指纹+章节标题→卷结构）

❌ **错误2**：Phase 2 用全书统计指标
✅ **正确**：Phase 2 必须用本区间10章的统计指标

❌ **错误3**：章纲照搬源文情节（不换利益冲突和动作反应）
✅ **正确**：章纲套用行文结构，更换利益冲突和动作反应

❌ **错误4**：跳过深度分析
✅ **正确**：每个区间必须重新做完整分析（逐章分析+蒸馏）

❌ **错误5**：不更新真相文件
✅ **正确**：每章必须更新真相文件

❌ **错误6**：照搬源文情节
✅ **正确**：保留情绪弧线，创新具体情节

❌ **错误7**：真相文件用 JSON 格式
✅ **正确**：真相文件用 Markdown 格式
