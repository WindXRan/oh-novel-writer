# story-rewrite：结构仿写

**核心信念：只仿"骨相"，不仿"皮相"。**

---

## 流程总览

```
Phase 1（主线程）    读源文本 → 提取框架 + 设定锁 + 语感样本
Phase 2（试水）      3章并行写 → 字数修复 → story-review → 输出试水文件
Phase 3（续写）      创建项目 → 循环：出章纲15章 → 5章并行写 → 修字数 → review → 更新追踪
```

---

## Phase 1：框架提取

主线程读源文本，提取：

1. **源文本分析**（不只是读前3章）：
   - 前3章：开篇方式、破冰方式、人设标签、叙述者风格
   - 中间转折点：读第1/4、1/2、3/4处各1章，提取节奏变化、情绪转折、冲突升级方式
   - 最后1章：结局走向、伏笔回收方式
   - 全书章数、卷结构、每卷核心事件
   - 提取3-5段语感样本（每段30-100字）+ 1章完整样本供agent深度模仿

2. **仿写框架**：新书名/主角/背景/核心冲突/开篇差异/破冰差异/节奏对齐表/角色名表
3. **设定锁**（所有agent共享的硬设定）：
   - 核心地名（具体地址）
   - 关键物件（来源+状态）
   - 人物关系锁（不可更改的关系）
   - 背景事件锁（统一的过去事件描述）
   - 时间线锁（各章时间约束）
4. **语感特征**：3-5个核心节奏特征（一句话描述+简短示例）

**反调色盘自检**：新书的"核心设定+核心冲突+角色原型"和源文本放一起对比，如果外人看了觉得"这不就是同一本书吗"，必须重写框架。具体维度：

| 维度 | 红线 |
|------|------|
| 核心设定 | 机制不能相同（只是换金额/地点不算差异化） |
| 核心冲突 | 驱动矛盾不能同类 |
| 角色原型 | 身份/处境/动机不能同类 |
| 开篇方式 | 第1章核心事件不能同类 |
| 破冰方式 | 第一次互动方式不能同类 |

## Phase 2：试水（前3章）

试水阶段**不创建项目文件夹**，只输出 `仿写试水库/试水_{书名}.txt`。

### Step 1：写正文（3章并行）→ ⚠️ 阻断点

启动3个 `narrative-writer` agent 并行写第1-3章。prompt注入设定锁+语感样本+框架中的关键事件锚点。agent直接返回完整正文（不写文件）。

**⚠️ 阻断点：字数自动修复（3章全部返回后，主线程逐章执行）**

```powershell
# 对每章正文执行：
$chars = ($content -replace '[\s]','').Length
if ($chars -gt 2400) { # 按段落边界截断 }
if ($chars -lt 1800) { # 用 task_id 要求补写 }
```

### Step 2：story-review 一致性审查

启动 `story-review` 检查3章之间的一致性。发现矛盾 → 主线程直接改文本。

### Step 3：输出

合并3章正文写入 `仿写试水库/试水_{书名}.txt`（含简介+框架摘要+正文），向用户汇报字数。

**⚠️ 试水阶段不生成任何项目目录。用户确认后才创建正式项目结构。**

---

## Phase 3：续写

用户确认试水后，**首先创建项目目录**：

```
{书名}/正文/
{书名}/设定/（写入设定锁.md）
{书名}/大纲/
{书名}/追踪/（上下文.md/伏笔.md/时间线.md/角色状态.md）
```

然后按以下循环写完全书。**每一步都是阻断点，不完成无法进入下一步。**

```
循环（每批）：
  1. story-architect 出15章章纲 → 写入文件
     ⚠️ 阻断：主线程读章纲检查质量
  2. 5章并行写 → 各自返回 → 主线程逐章自动修字数
     ⚠️ 阻断：字数不修复无法进入下一步
  3. story-review 一致性审查
     ⚠️ 阻断：不通过则修正文本，通过后才能写入正文目录
  4. 写入 {书名}/正文/第{NNN}章_{章名}.md
  5. 更新追踪文件（每批必做）：
     - {书名}/追踪/上下文.md（进度摘要+近6章详记+更早概要）
     - {书名}/追踪/伏笔.md（新增/回收伏笔）
     - {书名}/追踪/时间线.md（事件时序）
     - {书名}/追踪/角色状态.md（角色状态变化）
  6. 出下一批章纲（基于已写内容+追踪文件调整）
```

---

## Phase 4：全书写完后

1. **全书 deslop**：对所有正文执行 story-deslop 的结构去AI（句长随机化、段落打散、句首分散、动作-反应解耦）
2. **禁用词扫描**：grep 扫描一级禁用词，发现即替换
3. **一致性检查**：角色属性/时间线/伏笔状态，发现矛盾修正
4. **字数总校验**：每章 >2400 自动截断

```
{从章纲文件中读取，试水阶段从框架中的关键事件锚点表读取}

## 写作原则（按优先级执行）

你拥有以下参考文件，遇到具体问题时按需读取：
- `skills/story-long-write/references/style-craft.md`（镜头式写作、五感描写、文笔技巧、毒点规避）
- `skills/story-long-write/references/hooks-chapter.md`（章首7式+章尾13式+爽点设计）
- `skills/story-long-write/references/dialogue-mastery.md`（对话规则、权力博弈、人物语言差异化）
- `skills/story-long-write/references/emotional-arc-design.md`（六种弧线、期待感管理）
- `skills/story-long-write/references/commercial-core-methods.md`（卖点、节奏、情绪交付）
- `skills/story-long-write/references/anti-ai-writing.md`（去AI三遍法、7种模式检测、改写范例库）
- `skills/story-long-write/references/banned-words.md`（完整禁用词表）

### 1. 镜头式写作（最高优先级）
- 你是跟在主角身后的摄像机，只写能看到/听到/闻到/摸到的
- 不写"他心里想""他感到"，用动作和五感展示
- 每个段落=一个镜头，有明确的"拍摄对象"
- 五感：每个场景至少调动2种感官

### 2. 对话规则
- 每句对话必须推进剧情/塑造人物/制造张力之一
- 权力博弈：掌控者话短（≤10字），被动者话长（≥20字）
- 人物差异化：不同角色说话方式不同
- 口语化，不要书面腔

### 3. 情绪交付
- 不直接写"他愤怒/伤心/紧张"，用身体反应和行为展示
- 每章标注目的词：铺垫/高潮/爽点/打脸/人物塑造/设定
- 爽点公式：先不爽→制造希望→努力解决→得偿所愿
- 期待感：每章至少1个未闭环的期待

### 4. 章首钩子（选一种）
- 倒计时/意象预示/悬念对话/反差场景/闪前碎片/未完成动作/危机升级
- 同批内同类型钩子不超过2个

### 5. 章尾钩子（选一种）
- 突然揭示/紧急危机/未完成动作/两难抉择/神秘物品/意象钩子/留白
- 用动作或悬念收束，不要总结感悟

### 6. 去AI（必做）
- 禁用词：仿佛、犹如、宛若、如同、深吸一口气、缓缓、不禁、微微、轻轻、淡淡、眼中闪过、嘴角勾起、眉头微皱、心中一动、心头一震
- 句式：不用"脊背绷得笔直像XX""嘴角翘了一下不是笑是XX"等模板化比喻
- 段落：单句段≥30%，连续3段同句数必须打散
- 句首：连续3句同主语开头必须换
- 用具体动作代替抽象比喻，用生活化比喻代替万能比喻

## 格式
第一行：`第{N}章 {章名}`
每章末尾标注：目的词={xxx}
返回完整正文。
```

**prompt检查清单**（spawn前核对）：
- [ ] 设定锁 + 语感样本 + 角色名表 + 章纲 + 字数上限2400

## 字数自动修复

```powershell
# 对每个写入的章节文件执行：
$content = Get-Content $file -Raw -Encoding UTF8
$chars = ($content -replace '[\s]','').Length
if ($chars -gt 2400) {
    # 按段落边界截断
    $paragraphs = $content -split '(\r?\n\r?\n|\r?\n)'
    $result = ""
    foreach ($p in $paragraphs) {
        $test = $result + $p
        if (($test -replace '[\s]','').Length -gt 2350) { break }
        $result = $test
    }
    $result | Set-Content $file -Encoding UTF8
}
if ($chars -lt 1800) {
    # 用 task_id 恢复会话要求补写
}
```

## 章纲质量标准

主线程出章纲后必须检查：
1. **钩子多样性**：同批15章内，同类型章首钩子不超过2个
2. **情节点具体性**：必须写清"谁做了什么"，不能是"关系升温"这种空话
3. **情绪起伏**：每5章内至少1个情绪转折
4. 不通过 → 用 task_id 要求重写，告诉它哪些章不合格
