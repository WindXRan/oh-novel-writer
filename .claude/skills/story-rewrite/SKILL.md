# story-rewrite：结构仿写

提取爆款小说的深层叙事结构（骨架），用全新角色和背景填充。

**核心信念：只仿"骨相"，不仿"皮相"。**

**架构原则：主线程只做协调，agent自己读文件。**

**opencode限制**：只支持 `task(subagent_type: "general")`，在prompt中告诉agent去读取参考文件。

---

## 流程总览

```
Phase 1（结构提取）     Phase 2（正文并行）      Phase 3（合并输出）
┌─────────────┐          ┌──────────────┐          ┌─────────┐
│  主线程      │          │ 3个agent并行 │          │  主线程  │
├─────────────┤          ├──────────────┤          ├─────────┤
│ 读取源文本   │ ──────→  │ 仿写3章      │ ──────→  │ 合并TXT │
│ 提取结构     │          │              │          │         │
└─────────────┘          └──────────────┘          └─────────┘
   5分钟                    5分钟                    1分钟
```

**总耗时：~10分钟（试水3章）**

**输出：`仿写试水库/试水_{新书名}.txt`**

---

## Phase 1：接收源文本 + 结构提取（5分钟）

用户提供**源文本**（文件路径）后，主线程完成：

1. **读取源文本**：只读取前3章 + 最后1章
2. **提取结构要素**：
   - 节奏地图（每章情绪基调）
   - 转折点序列
   - 场景切换模式
   - 情绪拉扯技巧
   - 题材标签

3. **生成仿写框架**（保存在内存中，用于 Phase 2 传给 agent）：
   - 新书名、主角、背景、核心冲突
   - 节奏对齐表（3章）
   - 核心差异对照

**输出**：
```
## 仿写框架
[自动生成的框架]

⚡ 开始仿写...
```

---

## Phase 2：调用 agent 仿写（5分钟）

Phase 1 完成后，启动 3 个 general agent 并行仿写。

**主线程职责**：只分配任务，不读取任何文件

**每个 agent 的调用方式**：

```
task(subagent_type: "general", prompt: "
你是网文写作专家。

任务：仿写第{N}章，约2000字。

项目目录：{dir}
源文本：{源文本路径}

请先读取以下参考文件获取完整写作能力：
- .claude/agents/narrative-writer.md
  - 场景展开法（进入场景→展开子事件三层→收尾）
  - 情绪弧线执行（情弦理论、三机位法、拉扯节奏）
  - 去AI味6 Gate（禁用词替换、句式去套路、心理外化、节奏打碎、对话去腔调、结尾去升华）
  - 禁止事项（禁止写总结感悟、禁止连续排比、禁止直接写情绪词等）
- .claude/skills/story-deslop/references/banned-words.md
  - 禁用词表

然后读取源文本的第{N}章。

仿写框架：
{框架内容，直接嵌入prompt}

按narrative-writer的场景展开法仿写：
1. 进入场景：主角此刻在哪、在做什么（1-2句切入）
2. 展开子事件：每个子事件三层展开
   - 发生层：这件事出现了（1-2句叙事，含具体细节，~30-40字）
   - 感知层：主角注意到什么细节（2-3句感官/物件，至少2个不同感官，~60-80字）
   - 反应层：身体如何回应（1-3句身体动作，~40-60字）
3. 收尾：钩子或情绪定格（1-2句）

要求：
- 复刻源文本的节奏和情绪
- 替换所有具体细节（角色、背景、台词）
- 保持结构一致，但内容完全原创
- 禁止AI结尾

直接返回章节正文内容（不要写入文件）。
")
```

**简介生成**：在并行生成章节的同时，主线程生成简介（150-200字）

---

## Phase 3：合并输出（1分钟）

1. **等待所有章节完成**
2. **字数验证**：检查每章字数（目标2000字，误差±10%）
3. **合并为TXT**：写入 `仿写试水库/试水_{新书名}.txt`

**输出格式**：
```
========================================
{新书名}
========================================
作者：AI仿写
题材：{题材}

【简介】
{简介内容}

========================================
第1章 {章名}
========================================
{正文内容}

----------------------------------------
第2章 {章名}
----------------------------------------
{正文内容}

----------------------------------------
第3章 {章名}
----------------------------------------
{正文内容}

========================================
【后续方向】
- 第4章预告：[简述]
========================================
```

**完成后显示**：
```
✅ 仿写完成！
📁 文件：仿写试水库/试水_{新书名}.txt
📊 字数：Ch1=XXX | Ch2=XXX | Ch3=XXX
📖 简介：[生成的简介]
```

---

## 字数要求

- **字数统计方式**：统计汉字、标点、英文字母，排除空格和换行
- 先统计源文本每章字数
- 仿写每章字数误差不超过 ±10%
- 网文标准章节约 2000 字/章
- **每章必须达到 2000 字左右**（1800-2200字），不能过短

---

## 简介要求

**字数**：150-200字

**结构**：
1. **第一句**：主角身份 + 穿越/重生设定（1句话）
2. **第二句**：核心冲突 + 危机（1句话）
3. **第三句**：男主身份 + 关系（1句话）
4. **第四句**：悬念钩子 + 反转预期（1句话）
5. **补充**：题材标签 + 卖点关键词

**示例**：
```
【简介】
一朝穿越，沈念薇成了豪门替嫁新娘。
原主贪图彩礼准备跑路，她却要面对失明的冷面大佬。
三个月，是她洗白身份的唯一窗口。
可她不知道，这个"瞎子"根本没瞎。
【偏日常/甜宠/先婚后爱/双洁】
```

---

## 写作要求（传给 agent 的约束）

1. **严格复刻节奏**：源文本此处是压抑，新文本此处也必须压抑
2. **场景切换对齐**：保持相同的场景顺序
3. **情绪拉扯同步**：相同的手法（信息差、对比等）
4. **细节完全原创**：台词、道具、环境描写全部原创
5. **禁止模板化写法**：不能用泛滥的写法
6. **禁止AI结尾**：章尾禁止升华句、总结句、哲理句
