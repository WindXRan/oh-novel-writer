# story-rewrite：仿写引擎

## 仿写核心原则

**仿写 = 只抄骨架，不抄血肉**

```
骨架（保留）          血肉（必须换）
├── 情绪弧线           ├── 具体事件
├── 冲突类型           ├── 利益冲突
├── 钩子位置           ├── 动作与反应
└── 行文节奏           └── 场景/对话/细节
```

**判断标准：如果把新书的角色名换回源文名，读起来像"换皮"= 仿写失败**

举例：
```
源文：暴雨中，他被黑衣人追杀，逃入废弃仓库。
洗稿：暴雨中，叶白被黑社会追杀，逃入旧仓库。 ← 只换了名字，事件一样
仿写：叶白好心让同事搭顺风车，结果出车祸，同事反索赔10万。 ← 事件完全不同，但"危机+冲突"结构一致
```

**⚠️ 章纲写"核心事件"时，必须写与源文不同的事件，不能只换名字**

## Phase 1：初始化

### 1.0 拆章 + 统计指纹

```bash
# 源文拆章（缓存存在则跳过）
python source_chapter_splitter.py split <源文.txt> novel-download-authors/{作者名}/{书名}/

# 统计指纹（缓存存在则跳过）
python style_analyzer.py <源文.txt> --name "源文全书"
```

### 1.1 新书概念 → `{书名}/设定/新书概念.md`

必含项：
- 书名、类型、核心卖点
- 人物设定（主角+配角）
- **NPC命名映射表**（⚠️ 必填）
- 故事弧线、关键转折点、差异化

NPC映射表格式：
```
| 源文 | 新书 | 身份 |
|---|---|---|
| 楚靳寒 | 顾西洲 | 男主 |
| 宴金集团 | 盛恒集团 | 男主家族企业 |
| ... | ... | ... |
```

规则：源文所有有名有姓的NPC、组织名、地名必须换名。

### 1.2 世界观 → `{书名}/设定/story_bible.md`

世界规则、角色设定、地点设定、关键道具。

### 1.3 真相文件 → `{书名}/真相文件/`

```
current_state.md        # 当前状态
pending_hooks.md        # 伏笔池
chapter_summaries.md    # 章节摘要
character_matrix.md     # 角色关系矩阵
emotional_arcs.md       # 情绪弧线
```

初始化填框架，Phase 2 逐步填充。

## Phase 2：写作（每区间10章循环）

### 入口（新开/继续统一执行）

```
Step 0：检查进度
  → ls {书名}/正文/ → 已完成第1~N章
  → 下一区间 = (N+1) ~ (N+10)

Step 1：验证文件存在
  [1] 章纲 ← {书名}/大纲/章纲_{start}-{end}.md
  [2] 蒸馏 ← novel-download-authors/{作者名}/{书名}/蒸馏_{start}-{end}.md
  [3] 真相文件 ← {书名}/真相文件/*.md
  → 缺失则先执行2.1/2.2创建

Step 2：执行2.3写章
Step 3：执行2.4去AI
Step 4：执行2.5更新真相文件
```

禁止：读取源文章节原文、跳过验证直接写章

---

### 2.1 分析（缓存存在则跳过）

```bash
# 逐章分析
python source_chapter_splitter.py extract novel-download-authors/{作者名}/{书名}/ <start> <end> <{书名}/源文_{x-y}.txt>
# 用 prompts/chapter-analyzer.md 分析 → 源文分析_{x-y}.md

# ⚠️ 先跑脚本统计客观数据（必须先于大模型分析）
python style_analyzer.py {书名}/源文_{x-y}.txt --output {书名}/源文_{x-y}_stats.json
# 脚本会输出：句长分布、段长分布、对话占比、高频词等客观数据

# 再用大模型做定性分析
# 用 prompts/style-analysis.md 做8维度分析 → 蒸馏_{x-y}.md
```

**蒸馏输出要求**：提取"骨架模板"，不要求新书匹配具体节奏：
- ✅ 情绪弧线类型（如：失落→惊喜→尴尬→心动）
- ✅ 冲突类型（如：误会型、利益冲突型、身份揭露型）
- ✅ 钩子类型（如：悬念型、甜蜜型、反转型）
- ✅ 客观数据快照（从脚本结果复制，不能自己统计）
- ❌ 不要求：具体句长、段落数、对话占比等数值指标

### 2.2 章纲 → `{书名}/大纲/章纲_{x-y}.md`

模板：
```markdown
## 第N章 [章名]
- 源文对应：源文第M章 [源文章名]
- 源文事件：[源文发生了什么]（⚠️ 只作参考，不能照搬）
- 新书事件：[与源文完全不同的事件]（⚠️ 必须原创，不能只换名字）
- 情绪弧线：[情绪流动]（可与源文一致）
- 钩子：[章末钩子]（类型可与源文一致，但具体内容必须不同）
- 场景：[必须换]
- NPC：[必须用新书名称]
```

**⚠️ "新书事件"必须与"源文事件"不同，不能只换名字/对话/细节**

### 2.2.1 框架脱钩检查（章纲写完后必须执行）

章纲写完后，逐章对比核心事件，生成脱钩检查表：

```markdown
| 章节 | 源文核心事件 | 新书核心事件 | 相似度 | 判定 |
|------|-------------|-------------|--------|------|
| 第1章 | 被抓进派出所 | 被保安当跟踪狂 | 40% | ✅ 通过 |
| 第2章 | 雨中送伞 | 琴房偶遇 | 20% | ✅ 通过 |
| 第4章 | 军训唱歌上表白墙 | 军训唱歌上表白墙 | 80% | ❌ 需重写 |
```

**判定规则**：
- 相似度 ≤ 50%：✅ 通过
- 相似度 > 50%：❌ 必须重写该章事件，只保留情绪弧线

**相似度评估维度**：
1. 事件类型（如：都是"被误会"→相似）
2. 场景设定（如：都在"教学楼走廊"→相似）
3. 触发方式（如：都是"醉酒后偶遇"→相似）
4. 结果导向（如：都是"女主披外套"→相似）

**核心原则**：只要事件本身不同（被追杀 vs 出车祸），情绪弧线可以一致。

### 2.3 写章（3步强制管线）

**Step 1：验证文件**
```
[1] 章纲存在
[2] 蒸馏存在
[3] 真相文件存在
→ 3项全过才继续
```

**Step 2：并行启动所有章节（⚠️ 一个消息里发所有Task调用）**

```
⚠️ 禁止主agent直接写正文！必须用Task工具！
⚠️ 写10章时，必须在一个消息里同时启动10个Task，不要一个一个写！
⚠️ 每个Task独立，互不依赖，可以并行。

# 在一个消息里发送所有Task调用：
Task(description="写第31章", ...)  # 并行
Task(description="写第32章", ...)  # 并行
Task(description="写第33章", ...)  # 并行
...                                 # 并行
Task(description="写第40章", ...)  # 并行

每个Task的prompt相同结构，只有章号不同：
"""
你是{作者名}，请模仿{作者名}的写作风格，写出《{书名}}》第N章正文。

【章纲】请读取：{书名}/大纲/章纲_{x-y}.md 中第N章
【文风指南】请读取：novel-download-authors/{作者名}/{书名}/蒸馏_{x-y}.md
⚠️ 学习情绪弧线、冲突类型、钩子类型，不抄内容，不要求匹配具体节奏数值

【真相文件】请读取：{书名}/真相文件/ 下所有 .md 文件

【仿写指南】
仿写 = 只抄原书的骨架，填充自己的血肉（剧情）

❌ 抄袭：原封不动照抄
❌ 洗稿：没动剧情、场景、人设，只改变描写手法
✅ 仿写：只保留行文结构，其余全部更换

具体操作：
1. 结构照搬：情绪弧线、钩子位置、冲突推进方式与源文一致
2. 人物可以照搬：核心设定保留，但名字必须更换
3. 利益冲突必须更换：用新的利益冲突驱动剧情
4. 动作与反应全部更换：具体事件、场景、对话全部原创
5. 判断标准：如果把角色名换回源文名，读起来像"换皮"= 失败

【字数统计】写完后请运行以下命令统计字数：
python C:\Users\Administrator\Documents\trae_projects\AI网文小说项目\.claude\skills\story-rewrite\word_counter.py {书名}/正文/第N章.txt
目标：2000-2500番茄字符数（所有非空格字符），硬上限3000，硬下限1800

【输出】保存到：{书名}/正文/第N章.txt
"""
```

**Step 3：验证**
```bash
python word_counter.py {书名}/正文/第N章.txt
grep -n "楚靳寒\|宋云绯\|宴金集团\|海市\|青城\|柏庾\|李哲\|李妙\|胡瑶\|何总" {书名}/正文/第N章.txt
```

### 2.4 去AI（每10章）

用 `prompts/de-ai-system.md` 检测→改写→重检测循环。

### 2.5 更新真相文件

更新：current_state / pending_hooks / chapter_summaries / character_matrix / emotional_arcs

## 常见错误

| 错误 | 正确 |
|------|------|
| 不建NPC映射表 | Phase 1必须建 |
| 只换主角名 | 所有NPC/组织/地名都换 |
| 读源文章节原文 | 只读分析+蒸馏+真相文件 |
| 跳过验证直接写章 | 必须验证3文件存在 |
| 蒸馏用全书指标 | 用本区间10章指标 |
| 章纲照搬源文情节 | 只套结构，换事件 |
| ⚠️ 事件只换名字 | ⚠️ 事件必须完全不同（源文被追杀→新书出车祸） |
| ⚠️ 跳过框架脱钩检查 | ⚠️ 章纲写完后必须执行脱钩检查，相似度>50%的事件必须重写 |
