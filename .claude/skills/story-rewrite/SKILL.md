# story-rewrite：结构仿写

提取爆款小说的深层叙事结构（骨架），用全新角色和背景填充。

**核心信念：只仿"骨相"，不仿"皮相"。**

**架构原则：主线程只做协调，agent自己读文件。**

**opencode限制**：只支持 `task(subagent_type: "general")`，在prompt中告诉agent去读取参考文件。

---

## ⚠️ 核心约束：时间线连续

**这是本 skill 最重要的规则。** 3个并行 agent 各自只读自己的源章节，**对上一章在新书里结束在哪一无所知**，必然导致时间线错乱。

**解决方案**：Phase 1 框架必须包含「新书各章起始场景」，Phase 2 每个 agent 必须显式收到场景起点。

## 流程总览

```
Phase 1（结构提取）     Phase 2（正文并行）      Phase 3（合并输出）
┌─────────────┐          ┌──────────────┐          ┌─────────┐
│  主线程      │          │ 3个agent并行 │          │  主线程  │
├─────────────┤          ├──────────────┤          ├─────────┤
│ 读取源文本   │ ──────→  │ 仿写3章      │ ──────→  │ 一次write│
│ 提取结构     │          │              │    +      │ 合并TXT  │
│ + 定义各章    │          │ + 给定场景起点 │ ←────  │ 快检时间线│
│   场景起点   │          │  保证连续    │          │         │
└─────────────┘          └──────────────┘          └─────────┘
   5分钟                    5分钟                   30秒
```

**总耗时：~10分钟（试水3章）**

**输出：`仿写试水库/试水_{新书名}.txt`**

---

## Phase 1：接收源文本（仅需用户提供一次）

**用户只需要提供源文本（文件路径或直接粘贴原文），剩下全部由 AI 自主完成，不再问任何问题。**

主线程接收到源文本后，启动 1 个 general agent 并行完成结构提取 + 框架生成：

```
task(subagent_type: "general", prompt: "
你是网文结构分析专家。

项目目录：{dir}
源文本路径/内容：{源文本}

任务：读取前3章 + 最后1章，输出仿写框架。

⚠️⚠️⚠️ 最重要的规则：只提取抽象叙事模式，禁止提取任何具体情节！

## 什么是「结构」（可以提取）
- 叙事功能序列：铺垫→压抑→转折→爆发（抽象的情绪节奏）
- 人物关系模式：对立/保护/欺骗/试探（抽象的关系动态）
- 信息差类型：读者知道主角不知道 / 主角隐瞒身份 / 双向误解
- 情绪弧线：紧张→释放→暧昧→危机（抽象的情绪走向）
- 叙事技巧：倒叙、多视角、内心独白与行动交替

## 什么是「情节」（禁止提取）
- ❌ 具体场景：「撑伞站在巷口」「在汽车站等人」
- ❌ 具体道具：「柿子树」「母亲的信」「清汤挂面」
- ❌ 具体台词：任何可以直接搬运的对白
- ❌ 具体事件：「谎称警察解围」「进错门撞见催债」「敲门没人应」
- ❌ 具体细节：「座椅弹簧快戳出来的中巴」「三趟班车」

## 抽象化规则
将具体情节抽象为叙事功能：
- ✅ 「女主到达陌生小镇，进入未知环境」（抽象）
- ❌ 「女主坐了三趟班车到达雨中小镇」（具体）
- ✅ 「主角用谎言化解第三方冲突」（抽象）
- ❌ 「主角谎称派出所李警官找他来吓退催债人」（具体）
- ✅ 「关键人物出场，以食物表达隐性关怀」（抽象）
- ❌ 「老人煮了一碗清汤挂面加荷包蛋」（具体）

## 仿写框架
- 新书名：
- 新主角：（名字+核心标签）
- 新背景：（时代/世界观，必须与源文本不同时代/不同地域）
- 核心冲突：
- 节奏对齐表（3章）：
  - 第1章：情绪基调 / 叙事功能（如：铺垫→压抑→钩子） / 情绪弧线走向 / **新书起始情境**（抽象描述，不要具体场景）
  - 第2章：情绪基调 / 叙事功能 / 情绪弧线走向 / **新书起始情境（接第1章结尾）**
  - 第3章：情绪基调 / 叙事功能 / 情绪弧线走向 / **新书起始情境（接第2章结尾）**
- 人物关系模式（抽象）：
- 信息差设计：
- 核心差异对照（源 vs 新）：只写时代/地域/职业/身份层面的差异，不要写道具/台词层面的替换
- 题材标签：

直接输出框架内容，不要多余解释。
")
```

主线程等待 agent 返回框架后，直接进入 Phase 2。

---

## Phase 2：调用 agent 仿写（5分钟）

Phase 1 完成后，启动 3 个 general agent 并行仿写。

**主线程职责**：
1. 分配任务给 3 个 agent
2. 等待所有 agent 返回内容
3. **收集返回的章节内容（保存在内存中，不要写入单独文件）**
4. **同步生成简介（150-200字）**——与 Phase 2 等待并行，不额外耗时

**每个 agent 的调用方式**：

```
task(subagent_type: "general", prompt: "
你是网文写作专家。

任务：基于抽象叙事框架，原创第{N}章，约2000字。

项目目录：{dir}

请先读取以下参考文件获取写作能力：
- .claude/agents/narrative-writer.md（场景展开法、情绪弧线、去AI味6 Gate）
- .claude/skills/story-deslop/references/banned-words.md（禁用词表）

⚠️⚠️⚠️ 反抄袭硬约束（违反即0分）：
1. **禁止读取源文本**——你不允许读源文本的任何章节，你的唯一输入是下方的抽象框架
2. **禁止复制具体情节**——框架中只有抽象叙事功能（如「用谎言化解冲突」），你必须自行设计全新的具体事件
3. **禁止搬运句式**——所有句子必须原创，不得模仿源文本的句式结构
4. **禁止1:1映射**——不能把源文本的每个情节点都对应到新书，必须有自己的创作判断
5. **禁止保留源文本元素**——人名、地名、物品名、台词、比喻全部原创

## 时间线约束
本章在新书中的**起始情境**（抽象描述）：
{新书第N章起始情境}

上一章结尾（参考，不要重复）：
{新书第N-1章结尾情境（若N>1）}

## 仿写框架（抽象叙事模式，不是具体情节指南）
{框架内容}

## 写作要求
基于框架中的叙事功能和情绪弧线，**自行设计**具体情节：

1. **自行设计场景**：根据框架的「情绪基调」和「叙事功能」，自创符合该功能的具体场景。例如框架说「压抑」，你可以设计被排挤、被误解、陷入困境等任何压抑场景，不必与源文本的压抑场景对应
2. **自行设计事件**：每个叙事功能下需要什么事件，由你根据新背景和新人设独立构思
3. **场景展开**：按narrative-writer的三层展开法写每个场景
   - 发生层：具体事件（1-2句）
   - 感知层：感官细节（2-3句，至少2个感官）
   - 反应层：身体反应（1-3句）
4. **情绪弧线**：严格遵循框架的情绪走向（如紧张→释放），但具体的紧张/释放事件由你设计
5. **字数**：约2000字（1800-2200）

判断标准：写完后自检——如果把你的章节和源文本放在一起，读者会不会觉得「这就是换了名字的同一本书」？如果会，重写。

直接返回章节正文内容（不要写入文件）。
")
```

## Phase 3：合并输出（30秒内完成）

**核心原则：一次 write 搞定，不做多余步骤。**

Phase 2 的 agent 返回内容后，主线程**一口气完成以下操作**：

1. **生成简介**（150-200字，Phase 2 等待期间可提前构思）
2. **快速时间线核验**：只看第1章最后2句 → 第2章前2句、第2章最后2句 → 第3章前2句，确认衔接即可（不逐句审查）
3. **一次 write 合并为单个 TXT**：将简介 + 3章正文 + 后续方向拼接，直接写入 `仿写试水库/试水_{新书名}.txt`
4. **字数统计用 bash 一行命令完成**（写入文件后统计，不影响输出速度）
5. **抄袭自检**（可选，写入文件后执行）：如果 `.claude/agents/plagiarism-checker.md` 已部署，spawn `Agent(subagent_type: "plagiarism-checker")` 对试水 TXT 与源文本做五层对比检测。检测结果不写入 TXT，只在终端显示。如有 S1 级抄袭问题，提示用户「仿写质量不达标，建议重新试水」

**禁止的操作（拖慢速度）**：
- ❌ 不要先写临时章节文件再合并
- ❌ 不要逐章调 bash 统计字数再写文件
- ❌ 不要逐行审查时间线，只做首尾句快检
- ❌ 试水阶段不要生成仿写框架.md 等多余文件，只输出一个试水 TXT

> **框架持久化时机**：仿写框架（结构锚点+文风锚点+差异对照+质量红线）不随试水产出，而是在用户试水满意后执行 `/story-long-write` 开书时，由系统从试水 TXT + Phase 1 框架中提取并写入 `仿写框架.md`。

**输出格式**：
```

{新书名}
========================================
作者：AI仿写
题材：{题材}

【简介】
{简介内容}

源文本：{源书名} | {源题材} | {源章数}章
源文本路径：{源文本文件的完整路径，供后续继续仿写时使用}

结构锚点（节奏对齐表）：
| 章 | 情绪基调 | 叙事功能 | 情绪弧线走向 |
|----|----------|----------|-------------|

质量红线：

========================================

========================================
{正文内容}

----------------------------------------
{正文内容}

----------------------------------------
第3章 {章名}
----------------------------------------
{正文内容}

========================================

- 第4章预告：[简述]
========================================
```

**框架写入规则**：Phase 1 agent 返回的框架直接填入上方模板的「仿写框架」section，与简介、正文一起由一次 write 写入同一个 TXT。不单独生成任何文件。

## 字数要求

- **字数统计方式**：统计汉字、标点、英文字母，排除空格和换行
- 先统计源文本每章字数
- 仿写每章字数误差不超过 ±10%
- 网文标准章节约 2000 字/章
- **每章必须达到 2000 字左右**（1800-2200字），不能过短

---

## 简介要求

**字数**：150-200字

**结构**：
1. **第一句**：主角身份 + 穿越/重生设定（1句话）
2. **第二句**：核心冲突 + 危机（1句话）
3. **第三句**：男主身份 + 关系（1句话）
4. **第四句**：悬念钩子 + 反转预期（1句话）
5. **补充**：题材标签 + 卖点关键词

**示例**：
```
【简介】
一朝穿越，沈念薇成了豪门替嫁新娘。
原主贪图彩礼准备跑路，她却要面对失明的冷面大佬。
三个月，是她洗白身份的唯一窗口。
可她不知道，这个"瞎子"根本没瞎。
【偏日常/甜宠/先婚后爱/双洁】
```

---

## 写作要求（传给 agent 的约束）

1. **情绪弧线对齐**：源文本压抑的地方，新文本也必须压抑——但压抑的具体事件必须完全不同
2. **叙事功能对齐**：源文本用了「铺垫→冲突→钩子」，新文本可以复用这个功能序列——但每个功能下的具体情节必须原创
3. **禁止1:1映射**：不能把源文本的每个情节点都机械对应到新书。新书应该有自己的创作判断——哪些地方加戏、哪些地方删减、哪些地方反转
4. **禁止搬运句式**：所有句子必须原创，不得模仿源文本的句式结构、修辞手法、比喻方式
5. **禁止保留源文本元素**：人名、地名、物品名、台词全部原创
6. **自检标准**：写完后问自己——如果把新书和源文本放在一起，读者会不会觉得「这就是换了名字的同一本书」？如果会，重写
7. **禁止AI结尾**：章尾禁止升华句、总结句、哲理句

## ⚠️ 常见Bug与修复实录

| Bug | 原因 | 修复 |
|-----|------|------|
| **换皮抄袭**：新书被判定为源文本的洗稿 | Phase 1提取了具体情节而非抽象模式；Phase 2 agent读取了源文本章节 | Phase 1只提取叙事功能+情绪弧线（Layer 1-2），禁止提取具体场景/道具/台词；Phase 2禁止agent读源文本 |
| **时间线倒回** | Agent不知道上一章在新书里结束在哪 | Phase 1框架定义「新书各章起始情境」（抽象描述），Phase 2 prompt中嵌入 |
| **字数超纲** | 字数约束在prompt末尾，agent注意力衰减 | 字数要求放在靠前位置 |
| **源/新混淆**：新书出现源文本的角色名/地名 | 框架的差异对照表被忽略 | 差异对照表放在prompt前半部分，质量红线明确列出 |
