# story-rewrite：结构仿写

提取爆款小说的深层叙事结构（骨架），用全新角色和背景填充。

**核心信念：只仿"骨相"，不仿"皮相"。**

**架构原则：主线程只做协调，agent自己读文件。**

**opencode限制**：支持 `task(subagent_type: "general")` 和 `task(subagent_type: "narrative-writer")`。Phase 1 分析用 `general`，Phase 2 写作用 `narrative-writer`（内置场景展开法、去AI味等知识，无需重复读取参考文件）。

---

## ⚠️ 核心约束：时间线连续

**这是本 skill 最重要的规则。** 3个并行 agent 各自只读自己的源章节，**对上一章在新书里结束在哪一无所知**，必然导致时间线错乱。

**解决方案**：Phase 1 框架必须包含「新书各章起始场景」，Phase 2 每个 agent 必须显式收到场景起点。

## 流程总览

```
Phase 1（结构提取）      Phase 2（正文并行）       Phase 3（合并输出）
┌─────────────┐          ┌──────────────┐          ┌─────────┐
│  主线程      │          │ 3个agent并行  │          │ 2个agent│
├─────────────┤          ├──────────────┤          │  并行    │
│ 读取源文本   │ ──────→  │ 仿写3章      │ ──────→  ├─────────┤
│ 提取结构     │          │ (narrative-  │          │简介生成 │
│ + 定义各章    │          │  writer)     │          │+ 核验   │
│   场景起点   │          │  prompt精简  │          │+ 合并   │
└─────────────┘          └──────────────┘          └─────────┘
   ~4分钟                   ~2分钟                   ~30秒
```

**总耗时：~6.5分钟（试水3章），比旧版（general agent + 加载参考文件 + 串行合并）快约60%**

**输出：`仿写试水库/试水_{新书名}.txt`**

---

## Phase 1：接收源文本（仅需用户提供一次）

**用户只需要提供源文本（文件路径或直接粘贴原文），剩下全部由 AI 自主完成，不再问任何问题。**

主线程接收到源文本后，启动 1 个 general agent 并行完成结构提取 + 框架生成：

```
task(subagent_type: "general", prompt: "
你是网文结构分析专家。

项目目录：{dir}
源文本路径/内容：{源文本}

任务：读取前3章，输出仿写框架（直接输出，不要多余内容）。

⚠️ **时间线连续是核心约束**：3个agent各自仿写一章，互不知道对方的起始场景。因此框架必须明确定义每章在新书中的**起始场景**（从哪开始），确保 Agent 2 接 Agent 1 的结尾、Agent 3 接 Agent 2 的结尾。

## 仿写框架
- 新书名：
- 新主角：（名字+核心标签）
- 新背景：（时代/世界观）
- 核心冲突：
- 各章起始场景：
  - 第1章起始场景（主角此刻在哪、在做什么）：
  - 第2章起始场景（接第1章结尾）：
  - 第3章起始场景（接第2章结尾）：
- 核心差异对照（源 → 新：角色/背景/台词/道具的具体替换规则）：
- 题材标签：
")
```

主线程等待 agent 返回框架后，直接进入 Phase 2。

---

## Phase 2：调用 agent 仿写（~2分钟）

Phase 1 完成后，启动 3 个 `narrative-writer` agent 并行仿写。

> **速度优化说明**：旧版使用 `general` agent + prompt 嵌入完整场景展开法/去AI味指令 + agent 自行读取 `narrative-writer.md` 等参考文件，每个 agent 需要大量上下文加载。新版改用 `narrative-writer` subagent_type——该 agent 已内置场景展开法、情绪弧线执行、去AI味 6 Gate、禁用词表等全部写作知识，prompt 只需传递"写什么"而非"怎么写"，启动速度和生成速度均显著提升。

**主线程职责**：
1. 创建 `仿写试水库/` 临时目录
2. 分配任务给 3 个 agent（设置超时 5 分钟）
3. 等待所有 agent 返回完成信号
4. 如有 agent 超时，重新分配对应章节（最多重试 1 次）
5. 收集所有章节完成信息

**每个 agent 的调用方式**：

```
task(subagent_type: "narrative-writer", prompt: "
项目目录：{dir}
任务描述：仿写

## 核心信息
- 仿写章节：第{N}章
- 字数目标：约2000字（1800-2200）
- 输出文件：仿写试水库/temp_{2位序号}.txt
- 源文本路径：{源文本路径}（请读取第{N}章）
- 源文件格式：请按 {源文件名格式} 定位章节

## 时间线约束（最重要）
本章在新书中的**起始场景**（必须从此开始，不能回退）：
{新书第N章起始场景}

上一章结尾场景（参考，不要重复）：
{新书第N-1章结尾场景（若N>1）}

## 仿写框架
{框架内容，直接嵌入prompt}

## 写作硬约束
- 严格复刻源文本的节奏和情绪，替换所有具体细节（角色、背景、台词、道具），保持结构一致但内容完全原创
- 本章起始场景必须严格对齐新书时间线，不能倒退到上一章结束之前
- 禁止AI结尾：章尾禁止升华句、总结句、哲理句
- 将正文写入输出文件，不要返回内容
")
```

---

## Phase 3：合并输出（通用多章合并，~30秒 + 每5章+5秒）

**重要：输出必须是单个 TXT 文件，格式与原文一致！**

Phase 2 将每章写入了 `仿写试水库/temp_*.txt`，Phase 3 通过文件而非 inline 内容处理，prompt 大小不随章数增长，支持任意数量章节。

1. **字数验证**：直接用 bash 统计（即时完成，不启动 agent）：

   ```bash
   # PowerShell
   Get-ChildItem -LiteralPath "仿写试水库" -Filter "temp_*.txt" `
     | Sort-Object Name `
     | ForEach-Object { $c = Get-Content -Raw $_.FullName; "Ch$($_.Name -replace '\D','')=$($c.Length)" }
   # 或 python 跨平台
   # python3 -c "import os; [print(f'Ch{i+1}={len(open(f\"仿写试水库/temp_{i+1:02d}.txt\").read())}') for i in range(N)]"
   ```

2. **并行启动 2 个 task**（简介生成 + 时间线核验同时跑）：

   ```
   task 1 (subagent_type: "narrative-writer", prompt: "
   项目目录：{dir}
   任务描述：生成简介

   请读取仿写试水库/temp_01.txt 的前500字，了解故事开头。
   结合以下仿写框架，生成150-200字小说简介。

   ## 仿写框架
   {框架核心信息：新书名、新主角、新背景、核心冲突、题材标签}

   ## 简介结构
   1. 第一句：主角身份 + 穿越/重生设定
   2. 第二句：核心冲突 + 危机
   3. 第三句：男主身份 + 关系
   4. 第四句：悬念钩子 + 反转预期
   5. 补充：题材标签 + 卖点关键词

   直接返回简介文本（不要多余内容）。
   ")

   task 2 (subagent_type: "general", prompt: "
   项目目录：{dir}
   任务描述：时间线连续性核验

   请读取仿写试水库/ 下所有 temp_*.txt 文件（按文件名排序），检查每对相邻章节之间的场景衔接：
   - 第N章结尾场景 → 第N+1章开头场景是否连续（不能有时间跳跃或场景倒回）

   对每对相邻章节输出：
   - ✓ 连续 或 ✗ 断裂（建议：修改第N+1章开头2-3句改为...）

   全部检查完后输出最终结论。
   ")
   ```

3. **主线程收集结果**：
   - bash 字数统计已就绪
   - 从 task 1 获取简介
   - 从 task 2 获取时间线核验结果
   - 如有断裂，用 `edit` 微调对应 temp 文件开头

4. **合并为单个TXT**：
   ```bash
   # 读取简介，然后按文件名顺序读取所有 temp_*.txt，合并写入最终文件
   $intro = '（简介内容）'
   $chapters = Get-ChildItem -LiteralPath '仿写试水库' -Filter 'temp_*.txt' | Sort-Object Name
   $output = @"
   ========================================
   {新书名}
   ========================================
   作者：AI仿写
   题材：{题材}

   【简介】
   $intro

   "@
   $i = 1
   foreach ($ch in $chapters) {
     $body = Get-Content -Raw $ch.FullName
     $output += "========================================`n第${i}章`n========================================`n$body`n`n"
     $i++
   }
   Set-Content -Path '仿写试水库/试水_{新书名}.txt' -Value $output
   ```

5. **删除临时文件**：

   ```bash
   Remove-Item -LiteralPath '仿写试水库' -Filter 'temp_*.txt'
   ```

**输出格式**：
```
========================================
{新书名}
========================================
作者：AI仿写
题材：{题材}

【简介】
{简介内容}

========================================
第1章 {章名}
========================================
{正文内容}

----------------------------------------
第2章 {章名}
----------------------------------------
{正文内容}

----------------------------------------
第3章 {章名}
----------------------------------------
{正文内容}

```

**完成后显示**：
```
✅ 仿写完成！
📁 文件：仿写试水库/试水_{新书名}.txt
📊 字数：Ch1=XXX | Ch2=XXX | Ch3=XXX
📖 简介：[生成的简介]
```

## 字数要求

- **字数统计方式**：统计汉字、标点、英文字母，排除空格和换行
- 先统计源文本每章字数
- 仿写每章字数误差不超过 ±10%
- 网文标准章节约 2000 字/章
- **每章必须达到 2000 字左右**（1800-2200字），不能过短

---

## 简介要求

**字数**：150-200字

**结构**：
1. **第一句**：主角身份 + 穿越/重生设定（1句话）
2. **第二句**：核心冲突 + 危机（1句话）
3. **第三句**：男主身份 + 关系（1句话）
4. **第四句**：悬念钩子 + 反转预期（1句话）
5. **补充**：题材标签 + 卖点关键词

**示例**：
```
【简介】
一朝穿越，沈念薇成了豪门替嫁新娘。
原主贪图彩礼准备跑路，她却要面对失明的冷面大佬。
三个月，是她洗白身份的唯一窗口。
可她不知道，这个"瞎子"根本没瞎。
【偏日常/甜宠/先婚后爱/双洁】
```

---

## 写作要求（传给 agent 的约束）

1. **严格复刻节奏**：源文本此处是压抑，新文本此处也必须压抑
2. **时间线连续 ⚠️**：第N章必须从「新书第N章起始场景」开始，不能倒回第N-1章结束之前。这是最常见bug，必须重视
3. **场景切换对齐**：保持相同的场景顺序
4. **情绪拉扯同步**：相同的手法（信息差、对比等）
5. **细节完全原创**：台词、道具、环境描写全部原创
6. **禁止模板化写法**：不能用泛滥的写法
7. **禁止AI结尾**：章尾禁止升华句、总结句、哲理句

---

## ⚠️ 常见Bug与修复实录

| Bug | 原因 | 修复 |
|-----|------|------|
| **时间线倒回**：第1章结尾在公寓，第2章开头跳回医院 | Agent只读了源文本对应章节，不知道上一章在新书里结束在哪 | Phase 1框架必须定义「新书各章起始场景」，Phase 2 prompt中嵌入该信息 |
| **字数超纲**：要求2000字，实际产出2900字 | 字数约束在prompt末尾，agent注意力衰减 | 字数目标放在prompt靠前位置（核心信息区），narrative-writer 自带字数达标流程 |
| **源/新混淆**：新书出现源文本的角色名/地名 | 框架的核心差异对照表被忽略 | 差异对照表必须包含具体替换规则（源→新），且放在prompt前半部分 |
