# story-rewrite：仿写引擎（2.0版）

## 核心定义

**合规仿写 = 跨书融合借鉴创作技法 + 100%原创的故事内核**

```
合规仿写 = （3本以上不同书籍的）爽点逻辑融合
         + 通用叙事框架借鉴
         + 节奏结构参考
         + 100%原创人设
         + 100%原创事件
         + 100%原创因果逻辑链
```

**核心本质**：仿写是"用别人已经验证过的写作方法，写自己的故事"，而不是"用别人的故事，换自己的文字"。

---

## 可借鉴 vs 绝对不能照搬

| 层级 | 可以借鉴（思想/技法层） | 绝对不能照搬（表达/内容层） |
|------|------------------------|---------------------------|
| **故事类型** | 都市逆袭、玄幻修仙、悬疑探案等通用题材 | 原书独有的题材组合 |
| **爽点逻辑** | 打脸、逆袭、装逼等通用爽点公式 | 原书独有的爽点实现方式 |
| **叙事框架** | 三幕剧、英雄之旅、黄金三章等通用理论 | 原书独有的剧情推进顺序 |
| **节奏结构** | 每3章一个小爽点、每章结尾留钩子等通用节奏 | 原书的章节级节奏节点 |
| **人物** | 废柴主角、高冷女主等通用人物类型 | 有辨识度的人设细节、人物关系、口头禅 |
| **事件** | 拍卖会、学院大比等通用场景 | 原书具体的事件内容、参与人物、事件结果 |
| **因果逻辑** | "冲突→解决→新冲突"的通用故事逻辑 | 原书完整的因果链条 |

---

## 平台查重判定标准（满足任意一个即被标记）

1. **剧情节点相似度**：连续3个以上剧情节点完全相同，且因果关系一致
2. **人物关系相似度**：核心人物的数量、功能、互动模式完全相同
3. **核心创意相似度**：原书独有的核心设定被直接照搬

**最终判断标准：**
- ❌ 洗稿：把你的书给没看过原书的人看，他说"这和XX书一模一样"
- ✅ 合规：把你的书给没看过原书的人看，他说"看得真爽，和XX书有点像，但又不一样"

---

## 洗稿红线（绝对不能碰）

1. ❌ 不能照搬核心剧情线
2. ❌ 不能照搬人物关系
3. ❌ 不能照搬反转套路
4. ❌ 不能照搬场景设计（具体流程、规则）
5. ❌ 不能只仿写一本书（至少参考3-5本）

## Phase 1：初始化

### 1.0 拆章 + 统计指纹

```bash
python source_chapter_splitter.py split <源文.txt> novel-download-authors/{作者名}/{书名}/
python style_analyzer.py <源文.txt> --name "源文全书"
```

### 1.1 新书概念 → `{书名}/设定/新书概念.md`

必含项：
- 书名、类型、核心卖点
- 人物设定（主角+配角）
- **NPC命名映射表**（⚠️ 必填）
- 故事弧线、关键转折点、差异化

### 1.2 世界观 → `{书名}/设定/story_bible.md`

### 1.3 真相文件 → `{书名}/真相文件/`

```
current_state.md        # 当前状态
pending_hooks.md        # 伏笔池
chapter_summaries.md    # 章节摘要
character_matrix.md     # 角色关系矩阵
emotional_arcs.md       # 情绪弧线
```

## Phase 2：写作（每区间10章循环）

### 入口

```
Step 0：检查进度 → ls {书名}/正文/
Step 1：验证文件 → python scripts/validate_files.py {书名} {章号}
Step 2：执行2.3写章
Step 3：执行2.4去AI
Step 4：执行2.5更新真相文件
```

### 2.1 分析（缓存存在则跳过）

```bash
python source_chapter_splitter.py extract novel-download-authors/{作者名}/{书名}/ <start> <end> <{书名}/源文_{x-y}.txt>
python style_analyzer.py {书名}/源文_{x-y}.txt
```

### 2.2 章纲 → `{书名}/大纲/章纲_{x-y}.md`

**⚠️ 章纲必须通过"因果逻辑链测试"：**

```markdown
## 第N章 [章名]
- 源文对应：源文第M章 [源文章名]
- 源文事件：[源文发生了什么]（⚠️ 只作参考，不能照搬）
- 新书事件：[与源文完全不同的事件]（⚠️ 必须原创，不能只换名字）
- 因果逻辑：[为什么发生→如何发展→结果是什么]（⚠️ 必须与源文不同）
- 情绪弧线：[情绪流动]
- 钩子：[章末钩子]
```

**自检：**
- 删掉所有名字/地点/道具，只保留事件逻辑
- 如果能套用5部以上热门小说 → 立刻重写因果链

### 2.3 写章（3步强制管线）

**Step 1：验证文件（脚本保证一致性）**
```bash
python scripts/validate_files.py {书名} {章号}
```

**Step 2：并行启动所有章节**
```
⚠️ 禁止主agent直接写正文！必须用Task工具！
⚠️ 写10章时，必须在一个消息里同时启动10个Task！

每个Task的prompt：
"""
你是一个专业的网文写手。请写出《{书名}》第N章正文。

【章纲】请读取：{书名}/大纲/章纲_{x-y}.md 中第N章
【文风指南】请读取：novel-download-authors/{作者名}/{书名}/蒸馏_{x-y}.md
【真相文件】请读取：{书名}/真相文件/ 下所有 .md 文件

【合规仿写规则】
- 可以借鉴：爽点公式、宏观框架、节奏结构
- 必须原创：人设细节、具体事件、因果逻辑链
- 判断标准：读者能猜出"这和XX书一模一样"→失败；只觉得"有类似爽感"→成功

【输出】保存到：{书名}/正文/第N章.txt
"""
```

**Step 3：验证（脚本保证一致性）**
```bash
python scripts/validate_chapter.py {书名}/正文/第N章.txt "源文原名1|源文原名2"
python scripts/check_rewrite_quality.py {书名}/正文/第N章.txt {源文章节路径}
```

### 2.4 去AI（每10章）

用 `prompts/de-ai-system.md` 检测→改写→重检测循环。

### 2.5 更新真相文件（脚本保证一致性）

```bash
python scripts/update_truth_files.py {书名} {章号} "{章名}" "{摘要}" "{事件1}|{事件2}|{事件3}"
```


### 3.2 导出

```bash
cat {书名}/正文/*.txt > {书名}/{书名}.txt
```

## 仿写风险自测表（写完必查）

写完任何一段内容后，对照这个表检查：

1. 我是不是只参考了一本书？
2. 我的主角是不是和某本爆款书的主角性格、背景、目标几乎一样？
3. 我的剧情是不是可以用"把原书的人物名字换成我的"来描述？
4. 我的剧情因果链是不是和原书完全一样？
5. 连续3章的剧情节点是不是和原书完全对应？
6. 有没有读者看完后说"这和XX书一模一样"？

**只要有任何一个"是"，就立刻修改。**
