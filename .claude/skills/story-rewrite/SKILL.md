# story-rewrite：仿写流程引擎

**职责：调度 Agent、管理状态、校验质量。不包含任何写作规则。**

---

## 流程总览

```
Phase 1（主线程）    读源文本 → 提取框架 + 设定锁
Phase 2（全书写作）  章纲生成 → M×K 正文写作 → 循环至完本
Phase 3（收尾）      去AI + 一致性检查 + 字数校验
```

**默认直接写全书，不试水。** 试水用 `/story-rewrite-preview`。

---

## 上下文管理

### 核心原则

1. **分层架构**：主线程调度+校验，子 agent 单一任务
2. **状态管理**：关键信息存文件，agent 通过 read 读取
3. **检查点机制**：关键步骤设检查点，发现问题立即修复
4. **摘要机制**：每步结束生成摘要，下一步用摘要

### 分层架构

```
主线程（调度+校验）
├── story-architect（章纲生成）
├── narrative-writer（正文写作）
└── story-review（一致性审查）
```

**主线程职责**：读源文本、提取框架、生成新书方案、创建目录、调度 agent、校验输出、管理状态

**子 agent 职责**：单一任务、read 读取信息、直接写文件不返回正文

### 状态管理

| 文件 | 内容 | 读取方式 |
|------|------|---------|
| `{书名}/设定/设定锁.md` | 核心地名、关键物件、人物关系、背景事件 | agent read |
| `{书名}/大纲/章纲_全书.md` | 全书章纲 | agent read |
| `{书名}/追踪/上下文.md` | 当前写作上下文 | agent read |
| `{书名}/追踪/伏笔.md` | 伏笔状态 | agent read |
| `{书名}/追踪/时间线.md` | 时间线 | agent read |
| `{书名}/追踪/角色状态.md` | 角色状态 | agent read |

### 检查点

| 检查点 | 检查内容 | 不通过时 |
|--------|---------|---------|
| 章纲生成后 | 章数完整、格式正确 | spawn 补写 agent |
| 正文写完后 | 字数达标、文件存在 | 标记补写队列 |
| 每3批后 | 一致性审查 | 自动修正 |
| 全书完成后 | 去AI + 禁用词扫描 | 自动执行 |

### 摘要机制

**章纲生成后**：写入 `{书名}/大纲/章纲摘要.md`
```
章纲摘要：
- 总章数：{N}章
- 卷结构：{X}卷
- 关键转折点：第{X}章（事件）...
- 爽点分布：每{X}章一个小爽点
```

**正文写完后**：写入 `{书名}/追踪/上下文.md`
```
正文摘要（第{start}-{end}章）：
- 已完成：{N}章
- 角色状态：{角色名}（{状态}）...
- 伏笔状态：{伏笔}（{状态}）...
- 字数统计：合格{X}章、偏长{Y}章、偏短{Z}章
```

## Style 加载

`story-rewrite` 通过 `--style` 参数选配写作风格。风格由 `story-style` 管理。

### 用法

```
/story-rewrite --style=wenqi       # 用闻栖风格
/story-rewrite --style=wenqi --parallel=8 --per-agent=3
/story-rewrite                      # 不加--style则询问
```

### 加载原则

**prompt 只放路径，不注入内容。agent 通过 read 工具读取 SKILL.md。**

1. 获取 style 路径 → `.claude/skills/story-style/{name}/SKILL.md`
2. 注入 prompt → 只放 `{style_skill_path}`
3. agent 自主读取 → 读取风格特征、写作原则、钩子类型、质量检查清单

### 必须指定风格

不加 `--style` 时询问用户：
```
请选择写作风格：
1. 闻栖（--style=wenqi）
2. 其他（请指定 --style=名称）

可用风格列表：/story-style
```

### 当前可用风格

| 风格 | --style | 描述 |
|------|---------|------|
| 闻栖 | `--style=wenqi` | 番茄女频·穿书/甜宠/反套路 |

## Phase 1：框架提取

### 源文本要求

| 项目 | 要求 |
|------|------|
| 格式 | txt 或 md |
| 编码 | UTF-8 |
| 章节分隔 | 每章以 `第X章` 开头 |
| 最低章数 | ≥ 3 章 |

### 提取内容

主线程读源文本，提取：

1. **源文本分析**：
   - 前3章：开篇方式、破冰方式、人设标签
   - 中间转折点：读第1/4、1/2、3/4处各1章
   - 最后1章：结局走向、伏笔回收方式
   - 全书章数、卷结构、每卷核心事件
   - 提取3-5段语感样本 + 1章完整样本

2. **自动生成新书方案**（AI全自动，不询问用户）：
   - 书名：结合源书结构+风格特征
   - 女主角名、男主角名
   - 核心设定：穿书/重生/先婚后爱等
   - 章数：对齐源书章数
   - 题材：风格插件自动锁定

3. **设定锁**（所有agent共享的硬设定）：
   - 核心地名（一级锁定）
   - 关键物件（一级锁定）
   - 人物关系锁（一级锁定）
   - 背景事件锁（一级锁定）
   - 时间线锁（一级锁定）
   - 动态细节（二级锁定，写作过程中补充）

4. **语感特征**：3-5个核心节奏特征

### 反调色盘自检

新书的"核心设定+核心冲突+角色原型"和源文本对比，外人看了觉得"这不就是同一本书吗"→ 必须重写框架。

| 维度 | 红线 |
|------|------|
| 核心设定 | 机制不能相同 |
| 核心冲突 | 驱动矛盾不能同类 |
| 角色原型 | 身份/处境/动机不能同类 |
| 开篇方式 | 第1章核心事件不能同类 |
| 破冰方式 | 第一次互动方式不能同类 |

## Phase 2：全书写作

Phase 1 完成后直接开始，不试水，不等用户确认。

### 断点续写

用户说"继续写"且 `{书名}/正文/` 已存在时：
1. 扫描正文目录，找到最大章节编号 N
2. 读取章纲，确认总章数
3. 检查追踪文件（存在→读取；缺失→从最近3章重建）
4. 从第 N+1 章续写

### Step 1：创建项目 + 章纲生成

```
⚠️ 章纲必须由 story-architect subagent 生成，主线程禁止手写章纲

1. 创建项目目录：
   {书名}/正文/
   {书名}/设定/（写入设定锁.md）
   {书名}/大纲/
   {书名}/追踪/（上下文.md / 伏笔.md / 时间线.md / 角色状态.md）

2. 章纲生成（强制多 agent 并行）：
   - ⚠️ 章数硬性对齐：新书章数 = 源文本总章数
   - ⚠️ 必须使用 story-architect subagent
   - ⚠️ 多 agent 并行生成（每批最多50章）：
     - ≤50 章 → spawn 1 个
     - 51-100 章 → spawn 2 个并行
     - 101-150 章 → spawn 3 个并行
     - 151-200 章 → spawn 4 个并行
   - ⚠️ 每个 agent 必须知道：总章数、结局章范围、本批次是否是结局

3. 章纲完整性校验：
   - 用grep统计章纲文件中的章数，与源文本总章数对比
   - 发现断层 → 自动 spawn 补写 agent

4. 生成章纲摘要：写入 {书名}/大纲/章纲摘要.md

5. 并行度探测：spawn 5 个 → 成功则 +2 → 遇 429 则回退

6. 容错：全部失败 → 降级 K=1, M=3 重试；仍失败 → 暂停报告用户
```

### Step 2：写作循环（M 个 agent，每 agent 写 K 章）

**核心：每 agent 写 2-3 章**

```
Step 1：M 个 narrative-writer 并行 spawn
        - 每 agent 负责 K 章（K=2 或 3）
        - prompt 注入：设定锁 + K 章章纲 + style 路径
        - agent 直接写文件，不返回正文

        并行度自适应：
        - 推荐：M=5, K=3 → 每批 15 章
        - 质量优先：M=5, K=2 → 每批 10 章
        - 保守：M=3, K=2 → 每批 6 章
        - 激进：M=10, K=3 → 每批 30 章
        - K 上限 = 3，M 上限 = 10

Step 2：主线程字数校验（每批完成）
        - 目标：2200字
        - 范围：1800-2600字
        - 硬上限：3000字（超过截断）
        - 硬下限：1600字（低于补写）

Step 3：生成正文摘要（每批完成）
        - 写入 {书名}/追踪/上下文.md

Step 4（每 2 批执行一次）：
        - story-review 一致性审查
        - 更新追踪文件（伏笔.md / 时间线.md / 角色状态.md）
        - 字数异常章节批量处理

Step 5：检查剩余章纲，不足 2 批时 → story-architect 补充

Step 6：字数不足章节处理
        - <1600字：立即 spawn 补写
        - 1600-1800字：加入下一批补写队列
```

### 速度估算（200 章）

| 配置 | 每批产出 | 批次数 | 每批耗时 | 总耗时 |
|------|---------|--------|---------|--------|
| M=5, K=3（推荐） | 15 章 | 14 批 | 8-12 min | **2-3 小时** |
| M=5, K=2 | 10 章 | 20 批 | 6-10 min | **2.5-3.5 小时** |
| M=3, K=2（保守） | 6 章 | 34 批 | 5-8 min | **3-4.5 小时** |
| M=10, K=3（激进） | 30 章 | 7 批 | 12-18 min | **1.5-2 小时** |

### 质量衰减防护

每 5 批（约 50 章）自动注入"风格变异指令"：强制换句式结构、换节奏模式，防止模式固化。

---

## Phase 3：全书写完后

1. **全书去AI**：句长随机化、段落打散、句首分散、动作-反应解耦
   - 200+ 章建议 spawn 3-5 个 agent 并行处理
2. **一致性终检**：角色属性/时间线/伏笔状态
3. **字数总校验**：每章 >3000 自动截断

## Agent 调用规范

### story-architect（章纲生成）

**职责**：生成章纲，每章包含5个字段：章名、章首钩子类型、核心事件、情绪基调、章尾钩子类型

**prompt 构造原则**：
- 只放任务信息 + 文件路径
- 钩子类型、节奏模板等全部从 style SKILL.md 读取
- 不在 prompt 里写死任何写作规则

**prompt 模板**：
```
你负责为仿写小说生成章纲（第{start}章 ~ 第{end}章）。

## ⚠️ 硬性要求
1. 必须生成完整的第{start}章到第{end}章章纲，不能遗漏
2. 每章必须包含5个字段：章名、章首钩子类型、核心事件、情绪基调、章尾钩子类型
3. 核心事件必须写清"谁做了什么"，不能是"关系升温"这种空话
4. 钩子类型必须从风格 SKILL.md 读取，不要用通用模板

## ⚠️ 章节定位
- 总章数：{total_chapters}章
- 本批次：第{start}章 ~ 第{end}章
- 结局章：第{ending_start}章 ~ 第{total_chapters}章
- 本批次最后一章：{是否是结局？不是则必须留悬念}

## ⚠️ 写作前检查清单（必须先读取）
1. 读取风格 SKILL.md：{style_skill_path}
2. 读取设定锁：{书名}/设定/设定锁.md
3. 读取章纲摘要：{书名}/大纲/章纲摘要.md（如有）

## 输出格式
每章一行，用 | 分隔：
第{N}章 | 章名 | 章首钩子类型 | 核心事件 | 情绪基调 | 章尾钩子类型

将章纲写入：{书名}/大纲/章纲_{start}-{end}.md
```

### narrative-writer（正文写作）

**职责**：写正文，每章 2200 字左右

**prompt 构造原则**：
- 只放任务信息 + 文件路径
- 风格特征、写作原则、反模式全部从 style SKILL.md 读取
- 不在 prompt 里写死任何写作规则

**prompt 模板**：
```
# ⚠️ 叙事视角：必须第三人称。

## ⚠️ 角色锚点（写错角色名=致命错误）
{从设定锁提取的角色名表}

## ⚠️ 写作前检查清单（每章写之前必须核对）
1. 读取风格 SKILL.md：{style_skill_path} → 了解风格特征、写作原则、反模式、章纲模板
2. 读取设定锁：{书名}/设定/设定锁.md → 核对角色名、地名、人物关系
3. 读取章纲：{书名}/大纲/章纲_全书.md → 找第{start}-{end}章
4. 读取上章正文：{书名}/正文/第{start-1}章_{章名}.md

## 本次任务
写 {K} 章（第{start}章 ~ 第{end}章）

每章独立成篇，但章与章之间要有连贯性：
- 第2章开头自然承接第1章结尾
- 同一 agent 内后续章节不需要注入前章摘要
- 先一口气写完，写完后用 Python 字符统计验证，< 目标90%则自然扩充

## 格式
第一行：`第{N}章 {章名}`
用 write 工具直接写入文件。

## 参考文件（按需读取）
- {style_skill_path}（风格特征、写作原则、反模式）← 必须读取
- {书名}/设定/设定锁.md（角色名、地名、人物关系）
- {书名}/大纲/章纲_全书.md（章纲）
```

### story-review（一致性审查）

**职责**：检查一致性（角色属性、时间线、伏笔状态），不负责修改

## 字数自动修复

**统一用 PowerShell**（Windows 环境）。

```powershell
# 设置编码
$OutputEncoding = [System.Text.Encoding]::UTF8
[Console]::OutputEncoding = [System.Text.Encoding]::UTF8

# 批量校验
foreach ($file in $batchFiles) {
    $content = Get-Content $file -Raw -Encoding UTF8
    $chars = ($content -replace '[\s]','').Length
    if ($chars -gt 3000) {
        # 截断到 2950 字（按段落边界）
        $paragraphs = $content -split '(\r?\n\r?\n|\r?\n)'
        $result = ""
        foreach ($p in $paragraphs) {
            $test = $result + $p
            if (($test -replace '[\s]','').Length -gt 2950) { break }
            $result = $test
        }
        $result | Set-Content $file.FullName -Encoding UTF8
    } elseif ($chars -lt 1600) {
        Write-Output "TOO_SHORT: $($file.Name) -> $chars chars"
    }
}
```

## 章纲质量标准

1. **章数完整性**：章纲必须覆盖第1章到第N章，不能遗漏
2. **格式完整性**：每章5个字段，缺一不可
3. **钩子多样性**：同批内同类型钩子不超过2个
4. **情节点具体性**：必须写清"谁做了什么"
5. **情绪起伏**：每5章内至少1个转折
6. **动机合理性**：偶遇/绑定事件必须自洽

不通过 → 重写指令模板：
```
第 {X}-{Y} 章章纲不合格：
- [ ] 章数缺失
- [ ] 格式不完整
- [ ] 钩子重复
- [ ] 情节点空泛
- [ ] 动机不合理
请重写，保留章编号不变。
```
