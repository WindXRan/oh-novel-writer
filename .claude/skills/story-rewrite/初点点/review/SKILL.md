## 审稿流程

```
1. 跑脚本 → 自动检测质量阈值
2. 对照红线 → 逐条诊断问题
3. 对照处方 → 给出修改方案
4. 输出报告 → 通过 / 需修改 / 打回
```

| 判定 | 条件 | 动作 |
|------|------|------|
| ✅ 通过 | 无 🔴，🟡 ≤ 3 | 直接发布 |
| 🟡 需修改 | 有 🔴，可局部修 | 给处方，改后复审 |
| 🔴 打回 | 🔴 ≥ 5 或核心结构问题 | 给方向，重写 |

## 修改处方

> 详细引用见 `../references/edit-prescriptions.md`

| # | 问题类型 | 处方摘要 |
|---|---------|---------|
| 1 | 心理独白拖沓 | 删除「她心想」标签，转为自言自语对话体或「她想」+1句话 |
| 2 | 对话标签冗余 | 删「说道/轻声道」，改直接对话或动作+对话 |
| 3 | 副词堆砌 | 删「微微/轻轻/淡淡」，改具体动作。「微微蹙眉」→「蹙了蹙眉」 |
| 4 | 奇幻外貌 | 删「紫眸/冰眸」，改「眉目英俊、高鼻薄唇」 |
| 5 | 情绪直白 | 删「她很痛苦」，改动作。「攥了攥手指」「咬着唇」 |
| 6 | 章名无信息 | 改4-6字事件/情绪短语。「第一章」→「求娶宁祯」 |
| 7 | 节奏拖沓 | 压缩描写，加核心事件，用对话替代旁白 |
| 8 | 女主被动 | 删哭诉，改冷静反击。「跪地哀求」→「站起来，声音平静」 |
| 9 | 误会拖剧情 | 1-2章内安排对话解开误会 |
| 10 | 回忆章节 | 碎片化穿插当前叙事，对话交代往事，≤300字 |

## 爽点密度审查

> 详细公式见 `../references/candidates/satisfaction-points.md`

| 检查项 | 标准 | 🟡 | 🔴 |
|--------|------|-----|-----|
| 爽点间隔 | 4-5章/个 | >7章 | >10章 |
| 开篇爽点 | 前3章至少1个 | 前5章才有 | 前10章没有 |
| 类型多样性 | ≥2种类型 | 只有1种 | 只有1种且重复 |
| 爽点收尾 | 金句/动作收尾 | 旁白收尾 | 无收尾直接过 |

**爽点公式速查**：
- 打脸：委屈铺垫 → 对峙升级 → 暴力压制 → 金句收尾
- 逆袭：弱势铺垫 → 危机触发 → 智力/武力反杀 → 旁观震惊
- 发糖：冷淡铺垫 → 氛围催化 → 小动作突破 → 迂回表白

## 多agent审改流程（长篇专用）

> 章数 ≥ 30 时自动启用，章数 < 30 走单agent审稿

### Phase 0：自动分卷

| 步骤 | 动作 | 说明 |
|------|------|------|
| 1 | 读取章纲或扫描正文目录 | 获取总章数和章节标题 |
| 2 | 按每卷 20-40 章分卷 | 优先按章纲的卷结构分，无章纲则等分 |
| 3 | 标记边界章节 | 每卷前后各 2 章为边界，供 Phase 2 交叉复审 |

**分卷规则**：
- 有章纲：按章纲的卷结构（如「第一卷 1-40」「第二卷 41-80」）
- 无章纲：按章数等分（如 157 章 → 4 卷：1-40、41-80、81-120、121-157）
- 边界章节：每卷的最后 2 章 + 下一卷的前 2 章

### Phase 1：并行审稿

**并行** spawn N 个 Task sub-agents（每卷一个）：

每个 agent 负责一卷的：

```
你的任务：审稿《{书名}》第{起始章}-{结束章}章。

读取数据：
- 读取 {正文目录}/第{X}章.txt 到 第{Y}章.txt
- 读取 {设定目录}/设定锁.md（人设审查用）
- 读取 {章纲目录}/章纲.md（如存在，节奏审查用）

审稿步骤：
1. 对照红线逐章检查（RL-01 到 RL-12）
2. 统计质量阈值（对话占比、副词密度等）
3. 审查爽点分布（间隔、类型、收尾）
4. 审查人设（女主主体性、男主区分度、配角弧光）
5. 审查节奏（短周期、冲突密度、章尾钩子）

输出要求：
- 写入 {输出目录}/审稿报告_卷{N}.md
- 格式：逐章问题清单 + 卷级汇总
- 每个问题附：章节、位置、问题描述、处方建议
- 卷级汇总：通过/需修改/打回 + 评分
```

**输出格式**（每卷）：

```markdown
## 审稿报告 · 卷{N}（第{X}-{Y}章）

### 逐章问题清单

| 章 | 编号 | 问题 | 处方 |
|---|------|------|------|
| 3 | RL-08 | 无章名 | 添加「第3章 交易达成」 |
| 7 | RL-01 | 副词堆叠（6次） | 删「微微/轻轻」，改具体动作 |

### 卷级汇总

| 指标 | 数值 | 判定 |
|------|------|------|
| 🔴 问题数 | 2 | |
| 🟡 问题数 | 3 | |
| 判定 | 🟡 需修改 | |

### 卷级评分

| 维度 | 得分 | 说明 |
|------|------|------|
| 故事 | 90 | 结构完整，节奏紧凑 |
| 人设 | 88 | 女主有主体性 |
| 文笔 | 85 | 对话占比适中 |
| 商业 | 87 | 钩子充足 |
| 总分 | 88 | |
```

### Phase 2：交叉复审

**并行** spawn 2 个 Task sub-agents（首尾卷各一个）：

```
你的任务：交叉复审《{书名}》相邻卷的边界章节。

读取数据：
- 读取 卷{N}的最后2章 + 卷{N+1}的前2章

检查内容：
1. 节奏断裂：两卷之间是否有情绪/节奏断层
2. 人设不一致：同一角色在两卷中是否有性格差异
3. 伏笔断裂：上卷埋的伏笔在下卷是否接上
4. 信息重复：两卷边界是否有重复交代的信息

输出要求：
- 写入 {输出目录}/审稿报告_边界_{N}-{N+1}.md
- 格式：问题清单 + 建议
```

### Phase 3：汇总报告

主线程完成：

| 步骤 | 动作 |
|------|------|
| 1 | 读取所有 `审稿报告_卷*.md` 和 `审稿报告_边界_*.md` |
| 2 | 合并逐章问题清单（去重） |
| 3 | 生成总评（判定/评分/修改清单） |
| 4 | 写入 `{书名}/追踪/审稿报告.md` |

**总评格式**：

```markdown
## 审稿总评 · {书名}

### 判定：✅ 通过 / 🟡 需修改 / 🔴 打回

### 各卷评分

| 卷 | 章节 | 得分 | 主要问题 |
|---|------|------|----------|
| 1 | 1-40 | 90 | 2个🔴 |
| 2 | 41-80 | 88 | 1个🔴 |
| 3 | 81-120 | 92 | 无🔴 |
| 4 | 121-157 | 85 | 3个🔴 |

### 全书问题汇总

| 编号 | 问题 | 出现次数 | 涉及章节 |
|------|------|----------|----------|
| RL-08 | 无章名 | 7 | 4,5,6,65,75,125,135 |
| RL-01 | 副词堆叠 | 2 | 10,30 |

### 修改清单（按优先级）

| 优先级 | 章节 | 问题 | 处方 |
|--------|------|------|------|
| 🔴 | 4,5,6,65 | 无章名 | 补章名 |
| 🟡 | 157 | 节奏拖沓 | 压缩重复描写 |

### 全书评分

| 维度 | 权重 | 得分 | 说明 |
|------|------|------|------|
| 故事 | 35% | 89 | |
| 人设 | 30% | 90 | |
| 文笔 | 20% | 87 | |
| 商业 | 15% | 88 | |
| **总分** | 100% | **89** | |
```

### 触发方式

| 章数 | 审稿方式 | 说明 |
|------|----------|------|
| < 30 | 单agent审稿 | 直接读取全部章节，一次性审完 |
| 30-100 | 多agent审稿（2-3卷） | 每卷30-40章 |
| > 100 | 多agent审稿（4-6卷） | 每卷20-40章 |

### 示例：157章审稿流程

```
Phase 0：分卷
├── 卷1：第1-40章（绝境求生）
├── 卷2：第41-80章（立足北城）
├── 卷3：第81-120章（夺回家产）
└── 卷4：第121-157章（大仇得报）
边界：38-42、78-82、118-122

Phase 1：并行审稿（4个agent同时跑）
├── Agent A → 审卷1（1-40）→ 审稿报告_卷1.md
├── Agent B → 审卷2（41-80）→ 审稿报告_卷2.md
├── Agent C → 审卷3（81-120）→ 审稿报告_卷3.md
└── Agent D → 审卷4（121-157）→ 审稿报告_卷4.md

Phase 2：交叉复审（2个agent同时跑）
├── Agent E → 审边界38-42、78-82 → 审稿报告_边界_1-2.md
└── Agent F → 审边界118-122 → 审稿报告_边界_3-4.md

Phase 3：汇总
└── 主线程 → 合并6份报告 → 审稿报告.md
```
