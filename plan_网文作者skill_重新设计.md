# 网文作者 Skill 重新设计

> **核心问题**：怎么样才能够像真实的大佬写的一样？
> **答案**：提取他的决策框架，不是他的表面特征。

---

## 一、当前设计的根本问题

### 错误：提取「是什么」

```
当前做法：
- 句式偏好：短句为主
- 钩子类型：悬念型占60%
- 对话占比：35%
- 爽点密度：每5章一个
```

这是**描述**，不是**方法论**。知道「短句为主」有什么用？你不知道**什么时候**该用短句。

### 正确：提取「为什么」

```
应该提取：
- 什么时候用短句？→ 情绪紧张时、动作场景时、揭示关键信息时
- 为什么用短句？→ 短句压缩读者呼吸，制造紧迫感
- 什么时候用长句？→ 铺垫氛围时、描写心理时、制造舒缓感时
- 为什么用长句？→ 长句让读者放慢阅读速度，沉浸在情绪里
```

**大佬的写作是一个个决策组成的。我们要提取的是决策框架。**

## 二、真正需要提取什么

### Layer 1：心智模型（Mental Models）

大佬看世界的镜片。他们怎么理解「好故事」。

| 维度 | 问题 | 示例 |
|------|------|------|
| 故事观 | 什么是好故事？ | 「好故事=读者想看下一章」 |
| 角色观 | 什么是好角色？ | 「好角色=有缺陷但有魅力」 |
| 冲突观 | 什么是好冲突？ | 「好冲突=两难选择，没有完美答案」 |
| 爽感观 | 什么是爽？ | 「爽=延迟满足后的释放」 |

**提取方法**：从多本书中找「反常识」的写法——大佬做的事情和普通作者不一样，这些差异点就是心智模型。

### Layer 2：决策启发式（Decision Heuristics）

大佬做判断时的快速规则。可以表述为「如果X，则Y」。

| 场景 | 决策规则 | 来源 |
|------|---------|------|
| 章尾断点 | 如果本章情绪达到高点，则在这里断章 | 从多本书的章尾分析 |
| 信息揭示 | 如果读者已经猜到真相，则提前揭示，制造新悬念 | 从反转设计分析 |
| 角色对话 | 如果角色在情绪激动时，则用短句+动作beat，不用「说道」 | 从对话模式分析 |
| 节奏控制 | 如果连续3章紧张，则第4章必须舒缓 | 从节奏变化分析 |

**提取方法**：从多本书中找「模式」——什么条件下做什么事，这些条件-动作对就是决策规则。

### Layer 3：节奏直觉（Rhythm Intuition）

大佬对读者心理的把控。不是「每5章一个爽点」，是「为什么在这里放爽点」。

| 维度 | 问题 | 示例 |
|------|------|------|
| 读者预期 | 读者现在期待什么？ | 「读者等了3章，这章必须给糖」 |
| 情绪曲线 | 现在应该上升还是下降？ | 「刚虐完，现在要甜，但不能太甜」 |
| 信息差 | 读者知道什么不知道什么？ | 「读者知道男主身份，女主不知道，制造悬念」 |
| 承诺兑现 | 前面埋的伏笔什么时候收？ | 「第10章埋的伏笔，第25章必须收」 |

**提取方法**：从多本书的「转折点」分析——大佬在哪里改变节奏，为什么。

### Layer 4：表达DNA（Expression DNA）

这才是「怎么写」。但不是「短句为主」这种表面特征，是更深层的东西。

| 维度 | 问题 | 示例 |
|------|------|------|
| 信息密度 | 每段传递几个信息点？ | 「一段只传递一个信息，但这个信息必须有冲击力」 |
| 感官选择 | 优先用哪个感官？ | 「优先用视觉和触觉，少用味觉」 |
| 情绪表达 | 直接命名还是行为展示？ | 「永远不写'她很伤心'，写'她的眼眶红了'」 |
| 对话原则 | 对话的目的是什么？ | 「对话必须推进剧情或揭示性格，否则删掉」 |

**提取方法**：从多本书中找「高频模式」——大佬反复做的事情。

### Layer 5：反模式（Anti-Patterns）

大佬绝对不会做的事情。这是最能识别「AI味」的部分。

| 反模式 | 说明 | 来源 |
|--------|------|------|
| 不做什么 | 大佬避免的写法 | 从多本书中找「缺失」——大佬从不做的事 |
| 不说什么 | 大佬避免的词/句式 | 从词汇分析中找「禁忌」 |
| 不怎么写 | 大佬避免的结构 | 从结构分析中找「回避」 |

**提取方法**：从多本书中找「缺失」——大佬从不做的事。

## 三、提取流程（重新设计）

### Phase 1：深度分析（不是统计，是理解）

**目标**：理解作者的决策逻辑

**方法**：
1. 选3-5个「关键场景」（开篇、转折、高潮、结局）
2. 对每个场景做「决策分析」：
   - 作者在这里做了什么选择？
   - 为什么做这个选择？
   - 如果换一种选择会怎样？
3. 提取「决策规则」

**输出**：`references/decision-framework.md`

### Phase 2：模式识别（不是统计频率，是识别决策模式）

**目标**：从多本书中识别「如果X，则Y」的决策规则

**方法**：
1. 找「转折点」——情绪/节奏发生变化的地方
2. 分析「触发条件」——什么导致了变化
3. 提取「决策规则」——如果X，则Y

**输出**：`references/decision-heuristics.md`

### Phase 3：心智模型提取（不是描述，是理解）

**目标**：理解作者的「故事观」「角色观」「冲突观」

**方法**：
1. 找「反常识」——作者做的事情和普通作者不一样
2. 分析「为什么」——为什么这样做
3. 提取「心智模型」——作者看世界的镜片

**输出**：`references/mental-models.md`

### Layer 4：表达DNA（不是统计句式，是理解信息密度）

**目标**：理解作者的「信息传递方式」

**方法**：
1. 分析「信息密度」——每段传递几个信息点
2. 分析「感官选择」——优先用哪个感官
3. 分析「情绪表达」——直接命名还是行为展示
4. 分析「对话原则」——对话的目的是什么

**输出**：`references/expression-dna.md`

### Layer 5：反模式（不是禁忌词，是决策禁区）

**目标**：识别作者「绝对不会做的事」

**方法**：
1. 找「缺失」——作者从不做的事
2. 分析「为什么」——为什么不做
3. 提取「反模式」——决策禁区

**输出**：`references/anti-patterns.md`

## 五、与下游的集成（重新设计）

### 当前问题

现在的 rewrite 只是「按模板写」，没有「按决策框架思考」。

### 新的集成方式

```
novel-distill → story-style/{作者名}/SKILL.md
                    │
                    ├─→ story-rewrite 读取「决策启发式」
                    │       └─→ narrative-writer 在写每章时：
                    │           1. 读取当前场景的决策规则
                    │           2. 问自己：「如果{作者}在这里，他会怎么做？」
                    │           3. 按决策规则写作，而不是按模板写作
                    │
                    └─→ story-rewrite 读取「心智模型」
                            └─→ story-architect 在设计章纲时：
                                1. 读取作者的「故事观」「角色观」「冲突观」
                                2. 按心智模型设计，而不是按通用模板设计
```

### 关键改变

**从「按模板写」变成「按决策框架思考」**

```
当前：
- 章首钩子：悬念型（从模板选）
- 章尾钩子：反转钩（从模板选）

应该：
- 章首钩子：如果读者此时期待{X}，则用{Y}型钩子（从决策规则推导）
- 章尾钩子：如果本章情绪达到{X}，则在这里断章（从决策规则推导）
```

## 八、核心洞察

**大佬写的不是「文字」，是「决策」。**

每一个句式选择、每一个钩子设计、每一个节奏变化，背后都有一个决策逻辑。

我们要提取的不是「他写了什么」，是「他为什么这样写」。

当我们把决策框架提取出来，AI就不再是「模仿大佬的文字」，而是「像大佬一样思考」。

这才是真正「像大佬写的一样」的方法。
