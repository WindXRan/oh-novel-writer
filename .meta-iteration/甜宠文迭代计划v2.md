# 甜宠文迭代计划 v2：反抄袭专项修复

## 根因分析

当前 SKILL.md 已有反抄袭规则（L62-86 抽象化规则、L133-138 反抄袭硬约束），但 Round 1 全部违反。
**根因不是规则缺失，而是架构上无法强制执行。**

### 3 个架构漏洞

| 漏洞 | 表现 | 后果 |
|------|------|------|
| **V1: Phase 2 能读源文本** | prompt 中包含源文本路径，agent 有 Read 权限 | 「禁止读取源文本」= 废话 |
| **V2: Phase 1 提取过深** | 抽象化规则存在但 agent 仍提取了具体场景序列 | Phase 2 拿到了「半具体」的框架，直接照搬 |
| **V3: 无抄袭检测环节** | Phase 3 只检查时间线和字数，不检查抄袭 | 洗稿输出直接交付 |

## 修复方案：3 层防护

### 防护层 1：Phase 1 物理隔离源文本

**当前问题**：Phase 1 agent 读取源文本后输出框架，但框架中混入了具体情节。

**修复**：Phase 1 拆为两个子步骤——
1. **Step 1a（信息提取）**：agent 读取源文本，输出**原始分析**（包含具体情节，作为中间产物）
2. **Step 1b（抽象化）**：主线程将原始分析传给**另一个 agent**，明确要求「将以下分析抽象为叙事模式，禁止保留任何具体情节」

关键：Step 1b 的 agent **不接触源文本**，只拿到 Step 1a 的输出。双重抽象降低抄袭风险。

### 防护层 2：Phase 2 物理隔离源文本 + 降级框架

**当前问题**：Phase 2 prompt 包含源文本路径，agent 可以直接读取。

**修复**：
1. Phase 2 prompt 中**删除源文本路径**
2. Phase 1 输出的框架**进一步降级**——只保留「每章情绪弧线 + 叙事功能 + 人物关系模式」，删除所有具体信息
3. 在 prompt 开头用 **⚠️ 红字**强调：你的唯一输入是下方的抽象框架，你没有源文本

### 防护层 3：Phase 3 抄袭检测

**当前问题**：无抄袭检测。

**修复**：在 Phase 3 合并输出后，增加 Step 3.5——

```
task(subagent_type: "general", prompt: "
你是反抄袭检测专家。

任务：检测仿写是否抄袭源文本。

仿写文件：{仿写路径}
源文本路径：{源文本路径}

请读取两个文件，从以下5层对比：

1. **逐句对比**：是否有句式雷同（仅替换个别词汇）？
2. **场景序列对比**：场景切换顺序是否1:1对应？
3. **角色映射对比**：角色是否与源文本一一对应（换名不换人）？
4. **情节装置对比**：关键情节装置是否直接搬运？
5. **台词对比**：是否有台词逐字相同或仅换名词？

判定标准：
- S1（致命）：逐句搬运/台词相同/场景序列1:1 → REJECT
- S2（严重）：角色一一映射/情节装置照搬 → REJECT
- S3（一般）：个别句式相似/部分场景类似 → CONCERNS
- S4（轻微）：整体框架相似但细节原创 → PASS

输出 JSON：
{
  "verdict": "PASS/CONCERNS/REJECT",
  "s1_count": 0,
  "s2_count": 0,
  "s3_count": 0,
  "examples": ["具体抄袭示例"],
  "suggestion": "修改建议"
}

如果 verdict=REJECT，必须重新仿写。
")
```

## 修改后的 SKILL.md 流程

```
Phase 1（结构提取，拆为两步）     Phase 2（正文并行）      Phase 3（合并+检测）
┌──────────────────┐          ┌──────────────┐          ┌──────────────┐
│ Step 1a: 读源文本  │          │ 3个agent并行 │          │ 合并TXT      │
│ → 输出原始分析     │          │ ⚠️ 无源文本路径│          │ + 抄袭检测    │
│ Step 1b: 抽象化   │ ──────→  │ 只拿到抽象框架 │ ──────→  │ PASS→交付    │
│ → 输出纯抽象框架   │          │              │          │ REJECT→重写  │
└──────────────────┘          └──────────────┘          └──────────────┘
```
