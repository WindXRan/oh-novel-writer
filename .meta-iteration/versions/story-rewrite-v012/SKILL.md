# story-rewrite：结构仿写

提取源文本的深层叙事"骨架"。用全新角色、背景、情节填充。生成逻辑自洽的新书。

**核心信念：只仿"骨相"，不仿"皮相"。**

---

## 核心原则

### 原则 1：骨架先行，血肉后填

先理解源文本的叙事引擎。引擎是可迁移的，燃料（具体情节）必须是新的。

**案例**：
- ✅ 复刻"压抑3章→爽点爆发"的节奏，换成全新角色和情节
- ❌ 只换了名字，情节高度相似

### 原则 2：节奏是灵魂，细节是皮囊

读者追的是"压抑-积累-爆发"的情绪节奏。复刻节奏，替换细节。

**案例**：
- ✅ 原文第3章是爽点，仿写第3章也是爽点（节奏对齐）
- ❌ 原文第3章是爽点，仿写第3章还在铺垫（节奏错位）

### 原则 3：逻辑自洽是底线

新故事不能因套用结构而出现逻辑漏洞。

**案例**：
- ✅ 原文靠"系统限制"制造冲突，仿写靠"地域限制"制造冲突（逻辑自洽）
- ❌ 原文靠"系统限制"，仿写硬套但没有合理解释（逻辑漏洞）

### 原则 4：青出于蓝

仿写不是降级复制。保持结构优势，尝试微创新。"熟悉但更好"。

**案例**：
- ✅ 仿写版在保持节奏的同时，增加了更接地气的细节
- ❌ 仿写版只是原文的降级复制

### 原则 5：金手指设计三要素

金手指必须同时满足：
1. **数字大**：金额足够大，制造爽感（每天至少百万级别）
2. **限制简单**：一句话说清限制，读者理解成本低
3. **场景丰富**：能衍生多种消费/投资场景，支撑长篇剧情

**限制条件的核心作用是制造戏剧冲突，不是限制故事发展。**

**设计公式**：
```
金手指 = 大额资金 + 简单限制 + 丰富场景
```

**案例**：
- ✅ 每天6000万，只能在县城花（数字大、限制简单、场景丰富）
- ✅ 每天500万，只能投资产业（数字大、限制简单、场景丰富）
- ❌ 每天5000万，只能在直播间花（场景单一，无法支撑长篇）
- ❌ 每天100万，只能投资新能源（数字太小，爽感不足）

**检查清单**：
- [ ] 数字足够大（每天至少百万级别）
- [ ] 限制条件一句话能说清
- [ ] 能自然衍生冲突（触动别人利益）
- [ ] 有清晰的升级路径

### 原则 6：爽点节奏对齐

仿写的爽点节奏必须与原文完全一致：
- 原文第几章有爽点，仿写就必须第几章有爽点
- 原文爽点是什么类型，仿写就必须是什么类型
- 原文爽点释放多长时间，仿写就必须释放多长时间

**案例**：
- ✅ 原文第4章"买迈巴赫，刷卡打脸"，仿写第4章"签约付款，转账打脸"
- ❌ 原文第4章是爽点，仿写第4章还在谈判（节奏错位）

## 工作流程

### Phase 1：接收源文本

用户只需提供**源文本**（文件路径、粘贴文本、或 `拆文库/` 引用）。

**AI 自动决策**：新主角设定、背景、金手指等全部由 AI 自动选择最优方向。

**源文本处理**：
- 文件路径 → 读取文件内容
- 对话粘贴 → 直接使用
- 拆文库引用 → 读取拆解报告
- 书名+平台 → 提示用户先用 `/story-long-analyze` 拆解

### Phase 2：结构提取（内部过程）

**此阶段在 AI 内部完成，不输出给用户。**

提取以下要素：

| 要素 | 提取内容 |
|------|----------|
| 节奏地图 | 每章情绪基调、压抑段落长度、爽点频率 |
| 转折点序列 | 关键转折位置、类型、铺垫手法 |
| 场景切换模式 | 切换顺序、节奏、功能分配 |
| 情绪拉扯技巧 | 延迟满足、危机制造、反转手法 |
| 描写手法分布 | 动作/心理/对话/叙述比例 |
| 金手指分析 | 核心机制、限制条件、升级路径 |
| 角色塑造分析 | 主角性格、配角功能、关系图谱 |

### Phase 3：结构映射与确认

将提取的结构映射为可执行的仿写框架。输出**精简版**给用户确认：

```
## 仿写框架

### 新故事设定
- 书名：[AI生成]
- 主角：[姓名、年龄、性格关键词]
- 金手指：[一句话描述]
- 核心冲突：[一句话描述]

### 节奏对齐
| 章节 | 源结构功能 | 新剧情方向 |
|------|------------|------------|
| 第1章 | [功能] | [新情节] |
| 第2章 | [功能] | [新情节] |
| ... | ... | ... |

### 核心差异
- 主角：[源] → [新]
- 背景：[源] → [新]
- 金手指：[源] → [新]

确认无误？有需要调整的地方吗？
```

> 详细的角色设计、金手指分析等参考 `references/` 目录下的文档。

### Phase 4：试水仿写（前3章）

**默认只写前3章**。快速试水验证结构可行性。

#### 字数要求

- 先统计源文本每章字数（去除空行、缩进、标点后的纯文字）
- 仿写每章字数误差不超过 ±10%
- 网文标准章节约 2000 字/章
- **每章必须达到 2000 字左右**（1800-2200字），不能过短
- **禁止章节字数不均衡**：每章字数差异不超过 20%
- **字数不足时的扩充方法**：
  - 增加细节描写（环境、动作、心理）
  - 增加对话内容（金句、信息传递）
  - 增加思考过程（主角的内心独白）
  - 增加场景切换（多个场景串联）

#### 原创性要求

- **必须对比原文检查抄袭**：每章写完后，逐段对比原文，确保无雷同
- **禁止模仿原文句式**：不能用相同的句式结构，只能复刻节奏
- **禁止相似表达**：不能用近义词替换的方式抄袭原文
- **检查方法**：将仿写内容与原文逐句对比，确保表达方式完全不同

#### 输出方式

**单个 txt 文件，存放在仿写试水库**：
```
仿写试水库/{新书名}_试水.txt
```

#### 文件内容格式

```

- 第3章：情绪基调 [X]，爽点 [X]，章尾钩子 [X]

【后续方向】
- 第4章预告：[简述]
- 全书规划：[卷数/总章数/主线走向]

```

#### 写作要求

1. **严格复刻节奏**：源文本此处是压抑，新文本此处也必须压抑
2. **场景切换对齐**：保持相同的场景顺序
3. **情绪拉扯同步**：相同的手法（信息差、对比等）
4. **细节完全原创**：台词、道具、环境描写全部原创
5. **黄金三章钩子**：前3章完成「世界观建立 → 金手指展示 → 第一个爽点」
6. **金手指合理性**：符合"数字大+限制简单+场景丰富"三要素
7. **角色性格鲜明**：主角有2-3个鲜明特点，通过场景/对话/行为展现
8. **节奏对齐检查**：每章写完后对照原文检查节奏
9. **语言完全原创**：禁止模仿原文句式、表达方式、修辞手法。必须用全新的语言风格
10. **主角性格展现**：通过具体行为、对话、决策展现性格，而非直接描述
11. **禁止章节内重复段落**：同一章节内，任何段落的核心信息不能出现两次。如果发现重复，必须删除或改写为新信息
12. **复刻原文幽默感**：如果原文有幽默元素（如与AI对话、自嘲），仿写必须在相同位置保留幽默感
13. **禁止过度内心独白**：反思/哲学段落不得超过连续200字，必须用行动、对话或场景切换打断
14. **章尾钩子必须有冲突**：章尾不能以纯哲学反思结尾，必须有具体冲突、悬念或行动承诺

#### 技法复刻要求

15. **开头必须有钩子**：禁止线性叙事开头
    - 悬念式：抛出反常识问题
    - 结果式：先展示结果，再回溯过程
    - 冲突式：直接进入矛盾场景
16. **细节密度要求**：用具体数字堆砌真实感
    - 必须包含：具体金额、具体物品、具体数据
    - 禁止用概述替代细节
17. **系统/金手指出场要有仪式感**：
    - 必须有视觉冲击：金色字体、屏幕黑屏、特殊音效
    - 必须有情绪反应：主角震惊、怀疑、验证的过程
    - 必须有数字冲击：具体金额的视觉呈现
    - **示例**："手机屏幕突然黑了。屏幕上浮出一行字。金色的字。"
18. **对话必须有金句**：
    - 每章至少2句生活化、可传播的台词
    - **示例**："混不下去就回来，天塌不了"
    - **示例**："二十多岁谁留在这儿啊？去南方进厂，一个月包吃住还能挣四五千"
    - **示例**："是不是在外头混不下去了？"
    - **示例**："长期熬夜，肾气不足啊小伙子"
    - 禁止平淡对话（❌"你怎么回来了？" ✅"是不是在外头混不下去了？"）
    - **技巧**：对话要有人物性格，不能是纯信息传递
19. **场景分隔用省略号**：场景切换时用"……"分隔
20. **必须有系统验证过程**：主角测试金手指边界（成功+失败案例）
21. **必须有物价展示**：
    - 用具体物价展示环境特点，制造代入感
    - **必须包含**：日常消费（牛肉面八块）、房价（四千一平）、对比（"一天能买一百套房"）
    - **更多示例**：
      - 早餐：素粉加茶叶蛋四块
      - 交通：出租车起步价五块，摩的八块
      - 娱乐：足疗128元，KTV包厢88元
      - 购物：安踏特步是最高档品牌
    - 禁止空泛描述（❌"物价很低" ✅"一碗牛肉面八块，早餐五块搞定"）
22. **必须通过对话获取信息**：禁止用概述交代背景
23. **必须有消费场景**：
    - 主角获得金手指后，必须有具体的消费体验
    - **必须包含**：去实体店消费、体验服务、与服务人员互动
    - **消费场景要接地气**：洗脚、吃饭、买日用品，不要上来就买豪车豪宅
    - **消费场景要展示环境特点**：县城最高档的足疗198元
    - **更多消费场景**：
      - 吃饭：路边摊、小饭馆、奶茶店
      - 购物：超市、服装店、五金店
      - 娱乐：足疗、KTV、网吧
      - 服务：理发、修车、看病
24. **考察阶段至少2章**：第1章获得金手指，第2-3章考察环境
25. **第3章必须有深度思考**：
    - 主角对当地经济结构的分析
    - 主角对发展方向的规划
    - 主角对自身优势的思考
    - 章尾必须有明确的下一步计划

### Phase 5：质量检查

仿写完成后执行检查：

#### 结构一致性
- [ ] 节奏曲线与源文本一致
- [ ] 转折点位置对齐
- [ ] 场景切换顺序相同
- [ ] 情绪拉扯手法复刻

#### 原创性
- [ ] 角色姓名、设定完全不同
- [ ] 背景世界观完全不同
- [ ] 具体情节、台词完全原创
- [ ] 无直接抄袭或过度相似段落
- [ ] 每章字数达到 2000 字左右（1800-2200字）
- [ ] 逐句对比原文，无相似表达

#### 逻辑自洽
- [ ] 新故事内部逻辑通顺
- [ ] 角色动机合理
- [ ] 转折有铺垫支撑

#### 金手指合理性
- [ ] 符合"数字大+限制简单+场景丰富"三要素
- [ ] 限制条件一句话能说清
- [ ] 能自然衍生冲突
- [ ] 有清晰的升级路径

#### 角色塑造
- [ ] 主角有2-3个鲜明的性格特点
- [ ] 性格通过场景/对话/行为展现
- [ ] 配角有明确的功能定位（帮手/对手/导师/情感线/信息源）
- [ ] 角色关系图谱清晰
- [ ] 配角性格与主角形成对比或互补

#### 爽点节奏
- [ ] 爽点位置与原文对齐
- [ ] 爽点类型与原文一致
- [ ] 爽点释放时间与原文相当

## 异常处理

### 源文本质量差

**表现**：源文本逻辑混乱、节奏失控、金手指设计不合理

**处理**：
1. 提示用户源文本存在问题
2. 建议先用 `/story-long-analyze` 深度拆解
3. 如果用户坚持，提取可用部分，标记问题区域

### 源文本太短

**表现**：源文本不足3章，无法完整提取节奏

**处理**：
1. 提示用户源文本太短
2. 建议补充更多章节
3. 如果用户坚持，基于现有内容提取，标记缺失部分

### 仿写逻辑漏洞

**表现**：仿写过程中出现逻辑不自洽

**处理**：
1. 暂停仿写
2. 指出逻辑漏洞位置
3. 提供2-3个修复方案
4. 用户选择后继续

### 金手指设计失败

**表现**：金手指不符合三要素

**处理**：
1. 提示金手指设计问题
2. 提供修改建议
3. 用户确认后继续

### 节奏不对齐

**表现**：仿写节奏与原文不一致

**处理**：
1. 对比原文和仿写的节奏曲线
2. 指出不对齐的位置
3. 提供调整建议

## 常见错误

### 错误1：仿"皮相"不仿"骨相"

**表现**：只换了名字，情节高度相似

**正确做法**：只复刻节奏和结构，具体情节必须原创

### 错误2：金手指设计不合理

**表现**：数字太小、限制太复杂、场景单一

**正确做法**：严格遵循"数字大+限制简单+场景丰富"三要素

### 错误3：节奏不对齐

**表现**：原文第3章是爽点，仿写第3章还在铺垫

**正确做法**：逐章对照原文，确保节奏对齐

### 错误4：细节抄袭

**表现**：台词、道具、环境描写与原文雷同

**正确做法**：所有细节必须原创，不能有任何雷同

### 错误5：角色性格不鲜明

**表现**：主角没有特点，配角功能不清

**正确做法**：主角2-3个关键词，配角明确功能定位
