# 元迭代改进技能

让技能（SKILL.md）通过「生成 → 评估 → 分析 → 改进」的闭环循环自我进化。

## 触发条件

当用户说：
- `/meta-iterate`、`/迭代`、`/自我迭代`
- "迭代这个技能"、"让技能自己改进"
- "用迭代提升质量"、"持续改进"

## 工作流程

**核心行为**：迭代的是 SKILL.md 文件本身，不是生成的内容。

### 步骤 1: 确定迭代目标并收集输入

**自动完成**：
1. 读取用户提供的输入数据（源文本、范文等）
2. 分析输入数据的结构和特点
3. 设定迭代目标（基于技能描述自动推断）

**重要**：没有输入数据，无法进行真正的迭代！

### 步骤 2: 使用当前 SKILL.md 生成内容

- 使用当前版本的 SKILL.md 指导生成内容
- 保存生成结果到 `.meta-iteration/samples/`
- **关键**：生成的是内容，但迭代的是 SKILL.md

### 步骤 3: 评估生成的内容

评估维度（每项 0-100 分）：

| 维度 | 权重 | 评估要点 |
|------|------|----------|
| 内容质量 | 30% | 准确性、深度、相关性 |
| 结构完整性 | 25% | 组织性、连贯性、完整性 |
| 语言表达 | 25% | 流畅性、准确性、风格一致性 |
| 创新性 | 20% | 原创性、惊喜度、记忆点 |

### 步骤 4: 分析问题并更新 SKILL.md

**这是核心步骤！必须更新 SKILL.md 文件！**

1. 分析内容中的问题
2. 将问题转化为 SKILL.md 中的规则
3. 更新 SKILL.md 文件
4. 保存新版本到 `.meta-iteration/versions/v[N]/SKILL.md`

**示例**：
- 问题：内容开头平淡 → 更新 SKILL.md：添加"开头必须有钩子"规则
- 问题：对话缺乏金句 → 更新 SKILL.md：添加"每章至少2句金句"规则
- 问题：字数不均衡 → 更新 SKILL.md：添加"每章字数差异不超过20%"规则

### 步骤 5: 自动替换原技能文件

每次迭代后，**必须执行以下操作**：

```bash
# 1. 保存新版本存档
mkdir -p .meta-iteration/versions/v[N]
cp .claude/skills/[skill-name]/SKILL.md .meta-iteration/versions/v[N]/SKILL.md

# 2. 记录指标
echo '{"version":"v[N]","score":XX,"improvements":[...]}' > .meta-iteration/versions/v[N]/metrics.json

# 3. 【关键】自动替换原技能文件
# 将更新后的 SKILL.md 写入原技能目录，立即生效
cp .meta-iteration/versions/v[N]/SKILL.md .claude/skills/[skill-name]/SKILL.md
```

**重要**：步骤 3 是自动执行的，不需要用户确认。迭代完成后，原技能文件立即更新为最新版本。

### 步骤 6: 使用新 SKILL.md 重新生成内容

- 使用更新后的 SKILL.md 指导生成内容
- 保存生成结果到 `.meta-iteration/samples/`
- 评估新内容的质量
- 如果质量达标，迭代完成；否则返回步骤 4

### 步骤 7: 终止条件

迭代停止当：
- ✅ 质量分数 ≥ 阈值（默认 85）
- ✅ 连续 3 次改进幅度 < 2%（收敛）
- ✅ 达到最大迭代次数

### 步骤 8: 输出结果


## 仿写迭代发现的成功模式

通过实际使用 story-rewrite 技能迭代，发现以下成功模式：

### 1. 开头从系统出场开始（置信度 95%）
- **问题**：从日常对话开始，缺乏冲击力
- **解决**：直接从系统出场开始，更有悬念
- **示例**："手机屏幕黑了。不是没电，是突然黑了。"

### 2. 系统出场仪式感（置信度 90%）
- **问题**：系统出场平淡，缺乏视觉冲击
- **解决**：增加金色字体、屏幕黑屏、情绪反应
- **示例**："屏幕上浮出一行字。金色的字。"

### 3. 章尾钩子明确（置信度 85%）
- **问题**：章尾缺乏悬念，读者没有继续阅读的动力
- **解决**：以具体计划或冲突点结尾
- **示例**："第一步：买下整条商业街。"

### 4. 主角性格通过行为展现（置信度 80%）
- **问题**：直接描述性格，缺乏说服力
- **解决**：通过具体行为、对话、决策展现性格
- **示例**："他掐了自己一把，疼。不是做梦。"

## 快捷命令

| 命令 | 说明 |
|------|------|
| `/meta-iterate [skill]` | 迭代指定技能 |
| `/meta-iterate status` | 查看迭代状态 |
| `/meta-iterate compare v1 v2` | 对比两个版本 |
| `/meta-iterate rollback [version]` | 回滚到指定版本 |

## 配置文件

迭代配置保存在 `.meta-iteration/META-CONFIG.md`：

```markdown
# 元迭代配置

## 技能信息
- 技能名称: story-long-write
- 目标领域: 网文写作

## 迭代参数
- 最大迭代次数: 10
- 质量阈值: 85
- 收敛窗口: 3
- 收敛阈值: 2%

## 评估维度权重
- 内容质量: 0.3
- 结构完整性: 0.25
- 语言表达: 0.25
- 创新性: 0.2
```

## 注意事项

1. **备份原技能**：迭代前自动备份原始 SKILL.md
2. **渐进改进**：避免大幅修改导致退步
3. **保留核心**：不改变技能的核心定位
4. **可回滚**：保留所有版本，随时可回滚
