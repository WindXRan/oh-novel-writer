### 任务1：冲突检测

检查以下矛盾：
- 角色位置是否有跳跃矛盾（前章在A地，后章未交代怎么到B地）
- 关系阶段是否有跳跃矛盾（前章刚认识，后章就像情侣）
- 伏笔状态是否有矛盾（前章说伏笔已回收，后章又说还没解决）

发现冲突 → 只重写矛盾段落（最小单元），不整章重写 → 写回对应正文文件。

### 任务2：分布一致性检查 ⭐

从最近的20章正文（第A-19章到第B章）中提取：
- 每章平均句长 → 计算这20章的方差
- 每章对话占比 → 计算这20章的方差

如果两个方差的任一个低于阈值（句长方差 < 3，对话占比方差 < 0.05），意味着最近20章的写作分布在统计上过于均匀——这是AI生成的典型特征。

→ 标记警告，并在批后输出中给出下一批的写作指导：
```
【分布警告】最近20章句长方差=X（阈值3），对话占比方差=Y（阈值0.05）
【建议】下一批在每章中主动变异：1-2章提高对话占比到60%，1-2章降低到30%
```

### 任务3：更新真相

汇总各章状态变更 → 更新真相.md → 追加本次批次记录。

```
### 分布一致性检查

- 20章对话占比方差：Y（阈值0.05）→ [通过/警告]
- 建议（如有警告）：

（更新后的位置/关系/信息差）

```
