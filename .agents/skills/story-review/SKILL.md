# story-review · 网文编辑审稿

> 来源：[review-template.md](../story-engine/prompts/review-template.md)，基于 @旅行的巨人 的审稿提示词模板。

## 功能

根据用户提供的小说内容（正文或大纲），扮演专业网文编辑进行审稿评估：

1. 通用版审稿 — 综合商业化+读者体验
2. 女频言情版审稿 — 针对女频赛道的深度审稿
3. 毒舌版 — 尖锐真实、不鼓励不客气
4. 平台编辑口吻版 — 模拟平台责编
5. 万能公式 — 可组合的模块化审稿
6. **衔接性审稿** — 专项检查章间衔接、情节因果、人物状态、时间线、设定一致性

## 工具

本 skill 自带以下脚本，负责长篇审稿的批处理、汇总和修复：

| 脚本 | 用途 |
|------|------|
| `tools/full_review.py` | 分批审稿 + 汇总合并（长篇核心策略） |
| `tools/full_fix.py` | 根据审稿报告并行修复章节 |
| `tools/novel_review_rewrite.py` | 审稿→DAG规划→波次修复→验证闭环 |

### 长篇审稿策略

长篇小说（>20章）审稿采用 **分批→汇总** 两阶段策略：

```
第一阶段：分批审稿（并行）
  每 batch_size 章一批 → 独立审稿 → 输出「审稿_第N批_X-Y.md」

第二阶段：汇总分析
  所有批次结果 → 跨章系统性问题分析 → 输出「全文审稿汇总报告.md」
  - 致命逻辑矛盾（人物生死/身份/时间线）
  - 人设漂移
  - 重复桥段
  - 债务/数值矛盾

第三阶段（可选）：修复
  读取审稿报告 → 并行修复章节 → 字数控制±10%
```

### CLI 用法

```bash
# 分批审稿 + 汇总
python .agents/skills/story-review/tools/full_review.py --config configs/xxx.json --start 1 --end 100 --batch-size 20

# 根据审稿报告修复
python .agents/skills/story-review/tools/full_fix.py --config configs/xxx.json --start 1 --end 100

# 完整审改流程（审稿→DAG规划→波次修复→验证）
python .agents/skills/story-review/tools/novel_review_rewrite.py --config configs/xxx.json --start 1 --end 100 --max-rounds 3
```

## 使用方式

```
/story-review <内容类型> <粘贴内容或文件路径>
```

### 参数

- **内容类型**：`正文` / `大纲` / `开篇`（默认正文）
- **审稿风格**：`通用` / `女频` / `毒舌` / `编辑` / `衔接`（默认通用）

## 审稿模板

### 一、通用版AI编辑审稿

```
请你扮演专业网文编辑，从网络文学商业化和读者阅读体验角度，对我提供的小说内容进行审稿。

审稿要求：
1．判断题材定位是否清晰，包括频道归属、目标读者、核心卖点、主线冲突。
2．分析开篇是否有抓力，是否能在前3章内抛出核心矛盾、人物关系和追读钩子。
3．分析主角人设是否鲜明，是否具备记忆点、反差感、成长空间与情绪价值。
4．分析剧情节奏是否流畅，是否存在起步慢、信息堆砌、冲突不足、爽点或虐点不明确的问题。
5．分析感情线或主线推进是否自然，是否存在动机不足、误会生硬、转折突兀等问题。
6．分析文风是否符合目标平台和目标读者，包括代入感、画面感、情绪浓度、对白表现力。
7．指出最影响签约率、追读率、付费率的关键问题。
8．给出具体修改建议，要求能直接落地，避免空泛评价。
9．按照"优点、问题、修改建议、签约潜力判断"四个部分输出。
10．最后给这篇文打分，满分100分，并说明依据。

补充要求：
- 语言直接、专业，不要只鼓励不批评。
- 优先从商业价值、读者爽点、情绪拉扯、人物关系张力来判断。
- 如果内容有潜力，请明确指出潜力点在哪里。
- 如果内容问题较大，请明确说明主要卡点是什么。
- 如果我发的是开篇章节，请重点审前3万字的起势能力。
- 如果我发的是大纲，请重点审故事结构、卖点和可写性。

以下是我的小说内容：
【把小说内容粘贴在这里】
```

### 二、女频言情版审稿

```
请你扮演资深女频网文编辑，按照起点女频、番茄女频、七猫女频、飞卢女频、晋江言情的商业审稿标准，对我的小说进行专业审稿。

请重点围绕以下维度分析：

一、基础定位
1．题材是否明确，是否属于女频主流赛道。
2．核心梗是否清晰，能否用一句话概括卖点。
3．目标读者画像是否明确，读者会因为什么追更。

二、开篇质量
1．前1章是否有钩子。
2．前3章是否建立男女主关系张力。
3．前5章是否抛出核心悬念、情绪矛盾或身份冲突。
4．是否存在铺垫过长、信息密度低、剧情不推进的问题。

三、人物塑造
1．女主是否有代入感、委屈感、成长线、可怜点与可爱点。
2．男主是否有吸引力，是否具备高位者魅力、反差感、占有欲、情感迟钝或深情反转等记忆点。
3．男女主互动是否有拉扯感、暧昧感、压迫感或宿命感。
4．配角是否服务主线，是否抢戏或脸谱化。

四、情节与情绪
1．先婚理由是否合理。
2．婚后相处是否有持续冲突和情绪递进。
3．虐点是否成立，是否能让读者心疼女主。
4．误会、冷战、和好、心动、反转是否自然。
5．是否具备追妻火葬场、打脸、修罗场、身份反转等高追读元素。

五、商业价值
1．是否具备签约潜力。
2．是否具备长篇连载空间。
3．适合哪些平台风格。
4．读者可能弃文的节点在哪里。
5．最值得强化的3个卖点是什么。

请按照以下格式输出：
【一句话总结】
【核心卖点】
【最大优点】
【主要问题】
【详细审稿意见】
【修改方案】
【平台适配建议】
【签约潜力评分：XX/100】

以下是我的内容：
【粘贴大纲或正文】
```

### 三、毒舌版

```
请不要只做鼓励式评价，而是像真正要决定签约与否的编辑一样审稿。结论要明确，问题要尖锐，修改建议要具体。

请优先指出这篇文为什么可能不过稿、为什么追读差、为什么读者会弃文。
```

### 四、平台编辑口吻版

```
请模拟小说平台责编写审稿意见，重点判断：
1．是否具备签约价值；
2．开篇是否有付费转化潜力；
3．人物关系是否有持续拉扯空间；
4．题材是否符合当前女频热门方向；
5．故事是否存在明显短板。

输出要求：
- 用编辑口吻
- 简洁、专业、明确
- 先下结论，再讲原因
- 最后给出"建议过稿／建议观察／建议拒稿"的判断

小说内容如下：
【粘贴内容】
```

```
语气直接专业，不要泛泛而谈。
以下是正文：
【内容】
```

### 六、隐藏好用指令（加到任意模板里）

#### 1. 必须给结论
```
请不要模棱两可，必须明确回答：这篇文最大的卖点是什么，最大的问题是什么。
```

#### 2. 给可执行修改
```
每一个问题后面，必须附带至少1条可以直接修改正文的建议。
```

#### 3. 从读者视角判断
```
请结合读者追更心理，分析每一章之后读者为什么会继续看，或者为什么会弃文。
```

### 七、最短最好用版

```
【粘贴内容】
```
