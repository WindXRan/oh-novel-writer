# story-optimize

> 审稿反馈 → 可执行规则 → 沉淀到prompt → 后续生成自动应用

## 流程

```
读报告 → 提问题 → 判断通用性 → 提取规则 → 沉淀到prompt
```

**读取来源**：用户粘贴内容 / `compare/对比_*_AI分析.md` / `compare/对比_*_报告.md`

## 问题→规则映射

| 审稿反馈 | 规则 | 沉淀到 |
|---------|------|--------|
| 节奏太慢 | 禁止连续2章纯日常 | plot |
| 人设无记忆点 | 男主必须有钩子人设 | plot |
| 开篇无吸引力 | 前3章必须完成核心悬念 | plot |
| 情节套路化 | 禁止常见套路（继母威胁/亲戚上门/下药救场） | plot |
| 角色转变突兀 | 转变必须有铺垫和触发事件 | plot |
| 配角脸谱化 | 配角必须有反差/弱点/动机 | plot |
| 信息密度低 | 每章必须有信息增量 | plot |
| 冲突不足 | 每章必须有冲突或悬念 | plot |
| 对话占比偏高/偏低 | 以源文为准（±10%） | style |
| AI味太重 | 禁止直抒情词/AI路标词 | style |
| 省略号太多 | 省略号≤源文数量 | style |
| 文笔直白 | 增加细节/环境描写 | style |
| 简介质量差 | 简介必须有悬念+情感张力 | open-book |
| 人名不对 | 禁用字清单 | open-book |

## 规则格式

```markdown
### 🔴 规则名称

**禁止**：具体禁止行为
**正确做法**：具体正确行为
**示例**：
- ❌ 违规示例
- ✅ 正确示例
```

**质量标准**：可执行、可验证、有正反例、精准（一句话能说清不用三句）

## 输出

1. 修改对应prompt文件（plot-guide.md / style-guide.md / open-book.md）
2. 向用户报告：问题→规则→沉淀位置→预期效果
