# story-blurb · 网文书名+简介生成

## 功能

根据源文设定和新书设定，生成：
1. 网文书名（不与源文对标）
2. 网文化简介（口语化、有节奏、有钩子）
3. 封面（可选，需OpenAI key）

## 核心原则

### 简介风格要求

**必须做到**：
- 口语化，像真人写的，不是AI生成的
- 短句+长句交替，有节奏感
- 有钩子（悬念/反转/冲突）
- 有标签（开头亮标签）
- 人称正确（与源文一致，通常第三人称）
- 200-300字，不要太长
- 结尾留悬念，不要剧透结局

**必须避免**：
- AI感（"悬着的心终于死了"、"好家伙"这种还是有点AI）
- 太端着、太文艺
- 太长、太啰嗦
- 与源文简介结构雷同
- 与源文书名对标
- 剧透结局

### 书名风格要求

**必须做到**：
- 网文感强（参考番茄/七猫热榜）
- 有记忆点
- 不与源文书名撞梗

**必须避免**：
- 与源文书名相似度过高
- 太文艺、太小众

## 流程

### Step 1：分析源文

读取源文信息：
- 书名
- 简介
- 核心设定（穿书/替嫁/重生等）
- 男女主人设
- 核心冲突
- 大结局方向（HE/BE/开放式）

### Step 2：分析新书

读取新书设定（concept.md）：
- 核心设定差异
- 男女主人设
- 核心冲突
- 目标读者
- 大结局方向

### Step 3：生成书名

生成5个候选书名，要求：
- 不与源文书名撞梗
- 网文感强
- 有记忆点

### Step 4：生成简介

生成5个候选简介，要求：
- 人称与源文一致（通常第三人称）
- 开头亮标签
- 短句+长句交替
- 有钩子（悬念/反转/冲突）
- 200-300字
- 口语化，像真人写的
- 结尾留悬念

### Step 5：生成封面（可选）

调用 story-cover skill 生成封面prompt。

## 简介风格参考

### 好简介特征（源文风格）

```
【已签实体】
[偏日常/细水长流/先同居后爱/双洁/甜宠]

一朝穿越，宋云绯成了假冒失忆总裁女友的女配。
原书中，女配不小心开车撞了宴金集团的总裁，因怕坐牢，趁他失忆假冒他的女友。
不仅骗他是个穷小子，还带着他去了偏远城市躲起来，对他各种PUA。
后来男主被家人找到，男主刚恢复记忆，结果未婚妻要跟别人结婚了，他开始追妻火葬场。
至于宋云绯这个恶毒女配，男人秋后算账，不仅逼迫她打掉孩子，还坐了五年牢，出狱后又惨死在街头。
穿越过来的宋云绯一睁眼，看着刚送外卖回来的男人，两眼一黑，悬着的心终于死了。
没办法，宋云绯只能将错就错。
一边跟楚靳寒过着贫穷的日子，一边准备跑路。
然而，终于等到跑路那天，男人却愤愤地将她抓了回来。
"骗完我就想跑？"
宋云绯震惊，好家伙，这人什么时候偷偷恢复记忆的？
```

**特点**：
- 口语化，有网感
- 短句+长句交替
- 有钩子（"骗完我就想跑？"）
- 有反转（男主恢复记忆）
- 结尾留悬念

### 坏简介特征（AI感太重）

```
陈茉替姐出嫁那天，顾衍恨她恨得咬牙切齿。
三年后，他车祸失忆了。
忘了恨，忘了她是谁，每天冷着脸给她做饭、接她下班、雨天骑电动车来接她。
陈茉心想：完了，这人失忆后比失忆前还可怕。
```

**问题**：
- "陈茉心想"太文艺
- "完了，这人失忆后比失忆前还可怕"有点AI感
- 缺少具体细节
