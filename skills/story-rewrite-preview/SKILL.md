# story-rewrite-preview：仿写试水（3章预览）

**核心信念：只仿"骨相"，不仿"皮相"。**

---

## 流程总览

```
Phase 1（主线程）    读源文本 → 提取框架 + 设定锁 + 语感样本
Phase 2（试水）      出3章章纲 → 3章并行写 → 字数修复 → review → 输出试水文件
```

**只写3章，输出到 `仿写试水库/试水_{书名}.txt`，不创建项目目录。**

如需写全书，使用 `/story-rewrite`。

### 风格插件支持

与 `story-rewrite` 共享同一个风格插件系统（由 `story-style` skill 管理）。支持 `--style` 参数：

```
/story-rewrite-preview --style=wenqi    # 闻栖风格试水3章
```

不加 `--style` 时用源文本语感样本驱动（原行为）。

风格加载路径：`skills/story-style/{name}/meta.json` → `story-style` injection 引擎提取 section 注入 prompt。

> **Phase 1 复用**：如果项目已有 `仿写试水库/框架_{书名}.md`（由 `/story-rewrite` 生成），直接读取，跳过 Phase 1。

---

## Phase 1：框架提取

主线程读源文本，提取：

1. **源文本分析**（不只是读前3章）：
   - 前3章：开篇方式、破冰方式、人设标签、叙述者风格
   - 中间转折点：读第1/4、1/2、3/4处各1章，提取节奏变化、情绪转折、冲突升级方式
   - 最后1章：结局走向、伏笔回收方式
   - 全书章数、卷结构、每卷核心事件
   - 提取3-5段语感样本（每段30-100字）+ 1章完整样本供agent深度模仿

2. **自动生成新书方案**（AI全自动）：
   读源文本后，根据源文本结构分析 + 风格插件（如有）自动生成新书信息，直接写入框架：
   - **书名/主角/设定/章数/题材**：AI自行选最优，不询问用户，不弹选项，不阻断。

3. **设定锁**（所有agent共享的硬设定）：
   - 核心地名（具体地址）
   - 关键物件（来源+状态）
   - 人物关系锁（不可更改的关系）
   - 背景事件锁（统一的过去事件描述）
   - 时间线锁（各章时间约束）
4. **语感特征**：3-5个核心节奏特征（一句话描述+简短示例）

**反调色盘自检**：新书的"核心设定+核心冲突+角色原型"和源文本放一起对比，如果外人看了觉得"这不就是同一本书吗"，必须重写框架。具体维度：

| 维度 | 红线 |
|------|------|
| 核心设定 | 机制不能相同（只是换金额/地点不算差异化） |
| 核心冲突 | 驱动矛盾不能同类 |
| 角色原型 | 身份/处境/动机不能同类 |
| 开篇方式 | 第1章核心事件不能同类 |
| 破冰方式 | 第一次互动方式不能同类 |

## Phase 2：试水（前3章）

**不创建项目文件夹**，只输出 `仿写试水库/试水_{书名}.txt`。

### Step 1：出章纲（主线程）

主线程根据框架中的关键事件锚点，生成3章章纲（不启动agent）。章纲不单独写文件，直接注入Step 2的prompt。

### Step 2：写正文（黄金三章，同一 agent）

启动1个 `narrative-writer` agent 写第1-3章（K=3）。prompt注入设定锁+语感样本+3章章纲。**agent直接返回完整正文，不写文件。**

⚠️ 黄金三章必须由同一个 agent 写，保持开篇风格一致性。

3章全部返回后，主线程逐章执行字数自动修复：

```powershell
# 对每章正文变量 $content 执行：
$chars = ($content -replace '[\s]','').Length
if ($chars -gt 2400) {
    $paragraphs = $content -split '(\r?\n\r?\n|\r?\n)'
    $result = ""
    foreach ($p in $paragraphs) {
        $test = $result + $p
        if (($test -replace '[\s]','').Length -gt 2350) { break }
        $result = $test
    }
    $content = $result
}
if ($chars -lt 1800) {
    # 用 task_id 恢复同一批 agent 会话，要求补写
    # prompt："当前正文只有 {chars} 字，请补充细节/对话/场景描写，目标2000字，最低1800字。保持原有情节不变，只扩充。"
}
```

### Step 3：story-review 一致性审查

启动 `story-review` 检查3章之间的一致性。发现矛盾 → 主线程直接改文本。

### Step 4：输出

合并3章正文写入 `仿写试水库/试水_{书名}.txt`（含简介+框架摘要+正文），向用户汇报字数。

**⚠️ 试水阶段不生成任何项目目录。用户确认满意后，用 `/story-rewrite` 写全书。**

```
# ⚠️ 叙事视角：必须第三人称。

## 你的声音
{STYLE_VOICE}
{! 正面风格指引专区。禁用词等负面约束仅出现在Rule 6，勿在此重复。}
{! 无 --style 时：用源文本语感样本 }
{! --style 时：从 source_skill 读取 ## 表达DNA section（只取句式偏好/高频词/对话风格/节奏感，跳过禁用词列表）替换此处 }

## ⚠️ 设定锁（不可违反）
{设定锁内容}
⚠️ 不在设定锁中的细节，用最简化的处理，不要自行编造。

## 前情摘要
无，本章为开篇。

## 本章章纲
{由主线程从章纲中提取本章内容，直接注入}

## ⚠️ 字数：目标{STYLE_TARGET_WORDS} | 范围{STYLE_MIN_WORDS}-{STYLE_MAX_WORDS} | 上限{STYLE_HARD_MAX}

## 写作原则（按优先级执行）
{STYLE_RULES}
{! --style 时：从 source_skill 提取追加到此，格式：}
{!   - ## 核心写作心智模型：只取模型名+一句话摘要，跳过案例/失效条件}
{!   - ## 决策启发式：全取}
{!   - ## 价值观与反模式：只取反模式条目（❌行）}
{!   - 删除所有与 voice 重复的内容}
{! 无 --style 时：此部分为空 }

你拥有以下参考文件，遇到具体问题时按需读取（路径相对于项目根目录）：

- `skills/story-long-write/references/dialogue-mastery.md`（对话规则、权力博弈、人物语言差异化）

### 1. 镜头式写作（最高优先级）
- 你是跟在主角身后的摄像机，只写能看到/听到/闻到/摸到的
- 不写"他心里想""他感到"，用动作和五感展示
- **例外**：女主内心OS/吐槽式旁白不受此限（属于人物声音而非叙述者声音）
- 每个段落=一个镜头，有明确的"拍摄对象"
- 五感：每个场景至少调动2种感官

### 2. 对话规则
- 每句对话必须推进剧情/塑造人物/制造张力之一
- 权力博弈：掌控者话短（≤10字），被动者话长（≥20字）
- 人物差异化：不同角色说话方式不同
- 口语化，不要书面腔

### 3. 情绪交付
- 不直接写"他愤怒/伤心/紧张"，用身体反应和行为展示
- 每章内部确定目的词（铺垫/高潮/爽点/打脸/人物塑造/设定），但不要写入正文
- 爽点公式：先不爽→制造希望→努力解决→得偿所愿
- 期待感：每章至少1个未闭环的期待
- **阶级反差**：贫富/地位差距大时用具体数字+生理细节写足震撼，不要轻描淡写

### 4. 章首钩子（选一种）
- 倒计时/意象预示/悬念对话/反差场景/闪前碎片/未完成动作/危机升级
- 同批内同类型钩子不超过2个

### 5. 章尾钩子（选一种）
- 突然揭示/紧急危机/未完成动作/两难抉择/神秘物品/意象钩子/留白
- 用动作或悬念收束，不要总结感悟

### 6. 去AI（必做）

**禁用词**（必须避免）：
- 仿佛、犹如、宛若、如同、深吸一口气、缓缓、不禁、微微、轻轻、淡淡
- 眼中闪过、嘴角勾起、眉头微皱、心中一动、心头一震
- 脊背绷得笔直、嘴角翘了一下不是笑是XX

**句式约束**：
- 不用模板化比喻（"像XX般""如XX般"）
- 不用连续排比（"第一反应……第二反应……第三反应……"）
- 不用对仗工整句（"不是A，而是B"句式）
- 不用升华结尾（"他终于明白了""这一刻，他懂了"）

**节奏约束**：
- 段落长度：1-3句为主，偶尔1句独占1行
- 单句段≥30%，连续3段同句数必须打散
- 句首：连续3句同主语开头必须换
- 段落长度不能均匀（禁止每段都是4-6句）

**对话约束**：
- 60%+ 对话无标签，用动作替代"说/问道"
- 对话不能太书面化，要有口语感（"行吧""算了""不是"）
- 对话不能面面俱到，要有省略（读者自己脑补）

**情绪约束**：
- 用动作展示情绪，不用直接告诉（"他很紧张"→"手在抖"）
- 用生活化比喻，不用文学化比喻（"像哈士奇护食"而非"如寒冰般"）
- 用具体动作代替抽象比喻

## 格式
第一行：`第{N}章 {章名}`
⚠️ 目的词是你的内部写作指导，不要写入正文。
返回完整正文。
```
