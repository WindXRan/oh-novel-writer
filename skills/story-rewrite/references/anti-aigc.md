# 反AIGC检测参考

> 本文件供主线程在 Phase 5 全书质量检查时参考，不自动注入 agent prompt。

## 额外反检测技巧

**叙事视角微偏移**
- 偶尔从第三人称切到接近第一人称的内心视角（"他心想"→直接写想法不加标签）
- 偶尔用第二人称插入（"你要是大壮，你也喝"）

**时间线碎片化**
- 不要完整线性叙事，偶尔用倒叙/插叙（1-2句闪回）
- 时间标记可以模糊（"那天下午"而不是"下午三点"）

**方言/口语注入**
- 闽南语/当地口语词汇偶尔出现（不用多，2-3个/章）
- 角色对话必须有口语特征（省略主语、语气词、不完整句）

**物件叙事**
- 用物件状态代替情绪描写（"啤酒沫子塌了"代替"他等了很久"）
- 物件可以"无意义出现"（不要每个物件都必须有叙事功能）

**数字/细节不对称**
- 有的地方给精确数字（"一百二十万"），有的地方故意模糊（"差不多那个数"）
- 不是每个细节都要有叙事功能，允许"废细节"存在

## Phase 5 AIGC结构特征扫描标准

对每章执行以下结构检查，不达标则标记需重写：

| 检查项 | 达标标准 | 不达标处理 |
|--------|----------|-----------|
| 句长标准差 | 10句窗口 > 5字 | 插入超短句/长句打散 |
| 单句段数量 | 每章 ≥ 3个 | 拆分部分段落为单句 |
| 连续同句数段 | ≤ 2段 | 打乱段落结构 |
| 动作-反应耦合率 | ≤ 40% | 删掉部分反应描写 |
| 连续同主语句首 | ≤ 2句 | 换句首 |
| 情绪模板词 | 0个 | 替换或删除 |
| 信息密度对比 | 有极简过渡段 | 增加一笔带过的过渡 |

## 验证 PowerShell 脚本

```powershell
# 统计单句段数量（空行分隔的、只有1行内容的段落）
$content = Get-Content '文件路径' -Encoding UTF8
$paragraphs = @(); $current = @()
foreach ($line in $content) {
    if ($line -match '^\s*$') {
        if ($current.Count -gt 0) { $paragraphs += ,@($current); $current = @() }
    } else { $current += $line }
}
if ($current.Count -gt 0) { $paragraphs += ,@($current) }
$singleSentence = ($paragraphs | Where-Object { $_.Count -eq 1 }).Count
Write-Output "单句段数: $singleSentence (目标: ≥5)"
```
