# story-rewrite 踩坑记录

> 历史问题与解法，供排查时参考。

---

### Windows 中文编码问题（2025-05-30）

**现象**：`python -c` 执行包含中文的三引号字符串时，PowerShell 编码与 Python 不一致，导致 `SyntaxError: unterminated triple-quoted string literal`。

**根因**：Windows PowerShell 5.1 默认编码（GBK/cp936）与 Python 的 UTF-8 不匹配，中文字符被截断。

**解法**：
- 文件写入 → 用 `write` 工具，不用 `bash` + `python -c`
- 字数统计 → 先 write 文件，再单独 `python wordcount.py` 读文件统计
- 绝不在 `python -c` 命令里包含多行中文字符串

### 多agent并行导致故事断裂（2025-05-30）

**现象**：多个agent并行写多章，每个agent自行设计关键事件，导致章节间时间线断裂、事件版本冲突、人物行为矛盾。

**根因**：Phase 2 prompt 没有锁定关键事件锚点，agent各自发挥。

**解法**：
- Phase 1 框架新增「关键事件锚点表」，主线程锁定每章必须发生的1-2个关键事件
- Phase 2 新增章纲生成步骤，将锚点展开为具体情节点
- Phase 3 prompt 注入章纲内容，agent在章纲内自由发挥

### story-rewrite 章纲与 story-long-write 细纲对齐（2025-05-30）

**问题**：story-rewrite 章纲只有2个关键事件+收尾钩子，粒度太粗。

**对齐方案**：章纲格式对齐 story-long-write 细纲，包含：核心事件、情节点序列、目标情绪、章首钩子、爽点、章尾钩子（含期待度）、字数目标。

### opencode 逐字输出导致长文本慢（2025-05-30）

**解法**：agent 用 write 工具写入文件，不返回正文到终端。只返回状态信息。

### 多任务并行导致临时文件覆盖（2025-05-30）

**解法**：临时文件用新书名作前缀：`temp_{书名}_ch{N}.txt`

### 批量模式上下文溢出（2025-05-30）

**解法**：摘要压缩机制（近6章详记+更早概要+卷级总览），每批只注入压缩摘要。

### 朱雀AI检测AIGC值0.94-0.99（2025-05-30）

**现象**：rewrite产出的3章全文被朱雀AI检测标记为AI特征，AIGC值0.94-0.99。

**根因v1**：禁用词替换只能处理表层AI味，AIGC检测器看统计特征。

**解法v1（已证明无效）**：Phase 3 prompt 新增7条抗AIGC规则。问题：规则列表式约束对LLM无效。

**根因v2**：
1. 场景展开法三层结构本身是AI指纹
2. 段落结构过于均匀
3. 信息密度均匀
4. "禁止"式规则无法改变写作习惯

**解法v2（结构重写）**：
- 从"禁止AI模式"转向"强制注入人类写作特征"
- 新增写作前的结构模板（段落骨架规划、句长分布模板、句首轮换模板）
- 新增8条硬规则（单句段≥5个、信息跳跃≥3处、禁止机械三层展开等）
- 新增写后量化验证清单（10项检查，不达标则重写）
- 场景展开法从"必须三层"降级为"参考"

### 试水阶段字数约束失效（2025-05-30）

**现象**：3章正文全部超标（Ch1=3589, Ch2=3251, Ch3=5975）。

**解法**：字数约束提到 prompt 最高优先级，内置字数自检方法，Phase 4 主线程二次校验。

### 子agent执行卡顿：5个效率bug（2025-05-30）

**Bug 1：对"不是A"规则过度防御**
- agent 把"不是闹钟"误判为禁用的"不是A，而是B"句式
- 修复：禁用句式示例明确标注 ✅ 允许简单否定句

**Bug 2：手动逐字计数**
- agent 在思考中逐字数章节，多次重算，消耗大量 token
- 修复：新增"禁止手动逐字计数"规则，强制使用 PowerShell

**Bug 3：对话段落计数导致反复重构**
- agent 为满足"连续同句数段 ≤ 2段"规则，反复合并/拆分对话段落
- 修复：新增对话段落豁免规则

**Bug 4：角色名/地名在 prompt 中被淹没**
- prompt 框架太长，关键信息混在中间，agent 写错后才发现
- 修复：prompt 顶部新增「角色锚点」块

**Bug 5：发现问题后全文重写而非增量修改**
- agent 每发现一个违规词就全文重写，写了 5 个版本
- 修复：新增「增量修改」策略
