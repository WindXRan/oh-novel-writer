# story-rewrite：结构仿写

**核心信念：只仿"骨相"，不仿"皮相"。**

**架构原则：主线程预读一切，agent 拿到 prompt 就写，不读任何文件。**

---

## ⚠️ 核心约束

### 反调色盘（防止"扒骨肉"）

仿写必须在以下3个维度上与源文本**完全不同**：

| 维度 | 说明 |
|------|------|
| **开篇方式** | 如源文本用心理治疗/催眠开篇，新书必须用其他方式（商战/遗物/急救等） |
| **破冰方式** | 如源文本用送物被拒，新书必须用其他方式（替解围/共同任务/意外接触等） |
| **节奏安排** | 如源文本一章写重生前，新书必须压缩或改变节奏 |

### 时间线连续

Phase 1 框架只提供「情绪弧线+叙事功能」，不提供具体场景。Phase 2 生成章纲锁定关键事件。Phase 3 agent 在章纲内自由发挥。

---

## 流程总览

### 试水模式（3章）

```
Phase 1（主线程，~2min）  Phase 2（1 agent，~2min）  Phase 3（3 agent 并行，~3min）  Phase 4（主线程，~30s）
读源文本 + 提取框架        生成章纲（每章情节点）        仿写3章（prompt即写，不读文件）   合并TXT + 字数统计 + 自检
```

**总耗时：~7-8分钟（试水3章）**

### 批量模式（全书）

```
Phase 1（一次性）: 全书章纲生成 → 存储到文件
Phase 2（循环）: 4章并行写 → 质量检查 → 不达标重写 → 合并 → 摘要压缩 → 滚动
Phase 3（结尾）: 全局一致性检查 + 禁用词扫描
```

**20章耗时：约15-20分钟，一次完成**

---

## 批量模式入口

用户输入：`/story-rewrite 批量 {源文本路径} {目标章数}`

示例：`/story-rewrite 批量 仿写试水库/源文本.txt 20`

**主流程**：

```
1. Phase 1: 读取源文本 + 生成仿写框架 + 全书章纲（一次性）
2. Phase 2: 分批写作循环（每批4章）
   loop:
     读取下4章章纲
     并行启动4个agent写作
     每章质量检查（禁用词+字数+钩子）
     不达标则重写（最多2次）
     合并到正文目录
     压缩前文摘要（近6章详记+更早概要）
     if 已完成目标章数 → 结束循环
     else → 继续循环
3. Phase 3: 全书质量检查
```

## Phase 1：结构提取 + 预加载

**用户只提供源文本路径，剩下全部自主完成。**

主线程收到源文本后，**同步完成以下操作**（不启动 agent，直接在主线程做）：

### Step 1：读取源文本

读取前3章 + 最后1章，记录：
- 源书名、题材、章数
- 开篇方式（具体场景 → 抽象为叙事功能）
- 破冰方式（具体事件 → 抽象为叙事功能）
- 节奏安排（各章情绪基调）
- 人设标签（女主/男主/配角/反派）
- 结局走向

### Step 2：预加载参考文件

同步读取以下两个文件，提取关键内容嵌入 Phase 2 prompt（agent 不再读文件）：

1. `narrative-writer.md`（位于 `.opencode/agents/`）→ 提取**三层展开法**（发生层+感知层+反应层）和**情绪弧线规则**
2. `banned-words.md`（位于 `skills/story-deslop/references/`）→ 提取**一级禁用词**（情态/动作/表情/心理/判断/形容/过渡类，**控制在 20 词以内**，避免挤占 Agent 推理空间）

⚠️ 主线程读取时需拼接项目绝对路径，不能用 `~/.opencode/`（那是用户全局目录，文件不存在）。

### Step 3：生成仿写框架

输出以下框架（只保留抽象模式，禁止具体场景）：

```
- 新书名：
- 新主角：（名字+核心标签，必须与源文本不同）
- 新背景：（时代/世界观，必须与源文本不同时代/不同地域）
- 核心冲突：（1句话）
- 开篇叙事功能：（抽象，如「铺垫→创伤回忆→重生契机」）
- 破冰叙事功能：（抽象，如「主动靠近→遭遇冷遇→坚持」）
- 开篇差异点：（必须与源文本不同）
- 破冰差异点：（必须与源文本不同）
- 节奏对齐表（3章）：
  - 第1章：情绪基调 / 叙事功能 / 弧线走向
  - 第2章：情绪基调 / 叙事功能 / 弧线走向
  - 第3章：情绪基调 / 叙事功能 / 弧线走向
- 人物关系模式（抽象）：
- 信息差设计：
- 核心差异对照表（时代/地域/职业/身份层面）：
- 角色名表（主线程必须统一，禁止 Agent 自行改名）：
  - 女主：{名字}
  - 男主：{名字}
  - 配角A：{名字+功能}
  - 配角B：{名字+功能}
- 题材标签：

## 关键事件锚点表（Phase 1 主线程填写，Phase 2 章纲展开，Phase 3 agent执行）
- 第1章锚点：{必须发生的事件A}，{必须发生的事件B}，本章以{悬念/情绪点}收尾
- 第2章锚点：时间线{紧接第1章/次日/数日后}，{必须发生的事件A}，{必须发生的事件B}
- 第3章锚点：时间线{紧接第2章/次日/数日后}，{必须发生的事件A}，{必须发生的事件B}
```

**自检**：框架给没读过源文本的人，能据此写出完全不同但结构相似的故事吗？不能 → 重新抽象化。

```
## ⚠️ 修改策略（防止反复全文重写）
- 发现违规词/句式时：**只修改该句及其上下文**，不要全文重写
- 用增量修改（找到具体句子替换），不要删除全文从头写
- 禁止写完后在思考中逐字检查禁用词——写入文件后用 bash 命令批量扫描
- 如果多个违规点分散在全文，逐个修改，不要一次性重写

## ⚠️ 结构反检测（AIGC 检测专项）

> AIGC 检测器看的是**统计特征分布**，不是单个禁用词。你需要让文本的统计特征**看起来像人写的**。

### ⚡ 写作前：强制结构模板（agent 必须按此执行）

**段落结构模板**——写正文前，先在脑中规划段落骨架（2000字约35段）：

| 段落类型 | 比例 | 段数（2000字） | 特征 |
|----------|------|----------------|------|
| 单句段（1句） | 30% | ~10段 | 独立成段，制造节奏断裂感 |
| 短段（2句） | 25% | ~9段 | 快速推进 |
| 中段（3句） | 20% | ~7段 | 正常叙事 |
| 长段（4句） | 15% | ~5段 | 场景展开 |
| 超长段（5句+） | 10% | ~4段 | 关键场景或对话密集区 |

**⚠️ 连续3段句数相同 = AI指纹。必须打乱。**

**句长分布模板**——每写10句话，检查：
- 至少1句 ≤ 5字（超短句）
- 至少1句 ≥ 25字（长句）
- 10-20字区间的话 ≤ 6句（不能扎堆）
- 连续3句都在10-20字区间 → 必须打断

**句首模板**——连续3句不能以同一主语开头。强制轮换：
- 人名/代词开头（最多连续2句）
- 物件开头（"手机震了一下""铁门晃了晃"）
- 环境开头（"海风灌进来""阳光从破洞漏进来"）
- 声音开头（"咣一声""滴"）
- 对话开头（直接引语）
- 时间/地点开头（"四点半""码头东侧"）

### ⚡ 写作中：必须执行的8条硬规则

**R1. 单句段必须存在**
- 每章至少5个单句独占一段（不是3个，是5个）
- 单句段必须"不完整"——有意省略主语、省略上下文、只写动作不写反应
- 示例："他恨过这个渔村。"（独立成段，不解释为什么）

**R2. 信息跳跃必须存在**
- 每章至少3处场景直接跳切（不是2处，是3处）
- 跳切方式：空行后直接进入新场景/新时间，不用过渡句
- 允许读者不知道的信息存在（不交代来龙去脉）

**R3. 极简过渡段必须存在**
- 每章至少2段"极简过渡"（≤2句，只交代事实，不展开）
- 用途：时间跳转、空间转移、省略不重要的过程
- 示例："告别老陈头，林昭直接去了镇政府。"（1句完成转场）

**R4. 对话必须不规则**
- 禁止"A说完B回应，B说完A回应"的均匀交替
- 对话中必须有：打断（话没说完就切）、沉默（不回应）、答非所问
- 对话穿插动作/环境描写，不要连续5句纯对话
- ⚠️ 对话段落豁免：连续纯对话行（引号开头）不计入"连续同句数段"检查。对话节奏由内容决定，不由段落结构约束

**R5. 感官描写必须随机分布**
- 不能每个场景都写满3层（发生+感知+反应）
- 关键场景：3层全写
- 普通场景：只写发生层（1-2句完事）
- 过渡场景：跳过或只写1个感官
- **感知层和反应层不是必须的，根据场景重要性决定是否展开**

**R6. 句式必须打散**
- 禁止连续3句用相同句式结构（主谓宾/主谓补/主谓宾宾补）
- 每10句内至少出现2种不同的句式类型
- 口语化短句和书面化长句必须交替出现

**R7. 人味注入（最重要）**
- 每章至少2处"不完美"：犹豫、口误、自我否定、前后矛盾、说一半改口
- 每章至少1处"无意义细节"：和情节无关的生活细节（吃什么、天气、噪音）
- 每章至少1处"语言粗糙"：口语化表达、不完整的句子、语气词

**R8. 禁止"三层展开法"的机械执行**
- 场景展开法是**参考**，不是**模板**
- 禁止每个子事件都严格按"发生→感知→反应"三段式
- 有的子事件只写发生（1句），有的只写反应（跳过发生），有的三层全写
- **机械三层展开 = 最强AI指纹之一**

## 写作红线
- 情绪弧线严格遵循框架，但具体事件由你自行设计
- 禁止1:1映射源文本情节点
- 自检：如果和源文本放在一起，读者会不会觉得「换名同一本书」？会则重写

## 字数自检流程（agent 写完必须执行）
1. 用 write 工具写入文件 `temp_{书名}_ch{N}.txt`
2. 用 bash 执行 PowerShell 命令精确统计字数：
   ```
   (Get-Content 'temp_{书名}_ch{N}.txt' -Raw -Encoding UTF8) -replace '[\s]','' | Measure-Object -Character | Select-Object -ExpandProperty Characters
   ```
3. 字数不在 1600-2400 → 删除文件，从头重写（最多3次）
4. 只返回状态：`第{N}章 {章名} | 字数：{X}字 | 状态：达标`
```

### ⚠️ 关键事件锚定规则（防止多agent各自为政）

**问题根因**：多agent并行时，如果prompt没有锁定关键事件，每个agent会自行设计，导致章节间时间线断裂、事件版本冲突、人物行为矛盾。

**解法**：
1. Phase 1 框架包含「关键事件锚点表」，主线程在生成框架时把每章必须发生的1-2个关键事件锁死
2. Phase 2 生成章纲，将锚点展开为具体情节点（时间线+事件+钩子）
3. Phase 3 agent prompt 注入章纲内容，agent在章纲内自由发挥

锚点是「叙事功能+关键事件」层面的抽象描述，章纲是具体的情节点，agent在章纲框架内自行设计场景。

### 主线程构建 prompt 时的变量填充

| 变量 | 说明 |
|------|------|
| `{N}` | 章节号（1/2/3...） |
| `{总章数}` | 目标章数 |
| `{前情摘要}` | 批量模式：近6章详记+更早概要；试水模式：逐章递增 |
| `{框架内容}` | Phase 1 输出的完整框架（含角色名表+关键事件锚点表） |
| `{章纲内容}` | Phase 2 输出的当前章节章纲（章名+核心事件+情节点序列+目标情绪+章首钩子+爽点+章尾钩子+字数目标） |
| `{禁用词列表}` | 从 `banned-words.md` 提取的一级禁用词（情态/动作/表情/心理类，控制在 20 词以内，避免挤占推理空间） |

## Phase 4：合并输出

Phase 3 的 agent 返回后，主线程执行以下步骤：

### Step 0：字数校验（主线程兜底）

⚠️ **即使 agent 已自检，主线程仍必须二次校验，超标直接截断。**

```powershell
# 字数统计（汉字+标点+英文字母，排除空格换行）
(Get-Content '文件路径' -Raw -Encoding UTF8) -replace '[\s]','' | Measure-Object -Character | Select-Object -ExpandProperty Characters
```

对每章正文执行：
1. 统计字数
2. 如果 > 2400：按段落边界从末尾截断，保留前 2400 字（截断到最近的完整段落）
3. 如果 < 1600：在终端警告「第N章字数不足，需补充」
4. 截断后的正文才写入最终文件

### Step 1：读取临时文件

从 `temp_{书名}_ch{N}.txt` 读取各章正文。

⚠️ 临时文件命名规则：`temp_{书名}_ch{1,2,3}.txt`，书名从框架中取，去掉特殊字符（空格保留为下划线）。合并完成后删除临时文件。

### 试水模式：合并为TXT

写入 `仿写试水库/试水_{新书名}.txt`，格式：

```
========================================
{新书名}
========================================
作者：AI仿写
题材：{题材}

【简介】
{简介}

========================================
【仿写框架】（供 /story-long-write 开书时提取）
========================================
{Phase 1 输出的完整框架}

========================================
第1章 {章名}
========================================
{正文}

----------------------------------------
第2章 {章名}
----------------------------------------
{正文}

----------------------------------------
第3章 {章名}
----------------------------------------
{正文}

========================================
【后续方向】
- 第4章预告：[简述]
========================================
```

### 批量模式：逐章写入正文目录

每批4章写完后，主线程执行：

1. **读取临时文件**：从 `temp_{书名}_ch{N}.txt` 读取每章正文
2. **质量检查**：
   - 字数校验：用 `python skills/story-rewrite/tools/wordcount.py` 统计
   - 禁用词扫描：对照 banned-words.md
   - 钩子检查：确认章尾有钩子
3. **写入正文目录**：`{书名}/正文/第{NNN}章_{章名}.md`
4. **删除临时文件**
5. **更新追踪文件**：
   - 追踪/伏笔.md（新增/回收伏笔）
   - 追踪/时间线.md（记录事件时序）
   - 追踪/角色状态.md（角色状态变化）
   - 追踪/上下文.md（进度摘要）

### 摘要压缩机制（批量模式专用）

**目的**：防止上下文溢出，保持连贯性。

**规则**：
- 近6章：逐章详记（事件+状态变化+伏笔）
- 更早章节：压缩为概要（每10章一段，含核心事件+角色状态变化）
- 卷级总览：每卷一句话总结

**压缩格式**：

```markdown
## 近6章详记（第{N-5}-{N}章）
第{N-5}章：{事件摘要}
第{N-4}章：{事件摘要}
...
第{N}章：{事件摘要}

## 十章概要
第1-10章 | {核心事件} | {角色状态变化}
第11-20章 | {核心事件} | {角色状态变化}
...

## 卷级总览
第一卷·{卷名} | {主线进展} | {关键转折}
```

## Phase 5：全书质量检查（批量模式专用）

所有章节写完后，执行全局一致性检查：

### 1. 禁用词扫描

用 grep 扫描正文目录，对照 `banned-words.md` 一级禁用词列表逐词检查。

### 2. 标题去重检查

扫描所有章节标题，发现同名或明显重复时，按本章核心事件改名。

### 3. 伏笔状态检查

检查追踪/伏笔.md，确认：
- 所有已埋伏笔状态正确
- 无断线伏笔（预计回收章节已过但未回收）
- 伏笔密度合理（每卷15-25条）

### 4. 时间线自洽检查

检查追踪/时间线.md，确认：
- 时间线连续，无跳跃
- 事件时序合理
- 无时间矛盾

### 5. 角色属性一致性检查

检查追踪/角色状态.md，确认：
- 角色属性前后一致
- 关系变化有铺垫
- 无突然的性格转变
