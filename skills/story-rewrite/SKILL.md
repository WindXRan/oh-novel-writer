# story-rewrite：结构仿写

## ⚠️ 入口规则（强制执行）

**触发条件**：用户输入包含以下任一关键词时，**必须**执行本 skill 流程：
- `/story-rewrite`、`/仿写`、`/结构仿写`
- 包含「仿写」二字的自然语言（如「仿写这本书」「帮我仿写」「仿写xxx.txt」）

**默认章数**：如果用户未指定目标章数，**默认为 3 章（试水）**。

**流程强制性**：
1. **必须执行 Phase 1 框架提取**，未完成框架不得进入 Phase 2
2. **必须使用 narrative-writer subagent 写作**，禁止主 agent 直接写正文
3. **必须输出到 `仿写试水库/` 目录**，禁止输出到项目根目录或其他位置
4. **必须执行质量检查**（validate-aigc + 禁用词扫描）

**核心信念：只仿"骨相"，不仿"皮相"。**

**架构原则（分角色）：**
- **Phase 1（主线程）**：读源文本 + narrator-persona.md，提取框架和语感样本
- **Phase 2（章纲）**：`story-architect` subagent，自带钩子/反转/情绪弧线知识
- **Phase 3（正文）**：`narrative-writer` subagent，自带场景展开/6 Gate去AI/节奏规则。
  Phase 3 agent **可读源文本前 3 章学风格**（不学情节），章纲已锁定事件，不会抄。
- **主线程不做** narrative-writer.md 的预提取——agent 按需自读参考文件。

---

## ⚠️ 核心约束

### 反调色盘（防止"扒骨肉"）

仿写必须在以下维度上与源文本**完全不同**：

| 维度 | 说明 | 检查方法 |
|------|------|----------|
| **核心设定** | 系统/金手指/世界观的核心机制不能相同 | 源文本"每天X千万只能在某地花"→新书不能用"每天X万只能在某地花"这种换数字的套路 |
| **核心冲突** | 驱动故事的根本矛盾不能同类 | 源文本"消费系统"→新书不能用"消费系统"，换成别的（记忆系统/技能系统/倒计时系统等） |
| **角色原型** | 主角的身份/处境/动机不能同类 | 源文本"大城市辞职回乡的打工人"→新书不能也是"大城市辞职回乡的打工人" |
| **开篇方式** | 第1章的核心事件不能同类 | 源文本"体检发现高血压"→新书不能也是"体检/看病/猝死"开篇 |
| **破冰方式** | 男女主第一次互动的方式不能同类 | 源文本"送物被拒"→新书不能也是"送物被拒" |
| **节奏安排** | 各章的情绪弧线不能对齐 | 源文本一章写重生前，新书必须压缩或改变 |

**自检公式**：框架生成后，把新书的"核心设定+核心冲突+角色原型"和源文本的放在一起对比。如果一个不了解网文的人看了会觉得"这不就是同一本书吗"，必须重写。

**红线**：以下情况直接判定为"换皮"，必须推翻重来：
- 系统/金手指的核心机制相同（只是换了金额/地点/名称）
- 主角的身份处境相同（只是换了职业/城市）
- 开篇事件相同（只是换了具体疾病/意外类型）
- 配角关系网相同（三叔→二叔，只是换了个称呼）

### 时间线连续

Phase 1 框架只提供「情绪弧线+叙事功能」，不提供具体场景。Phase 2 生成章纲锁定关键事件。Phase 3 agent 在章纲内自由发挥。

## 流程总览

```
/story-rewrite {源文本路径} {目标章数}    # 目标章数不填则默认 3（试水）

Phase 1（主线程，~2min）    读源文本 + 提取框架 ← ⚠️ 必须完成，否则阻断
Phase 2（3 agent 并行）     写前3章（试水）→ 输出到 仿写试水库/试水_{书名}.txt
         ↓ 用户确认
Phase 3（循环，每批3章）    从第4章续写 → 输出到 {书名}/正文/ → 质量检查 → 滚动到完本
Phase 4（结尾）             全局一致性检查 + 禁用词扫描
```

**试水不是开书。** 试水阶段输出到 `仿写试水库/`，格式为单个 `试水_*.txt` 文件（包含简介+框架+正文）。用户确认后才进入正式写作，输出到 `{书名}/正文/`。

### ⚠️ 检查点（阻断机制）

| 检查点 | 条件 | 阻断动作 |
|--------|------|----------|
| CP-1 | Phase 1 框架未完成 | 禁止启动 Phase 2 agent |
| CP-2 | 框架缺少语感特征/语感样本 | 返回 Step 1 补充提取 |
| CP-3 | 反调色盘自检不通过 | 重新生成框架 |
| CP-4 | Phase 2 输出路径不在 `仿写试水库/` | 重新写入正确路径 |
| CP-5 | 质量检查未执行 | 禁止输出最终文件 |

## 入口

用户输入：`/story-rewrite {源文本路径} {目标章数}`

示例：`/story-rewrite 女配一睁眼，失忆男主冷脸洗床单.txt 20`

**参数解析规则**：
1. 如果只提供源文本路径，未指定目标章数 → **默认 3 章（试水）**
2. 如果用户说「仿写 xxx.txt」但没说章数 → **默认 3 章（试水）**
3. 只有用户明确说「写20章」「续写到完本」等，才使用用户指定的章数

**⚠️ 强制流程**（每一步都必须执行，不可跳过）：

```
1. Phase 1: 读取源文本 + 生成仿写框架
   └─ ⚠️ 检查点 CP-1：框架必须完整，否则阻断
2. Phase 2: 写前3章（试水）
   - ⚠️ 必须并行启动 3 个 narrative-writer subagent
   - ⚠️ 禁止主 agent 直接写正文
   - 每章质量检查（字数+禁用词+validate-aigc）
   - 不达标则重写（最多2次）
   - ⚠️ 合并为单个文件：仿写试水库/试水_{书名}.txt
   - ⚠️ 检查点 CP-4：输出路径必须在 仿写试水库/
   - 文件内容：简介 + 仿写框架 + 前3章正文
   - 输出试水结果，询问用户是否满意
3. 用户确认满意 → 进入 Phase 3（正式开书）
   用户不满意 → 调整框架，重写前3章
4. Phase 3: 从第4章开始续写（每批3章）
   - 生成章纲（从第4章到目标章数）
   - 按批并行写作（每批3个 narrative-writer subagent）
   - 每章质量检查
   - 写入 {书名}/正文/第{NNN}章_{章名}.md
   - 压缩前文摘要（近6章详记+更早概要）
5. Phase 4: 全书质量检查
```

## Phase 1：结构提取 + 预加载（⚠️ 必须完成才能进入 Phase 2）

**用户只提供源文本路径，剩下全部自主完成。**

主线程收到源文本后，**同步完成以下操作**（不启动 agent，直接在主线程做）：

### ⚠️ Phase 1 强制性说明

1. **Phase 1 是阻断点**：未完成框架提取，禁止启动 Phase 2 的 narrative-writer agent
2. **框架必须包含**：语感特征（3-5个）+ 语感样本（3-5段）+ 反调色盘自检通过
3. **输出到内存**：框架内容保存在主线程变量中，供 Phase 2 prompt 构造使用

### Step 1：读取源文本

先运行自动提取脚本获取初步分析：

```powershell
python skills/story-rewrite/tools/extract-style.py <源文本路径>
```

输出 JSON 包含：人格类型、节奏特征、语感样本。主线程在此基础上补充以下手动分析：

读取前3章 + 最后1章，记录：
- 源书名、题材、章数
- 开篇方式（具体场景 → 抽象为叙事功能）
- 破冰方式（具体事件 → 抽象为叙事功能）
- 节奏安排（各章情绪基调）
- 人设标签（女主/男主/配角/反派）
- 结局走向
- **叙述者风格**（脚本输出 + 人工校验）
  1. 叙述者是否跳出来评论（有→吐槽型/冷幽默型/故事人型；无→沉浸型/冷峻型）
  2. 是否有现代人视角/网感语言（强→吐槽型；中→冷幽默/沉浸型；弱→故事人/冷峻型）
  3. 句子平均长度（<15字→吐槽型/冷峻型；15-25字→冷幽默/沉浸型；>25字→故事人型）
  4. 是否有第四面墙碎裂（有→吐槽型；偶尔→沉浸型；无→其他）
  5. 情绪外放程度（高→吐槽型；中→沉浸型；低→冷幽默/冷峻型）
  6. 口语化程度（极高→吐槽型/故事人型；中→冷幽默/沉浸型；低→冷峻型）
   7. **从源文本前3章提取 3-5 个核心节奏特征**（不是具体句子，是选词和节奏倾向），作为仿写的"语感特征"。每个特征用一句话描述 + 一个简短示例（不超过10字）。示例：
      - 信息直给：不铺垫，第一句直接扔事实（"再睁眼，入目是一盏水晶吊灯"）
      - 段落极短：1-3句一段，没有4句以上的段落
      - 对话省略标签：超过70%的对话不写"他说""她说"
      - 跳过过渡：不写"她站起来走了出去"，直接切到下一场景
      - 吐槽打断叙事：吐槽出现在段落中间打断叙事，不在段落末尾当补丁
   8. **从源文本前3章提取 3-5 段语感样本**（原文片段，每段 30-100 字），供 Phase 3 narrative-writer 做风格参照。选标准：
      - 选最有辨识度的段落（读者一眼能认出"这是这本书的味道"）
      - 涵盖不同场景类型（对话/内心/动作/描写各 1 段）
      - 不选情节推动段落（如高潮反转），选日常段落（能体现 baseline 语感）
      - 每段标注风格标签（如"内心独白型""吐槽打断型""对话省略标签型"）

### Step 2：参考文件准备

Phase 3 使用 `narrative-writer` subagent，它自带参考文件体系（按需读取），主线程**不需要**预提取 narrative-writer.md。

主线程只准备以下内容：

1. **banned-words 摘要**（可选）：从 `{项目绝对路径}/skills/story-deslop/references/banned-words.md` 提取**一级禁用词 20 词以内**，仅用作 Phase 4 写后扫描，**不注入 Phase 3 prompt**
2. **语感样本**：从源文本前3章提取 3-5 段最有辨识度的原文片段（每段 30-100 字），用于 Phase 3 prompt 的`{语感样本}`占位符。不同书用不同样本，不写死

⚠️ **权限限制**：`Glob` 搜索范围必须限制在项目目录内，直接拼接项目绝对路径 `Read`。

### Step 3：生成仿写框架

输出以下框架（只保留抽象模式，禁止具体场景）：

```
- 新书名：
- 新主角：（名字+核心标签，必须与源文本不同）
- 新背景：（时代/世界观，必须与源文本不同时代/不同地域）
- 核心冲突：（1句话）
- 开篇叙事功能：（抽象，如「铺垫→创伤回忆→重生契机」）
- 破冰叙事功能：（抽象，如「主动靠近→遭遇冷遇→坚持」）
- 开篇差异点：（必须与源文本不同）
- 破冰差异点：（必须与源文本不同）
- 节奏对齐表（3章）：
  - 第1章：情绪基调 / 叙事功能 / 弧线走向
  - 第2章：情绪基调 / 叙事功能 / 弧线走向
  - 第3章：情绪基调 / 叙事功能 / 弧线走向
- 人物关系模式（抽象）：
- 信息差设计：
- 核心差异对照表（时代/地域/职业/身份层面）：
- 角色名表（主线程必须统一，禁止 Agent 自行改名）：
  - 女主：{名字}
  - 男主：{名字}
  - 配角A：{名字+功能}
  - 配角B：{名字+功能}
- 题材标签：
- **叙述者语感样本**：（从源文本前3章提取的 3-5 段原文片段，每段 30-100 字）
- **叙述者语感特征**：（从源文本提取的 3-5 个核心节奏特征，每个是一句话描述+简短示例，不是具体句子。示例见 Step 1.7）
- **叙述者语感样本**：（Phase 1 Step 1.8 提取的 3-5 段原文片段，每段 30-100 字 + 风格标签）

## 关键事件锚点表（Phase 1 主线程填写，Phase 2 章纲展开，Phase 3 agent执行）
- 第1章锚点：{必须发生的事件A}，{必须发生的事件B}，本章以{悬念/情绪点}收尾
- 第2章锚点：时间线{紧接第1章/次日/数日后}，{必须发生的事件A}，{必须发生的事件B}
- 第3章锚点：时间线{紧接第2章/次日/数日后}，{必须发生的事件A}，{必须发生的事件B}
```

**自检**：框架给没读过源文本的人，能据此写出完全不同但结构相似的故事吗？不能 → 重新抽象化。**叙述者语感特征是否已填入框架？** 没有 → 回到 Step 1 补充提取。

## Phase 2：试水写作（前3章）

框架生成后，**直接启动 3 个 `task(subagent_type: "narrative-writer")` 并行写前3章**。

### ⚠️ Phase 2 强制性说明

1. **必须使用 narrative-writer subagent**：禁止主 agent 直接写正文，必须通过 `task(subagent_type: "narrative-writer")` 调用
2. **必须并行启动 3 个 agent**：每章一个 agent，同时启动
3. **必须输出到 `仿写试水库/`**：最终文件路径必须是 `仿写试水库/试水_{书名}.txt`
4. **必须执行质量检查**：每章都要运行 validate-aigc 和禁用词扫描

**试水阶段不需要章纲**。框架中的「关键事件锚点表」已锁定每章必须发生的事件，narrative-writer 在锚点范围内自由发挥。

每个 agent 的 prompt 由以下模板生成：

```
# ⚠️ 叙事视角：必须第三人称

**长篇网文必须用第三人称叙述（角色名/"她/他"），绝对不能用第一人称（"我/我们"）。**
违反即重写。

---

## 你的声音

你不是在"写小说"。你是在随口讲一个你刚看到的故事。
你没有"角色设定"，你只是在跟朋友聊天。

模仿下面这些片段的感觉（不是模仿具体句子，是模仿那种"随口说"的语气）：
{语感样本1}
{语感样本2}
{语感样本3}

想吐槽就吐槽，想沉默就沉默。没有规则。

- 弧线走向：{节奏对齐表中的弧线走向}

## ⚠️ 字数硬性要求（最重要！）
- **目标：2000字**
- **通过范围：1800-2200字**
- **绝对上限：2400字**（超过即被截断）
- **写到1800字时检查余量**：开始准备收尾，不能再展开新情节点
- **写到2000字时必须收束**：写收尾句（悬念/情绪/场景切换）
- **收尾信号**：当正文接近1800字时，立即写一个收尾段落（可以是：悬念句、情绪句、场景切换句）
- **字数检查**：写完后估算字数（1个中文字符=1字），如果超过2200字，删减到2200字以内
- **宁可短一点也不要超**：1800字通过，2000字目标，2400字红线

## ⚠️ 段落节奏控制（解决段落标准差和连续同长度问题）
- **段落长度必须有变化**：不能每段都是1-2句，每3-5个短段落后必须有1个3句以上的长段落
- **禁止连续3个以上相同长度的段落**：短-短-长-短-长，打乱节奏
- **超短句不能连续3个**：连续2个超短句（≤5字）后，第3句必须是15字以上的长句
- **场景描写用长段落**：环境描写、心理描写用3-5句的段落，不要拆成单句

## ⚠️ 第一人称控制（解决"我"字过多问题）
- **叙述必须第三人称**：用"她想""她觉得"代替"我想""我觉得"
- **角色内心独白也用第三人称**：不要写"我应该这样做"，写"她觉得自己应该这样做"
- **全章"我"字出现不超过5次**：只在角色对话中允许"我"出现
- **自检**：写完后数一下"我"字数量，超过5个就改掉

## 格式
- 第一行写：`第{N}章 {章名}`（独占一行）
- 返回正文即可，不要额外注释

## ⚠️ 字数自检（写完后必须执行）
1. 数一下正文有多少个中文字符（不含标点和空格）
2. 如果超过2200字，删减到2200字以内
3. 如果不足1800字，扩展场景细节
4. **最终字数必须在1800-2200字之间**
```

**narrative-writer 会自动**（不需要在主线程约束）：
- 应用场景展开法（每章限1次完整三层展开）
- 执行核心节奏规则（动作不解释、内心独白砍半、跳过一个环节、每段换节奏）
- 执行6 Gate去AI流程（禁用词、句式打碎、心理外化、节奏打碎、去腔调、去升华）
- 控制字数达标
- 如果语感样本包含吐槽型/冷幽默型人格，叙事中会自然带出人格特征

**⚠️ 主线程 Prompt 构造检查清单**（每次 spawn agent 前必须逐项核对）：
- [ ] 字数范围是否为 1800-2200（不是 1600-2400）？
- [ ] 是否包含「写到1800字时检查余量」的收尾指令？
- [ ] 是否包含段落节奏控制规则（每3-5个短段落后插入长段落）？
- [ ] 是否包含超短句连续限制（最多2个，第3句变长）？
- [ ] 语感样本是否填入（不能遗漏）？
- [ ] 角色名表是否完整（不能让 agent 自行改名）？
- [ ] 本章锚点是否明确（不能模糊）？

**⚠️ AIGC验证阈值说明**（已根据实测调整）：
| 检查项 | 阈值 | 说明 |
|--------|------|------|
| 段落标准差 | ≥10 | 网文短段落风格，10即可通过 |
| 连续同长度 | ≤10段 | 按字符数范围（短≤15/中15-40/长>40）检查 |
| 超短句连续 | ≤2个 | 连续2个超短句后第3句必须长句 |
| 第一人称 | 每500字≤1次 | 排除对话后按比例计算，最低10次 |

### Step 1：生成章纲

**章纲生成 prompt**：

```

你拥有以下参考文件，按需读取：

## 章纲格式要求
每章输出以下内容：
- 章名：{章名}
- 时间线：{本章发生在什么时间}
- 核心事件：{一句话概括本章}
- 情节点序列：3-5个情节点，每个写清"谁做了什么"（约200-300字/个情节点）

- 第4-6章：发展深化，矛盾升级
- 第7-9章：第一次高潮，关键转折

- 第16-18章：最终对决，矛盾解决

```

每批 3 章并行，启动 3 个 `task(subagent_type: "narrative-writer")`。

prompt 模板与 Phase 2 相同，但「本章锚点」替换为「本章章纲」（从章纲文件中读取对应章节）。

**narrative-writer 会自动**（不需要在主线程约束）：
- 应用场景展开法（每章限1次完整三层展开）

- 执行6 Gate去AI流程（禁用词、句式打碎、心理外化、节奏打碎、去腔调、去升华）
- 控制字数达标
- 如果语感样本包含吐槽型/冷幽默型人格，叙事中会自然带出人格特征

### ⚠️ 关键事件锚定规则（防止多agent各自为政）

**问题根因**：多agent并行时，如果prompt没有锁定关键事件，每个agent会自行设计，导致章节间时间线断裂、事件版本冲突、人物行为矛盾。

**解法**：
1. Phase 1 框架包含「关键事件锚点表」，主线程在生成框架时把每章必须发生的1-2个关键事件锁死
2. Phase 2 生成章纲，将锚点展开为具体情节点（时间线+事件+钩子）
3. Phase 3 agent prompt 注入章纲内容，agent在章纲内自由发挥

锚点是「叙事功能+关键事件」层面的抽象描述，章纲是具体的情节点，agent在章纲框架内自行设计场景。

### 主线程构建 prompt 时的变量填充

| 变量 | 说明 |
|------|------|
| `{N}` | 章节号（1/2/3...） |
| `{总章数}` | 目标章数 |
| `{前情摘要}` | 近6章详记+更早概要（第1-3章为空） |
| `{语感样本1/2/3}` | Phase 1 Step 1.8 从源文本提取的原文片段（30-100字） |
| `{特征1/2/3}` | Phase 1 Step 1.7 从源文本提取的核心节奏特征 |
| `{章纲内容}` | Phase 2 输出的当前章节章纲（章名+核心事件+情节点序列+目标情绪+章首钩子+爽点+章尾钩子+字数目标） |
| `{源文本路径}` | 源文本文件的完整路径，供 narrative-writer 读前3章学风格 |

## Phase 4：合并输出

Phase 3 的 agent 返回后，主线程执行以下步骤：

### Step 0：字数校验（主线程全权负责）

**agent 不校验字数，主线程兜底。** 超标截断，不足警告。

```powershell
# 字数统计（汉字+标点+英文字母，排除空格换行）
(Get-Content '文件路径' -Raw -Encoding UTF8) -replace '[\s]','' | Measure-Object -Character | Select-Object -ExpandProperty Characters
```

对每章正文执行：
1. 统计字数
2. 如果 > 2400：按段落边界从末尾截断，保留前 2400 字（截断到最近的完整段落）
3. 如果 < 1800：在终端警告「第N章字数不足，需补充」
4. 截断后的正文才写入最终文件

### Step 1：读取临时文件

从 `temp_{书名}_ch{N}.txt` 读取各章正文。

⚠️ 临时文件命名规则：`temp_{书名}_ch{1,2,3}.txt`，书名从框架中取，去掉特殊字符（空格保留为下划线）。合并完成后删除临时文件。

### Step X：AIGC 结构特征验证（写入前必过）

运行统一验证脚本，检查 10 项结构特征：

```powershell
.\skills\story-rewrite\tools\validate-aigc.ps1 -Path 'temp_{书名}_ch{N}.txt'
```

检查项：
| # | 检查项 | 达标标准 | 不达标处理 |
|---|--------|----------|-----------|
| 1 | 情绪模板词 | 全章 ≤1 次 | 替换为具体动作 |
| 2 | 内心独白链 | ≤2 处 | 砍推理，留 2 句+动作 |
| 3 | 对话标签密度 | ≤3/100字 | 去掉多余标签，用动作替 |
| 4 | 段落标准差 | ≥30 | 拆或并段落制造落差 |
| 5 | 单句段数 | ≥3 个 | 拆部分段为单句 |
| 6 | 连续同句数段 | ≤2 段 | 打乱段落结构 |
| 7 | 网络梗频率 | ≤2 个 | 替换为具体判断 |
| 8 | 禁用词 | 0 个 | 替换 |
| 9 | 超短句连续 | ≤2 个 | 第 3 句变长 |
| 10 | 第一人称 | ≤5 次 | 改为第三人称 |

**触发规则**：
- 前3章（试水）必须执行，观察 narrative-writer 输出质量
- 如有警告 → 记录到终端，后续在 persona 定义中微调
- 无警告 → 人格注入生效，后续批次可以跳过此扫描

### 摘要压缩机制

**目的**：防止上下文溢出，保持连贯性。

**规则**：
- 近6章：逐章详记（事件+状态变化+伏笔）
- 更早章节：压缩为概要（每10章一段，含核心事件+角色状态变化）
- 卷级总览：每卷一句话总结

**压缩格式**：

```markdown
## 近6章详记（第{N-5}-{N}章）

第{N-4}章：{事件摘要}
...
第{N}章：{事件摘要}

## 十章概要
第1-10章 | {核心事件} | {角色状态变化}
第11-20章 | {核心事件} | {角色状态变化}
...

## 卷级总览
第一卷·{卷名} | {主线进展} | {关键转折}
```


### 2. 禁用词扫描

用 grep 扫描正文目录，对照 `banned-words.md` 一级禁用词列表逐词检查。

### 3. 标题去重检查

扫描所有章节标题，发现同名或明显重复时，按本章核心事件改名。

检查追踪/伏笔.md，确认：
- 所有已埋伏笔状态正确
- 无断线伏笔（预计回收章节已过但未回收）
- 伏笔密度合理（每卷15-25条）

### 4. 时间线自洽检查

检查追踪/时间线.md，确认：
- 时间线连续，无跳跃
- 事件时序合理
- 无时间矛盾

### 5. 角色属性一致性检查

检查追踪/角色状态.md，确认：
- 角色属性前后一致
- 关系变化有铺垫
- 无突然的性格转变
