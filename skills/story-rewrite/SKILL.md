# story-rewrite：结构仿写

**核心信念：只仿"骨相"，不仿"皮相"。**

**架构原则（分角色）：**
- **Phase 1（主线程）**：读源文本 + narrator-persona.md，提取框架和语感样本
- **Phase 2（章纲）**：`story-architect` subagent，自带钩子/反转/情绪弧线知识
- **Phase 3（正文）**：`narrative-writer` subagent，自带场景展开/6 Gate去AI/节奏规则。
  Phase 3 agent **可读源文本前 3 章学风格**（不学情节），章纲已锁定事件，不会抄。
- **主线程不做** narrative-writer.md 的预提取——agent 按需自读参考文件。

---

## ⚠️ 核心约束

### 反调色盘（防止"扒骨肉"）

仿写必须在以下3个维度上与源文本**完全不同**：

| 维度 | 说明 |
|------|------|
| **开篇方式** | 如源文本用心理治疗/催眠开篇，新书必须用其他方式（商战/遗物/急救等） |
| **破冰方式** | 如源文本用送物被拒，新书必须用其他方式（替解围/共同任务/意外接触等） |
| **节奏安排** | 如源文本一章写重生前，新书必须压缩或改变节奏 |

### 时间线连续

Phase 1 框架只提供「情绪弧线+叙事功能」，不提供具体场景。Phase 2 生成章纲锁定关键事件。Phase 3 agent 在章纲内自由发挥。

---

## 流程总览

```
/story-rewrite {源文本路径} {目标章数}

Phase 1（主线程，~2min）    读源文本 + 提取框架 + 生成全书章纲
Phase 2（3 agent 并行）     写前3章（试水）→ 输出到 {书名}/正文/
         ↓ 用户确认
Phase 3（循环，每批3章）    从第4章续写 → 质量检查 → 滚动到完本
Phase 4（结尾）             全局一致性检查 + 禁用词扫描
```

**试水就是开书。** 前3章直接写入正文目录，不是临时文件。用户确认后直接续写，不需要重跑。

---

## 入口

用户输入：`/story-rewrite {源文本路径} {目标章数}`

示例：`/story-rewrite 女配一睁眼，失忆男主冷脸洗床单.txt 20`

**主流程**：

```
1. Phase 1: 读取源文本 + 生成仿写框架 + 全书章纲（一次性）
2. Phase 2: 写前3章（试水）
   - 并行启动3个agent写作
   - 每章质量检查（字数+禁用词+validate-aigc）
   - 不达标则重写（最多2次）
   - 写入 {书名}/正文/第001章_{章名}.md ~ 第003章_{章名}.md
   - 输出试水结果，询问用户是否满意
3. 用户确认满意 → 继续 Phase 3
   用户不满意 → 调整框架/章纲，重写前3章
4. Phase 3: 从第4章开始续写（每批3章）
   loop:
     读取下3章章纲
     并行启动3个agent写作
     每章质量检查
     不达标则重写（最多2次）
     写入 {书名}/正文/第{NNN}章_{章名}.md
     压缩前文摘要（近6章详记+更早概要）
     if 已完成目标章数 → 结束循环
     else → 继续循环
5. Phase 4: 全书质量检查
```

## Phase 1：结构提取 + 预加载

**用户只提供源文本路径，剩下全部自主完成。**

主线程收到源文本后，**同步完成以下操作**（不启动 agent，直接在主线程做）：

### Step 1：读取源文本

先运行自动提取脚本获取初步分析：

```powershell
python skills/story-rewrite/tools/extract-style.py <源文本路径>
```

输出 JSON 包含：人格类型、节奏特征、语感样本。主线程在此基础上补充以下手动分析：

读取前3章 + 最后1章，记录：
- 源书名、题材、章数
- 开篇方式（具体场景 → 抽象为叙事功能）
- 破冰方式（具体事件 → 抽象为叙事功能）
- 节奏安排（各章情绪基调）
- 人设标签（女主/男主/配角/反派）
- 结局走向
- **叙述者风格**（脚本输出 + 人工校验）
  1. 叙述者是否跳出来评论（有→吐槽型/冷幽默型/故事人型；无→沉浸型/冷峻型）
  2. 是否有现代人视角/网感语言（强→吐槽型；中→冷幽默/沉浸型；弱→故事人/冷峻型）
  3. 句子平均长度（<15字→吐槽型/冷峻型；15-25字→冷幽默/沉浸型；>25字→故事人型）
  4. 是否有第四面墙碎裂（有→吐槽型；偶尔→沉浸型；无→其他）
  5. 情绪外放程度（高→吐槽型；中→沉浸型；低→冷幽默/冷峻型）
  6. 口语化程度（极高→吐槽型/故事人型；中→冷幽默/沉浸型；低→冷峻型）
   7. **从源文本前3章提取 3-5 个核心节奏特征**（不是具体句子，是选词和节奏倾向），作为仿写的"语感特征"。每个特征用一句话描述 + 一个简短示例（不超过10字）。示例：
      - 信息直给：不铺垫，第一句直接扔事实（"再睁眼，入目是一盏水晶吊灯"）
      - 段落极短：1-3句一段，没有4句以上的段落
      - 对话省略标签：超过70%的对话不写"他说""她说"
      - 跳过过渡：不写"她站起来走了出去"，直接切到下一场景
      - 吐槽打断叙事：吐槽出现在段落中间打断叙事，不在段落末尾当补丁
   8. **从源文本前3章提取 3-5 段语感样本**（原文片段，每段 30-100 字），供 Phase 3 narrative-writer 做风格参照。选标准：
      - 选最有辨识度的段落（读者一眼能认出"这是这本书的味道"）
      - 涵盖不同场景类型（对话/内心/动作/描写各 1 段）
      - 不选情节推动段落（如高潮反转），选日常段落（能体现 baseline 语感）
      - 每段标注风格标签（如"内心独白型""吐槽打断型""对话省略标签型"）

### Step 2：参考文件准备

Phase 3 使用 `narrative-writer` subagent，它自带参考文件体系（按需读取），主线程**不需要**预提取 narrative-writer.md。

主线程只准备以下内容：

1. **banned-words 摘要**（可选）：从 `{项目绝对路径}/skills/story-deslop/references/banned-words.md` 提取**一级禁用词 20 词以内**，仅用作 Phase 4 写后扫描，**不注入 Phase 3 prompt**
2. **人格定义**：从 `{项目绝对路径}/skills/story-rewrite/references/narrator-persona.md` 根据 Step 1 分析匹配最接近的人格类型，**保留完整定义文字**（"你是谁"块需要完整人格描述），用于 Phase 3 prompt 的`{人格完整定义}`占位符。如果源文本风格介于两种人格之间，允许混合

⚠️ **权限限制**：`Glob` 搜索范围必须限制在项目目录内，直接拼接项目绝对路径 `Read`。

输出以下框架（只保留抽象模式，禁止具体场景）：

```

- 新背景：（时代/世界观，必须与源文本不同时代/不同地域）
- 核心冲突：（1句话）

- 破冰叙事功能：（抽象，如「主动靠近→遭遇冷遇→坚持」）
- 开篇差异点：（必须与源文本不同）
- 破冰差异点：（必须与源文本不同）
- 节奏对齐表（3章）：
  - 第1章：情绪基调 / 叙事功能 / 弧线走向
  - 第2章：情绪基调 / 叙事功能 / 弧线走向
  - 第3章：情绪基调 / 叙事功能 / 弧线走向
- 人物关系模式（抽象）：
- 信息差设计：
- 核心差异对照表（时代/地域/职业/身份层面）：
- 角色名表（主线程必须统一，禁止 Agent 自行改名）：
  - 女主：{名字}
  - 男主：{名字}
  - 配角A：{名字+功能}
  - 配角B：{名字+功能}
- 题材标签：
- **叙述者人格**：（从 narrator-persona.md 匹配的人格类型名称 + 完整定义文字）
- **叙述者语感特征**：（从源文本提取的 3-5 个核心节奏特征，每个是一句话描述+简短示例，不是具体句子。示例见 Step 1.7）
- **叙述者语感样本**：（Phase 1 Step 1.8 提取的 3-5 段原文片段，每段 30-100 字 + 风格标签）

## 关键事件锚点表（Phase 1 主线程填写，Phase 2 章纲展开，Phase 3 agent执行）
- 第1章锚点：{必须发生的事件A}，{必须发生的事件B}，本章以{悬念/情绪点}收尾
- 第2章锚点：时间线{紧接第1章/次日/数日后}，{必须发生的事件A}，{必须发生的事件B}
- 第3章锚点：时间线{紧接第2章/次日/数日后}，{必须发生的事件A}，{必须发生的事件B}
```

**自检**：框架给没读过源文本的人，能据此写出完全不同但结构相似的故事吗？不能 → 重新抽象化。**叙述者语感特征是否已填入框架？** 没有 → 回到 Step 1 补充提取。

## Phase 2：章纲生成

框架生成后，启动 1 个 `task(subagent_type: "story-architect")` 生成全书章纲。

**一次性生成全书章纲**，存储到文件 `{书名}/大纲/章纲_全书.md`，后续按需读取。

**章纲生成 prompt（试水/批量通用）**：

```

- `story-long-write/references/outline-conflict.md`（AB交织法、高潮逆推）
- `story-long-write/references/outline-rhythm.md`（升级感三步设计法）

## 章纲格式要求
每章输出以下内容：
- 章名：{章名}
- 时间线：{本章发生在什么时间}
- 核心事件：{一句话概括本章}
- 情节点序列：3-5个情节点，每个写清"谁做了什么"（约200-300字/个情节点）

  必须拆成"触发→信息→回到当下"微循环，每个微循环写清触发方式和交付的信息内容。
  详见硬约束第9条。
- 目标情绪：{本章交付什么情绪}
- **章首钩子**：利用钩子章法设计（冲突前置/信息差钩/反常行为/悬念句等），具体到一句话
- **爽点**：{具体爽点}
- **章尾钩子**：参考章尾13式设计（突然揭示/紧急危机/两难抉择等），写具体内容 + 期待度（强/中/弱）
- 字数目标：{X} 字

## 硬约束
1. 章纲必须覆盖框架中的「关键事件锚点表」（前3章）
2. 后续章节必须延续前3章的情绪弧线和叙事功能
3. 章节之间时间线必须自然衔接
4. 禁止写具体场景描写，只写事件摘要
5. 角色名必须使用框架「角色名表」中的名字
6. 每章字数目标：2000字（通过范围 1600-2400）
7. 每3-4章必须有一个小高潮或转折点，参考反转工具箱设计
8. 每章章尾必须有钩子（强/中/弱）
9. **信息交付原则**：涉及信息密集的情节点（记忆导入/背景交代/世界观说明/过去闪回），
   章纲必须写明信息分几波交付，每波的触发方式（感官/物件/对话触发）和交付内容。
   禁止用"记忆涌入""了解了背景""知道了真相"等单句概括整块信息。
   这不是添加更多场景描写——是改变信息的交付节奏：信息不连续倒出，而是穿插在当下的动作和感受中逐波释放。
10. **期待感管理**：参考期待感管理六法则，确保每章结束时有续读动力

- 第16-18章：最终对决，矛盾解决
- 第19-20章：收尾，情感收束
```

### ⚠️ 关键事件锚定规则（防止多agent各自为政）

**问题根因**：多agent并行时，如果prompt没有锁定关键事件，每个agent会自行设计，导致章节间时间线断裂、事件版本冲突、人物行为矛盾。

**解法**：
1. Phase 1 框架包含「关键事件锚点表」，主线程在生成框架时把每章必须发生的1-2个关键事件锁死
2. Phase 2 生成章纲，将锚点展开为具体情节点（时间线+事件+钩子）
3. Phase 3 agent prompt 注入章纲内容，agent在章纲内自由发挥

锚点是「叙事功能+关键事件」层面的抽象描述，章纲是具体的情节点，agent在章纲框架内自行设计场景。

### 主线程构建 prompt 时的变量填充

| 变量 | 说明 |
|------|------|
| `{N}` | 章节号（1/2/3...） |
| `{总章数}` | 目标章数 |
| `{前情摘要}` | 近6章详记+更早概要（第1-3章为空） |
| `{人格类型名称}` | Phase 1 匹配的人格类型名称（如"吐槽型网文读者"） |
| `{人格完整定义}` | Phase 1 从 narrator-persona.md 提取的完整人格定义文字 |
| `{语感样本1/2/3}` | Phase 1 Step 1.8 从源文本提取的原文片段 + 风格标签 |
| `{特征1/2/3}` | Phase 1 Step 1.7 从源文本提取的核心节奏特征 |
| `{章纲内容}` | Phase 2 输出的当前章节章纲（章名+核心事件+情节点序列+目标情绪+章首钩子+爽点+章尾钩子+字数目标） |
| `{源文本路径}` | 源文本文件的完整路径，供 narrative-writer 读前3章学风格 |

## Phase 4：合并输出

Phase 3 的 agent 返回后，主线程执行以下步骤：

### Step 0：字数校验（主线程全权负责）

**agent 不校验字数，主线程兜底。** 超标截断，不足警告。

```powershell
# 字数统计（汉字+标点+英文字母，排除空格换行）
(Get-Content '文件路径' -Raw -Encoding UTF8) -replace '[\s]','' | Measure-Object -Character | Select-Object -ExpandProperty Characters
```

对每章正文执行：
1. 统计字数
2. 如果 > 2400：按段落边界从末尾截断，保留前 2400 字（截断到最近的完整段落）
3. 如果 < 1600：在终端警告「第N章字数不足，需补充」
4. 截断后的正文才写入最终文件

### Step 1：读取临时文件

从 `temp_{书名}_ch{N}.txt` 读取各章正文。

⚠️ 临时文件命名规则：`temp_{书名}_ch{1,2,3}.txt`，书名从框架中取，去掉特殊字符（空格保留为下划线）。合并完成后删除临时文件。

### Step X：AIGC 结构特征验证（写入前必过）

运行统一验证脚本，检查 10 项结构特征：

```powershell
.\skills\story-rewrite\tools\validate-aigc.ps1 -Path 'temp_{书名}_ch{N}.txt'
```

检查项：
| # | 检查项 | 达标标准 | 不达标处理 |
|---|--------|----------|-----------|
| 1 | 情绪模板词 | 全章 ≤1 次 | 替换为具体动作 |
| 2 | 内心独白链 | ≤2 处 | 砍推理，留 2 句+动作 |
| 3 | 对话标签密度 | ≤3/100字 | 去掉多余标签，用动作替 |
| 4 | 段落标准差 | ≥30 | 拆或并段落制造落差 |
| 5 | 单句段数 | ≥3 个 | 拆部分段为单句 |
| 6 | 连续同句数段 | ≤2 段 | 打乱段落结构 |
| 7 | 网络梗频率 | ≤2 个 | 替换为具体判断 |
| 8 | 禁用词 | 0 个 | 替换 |
| 9 | 超短句连续 | ≤2 个 | 第 3 句变长 |
| 10 | 第一人称 | ≤5 次 | 改为第三人称 |

**触发规则**：
- 前3章（试水）必须执行，观察 narrative-writer 输出质量
- 如有警告 → 记录到终端，后续在 persona 定义中微调
- 无警告 → 人格注入生效，后续批次可以跳过此扫描

### 写入正文目录

每章正文写入 `{书名}/正文/第{NNN}章_{章名}.md`，格式：

```markdown
# 第{N}章 {章名}

{正文}
```

- 章号用3位数字补零：001, 002, 003...
- 正文开头不重复章名（已在文件名和标题中）
- 写入后删除临时文件 `temp_{书名}_ch{N}.txt`

### 试水结果输出

前3章写完后，输出汇总：

```
========================================
试水完成：{新书名}
========================================
第001章 {章名} | {X}字
第002章 {章名} | {X}字
第003章 {章名} | {X}字
总字数：{X}字

质量扫描：{N}个问题
AIGC验证：通过/警告

正文目录：{书名}/正文/
========================================
是否满意？满意则继续写第4章起，不满意则调整框架重写。
```

### 每批写完后更新追踪文件

每批3章写入正文目录后，更新：
- 追踪/伏笔.md（新增/回收伏笔）
- 追踪/时间线.md（记录事件时序）
- 追踪/角色状态.md（角色状态变化）
- 追踪/上下文.md（进度摘要）

### 摘要压缩机制

**目的**：防止上下文溢出，保持连贯性。

**规则**：
- 近6章：逐章详记（事件+状态变化+伏笔）
- 更早章节：压缩为概要（每10章一段，含核心事件+角色状态变化）
- 卷级总览：每卷一句话总结

**压缩格式**：

```markdown
## 近6章详记（第{N-5}-{N}章）
第{N-5}章：{事件摘要}
第{N-4}章：{事件摘要}
...
第{N}章：{事件摘要}

## 十章概要
第1-10章 | {核心事件} | {角色状态变化}
第11-20章 | {核心事件} | {角色状态变化}
...

## 卷级总览
第一卷·{卷名} | {主线进展} | {关键转折}
```

## Phase 4：全书质量检查

所有章节写完后，执行以下检查：

### 1. 自动化检查（脚本）

```powershell
# AIGC 结构特征验证（10项检查）
python skills/story-rewrite/tools/validate-aigc.ps1 -Path '{书名}/正文/*.md'

# 人设一致性 + 古早梗 + 重复情绪检查
python skills/story-rewrite/tools/check-consistency.py '{书名}/正文/*.md'

# 质量扫描（标出可删除的AI废话）
python skills/story-rewrite/tools/scan-quality.py '{书名}/正文/*.md'
```

### 2. 禁用词扫描

用 grep 扫描正文目录，对照 `banned-words.md` 一级禁用词列表逐词检查。

### 3. 标题去重检查

扫描所有章节标题，发现同名或明显重复时，按本章核心事件改名。

检查追踪/伏笔.md，确认：
- 所有已埋伏笔状态正确
- 无断线伏笔（预计回收章节已过但未回收）
- 伏笔密度合理（每卷15-25条）

### 4. 时间线自洽检查

检查追踪/时间线.md，确认：
- 时间线连续，无跳跃
- 事件时序合理
- 无时间矛盾

### 5. 角色属性一致性检查

检查追踪/角色状态.md，确认：
- 角色属性前后一致
- 关系变化有铺垫
- 无突然的性格转变
